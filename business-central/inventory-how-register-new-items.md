# <a name="register-new-items"></a>Регистрация новых товаров

Основу вашего бизнеса составляют предметы, а также другие продукты, товары или услуги, которыми вы торгуете. Каждый товар должен быть зарегистрирован в виде карточки товара.

## <a name="to-create-a-new-item-card"></a>Создание новой карточки товара

В следующем видео показано, как создать товар на странице «Карточка товара». Однако вы также можете создавать новые товары путем копирования существующих. Подробнее см. в статье [Копирование существующих товаров для создания новых товаров](inventory-how-copy-items.md),

## <a name="use-item-templates"></a>Использование шаблонов товаров

Чтобы повторно использовать настройки для разных типов товаров при создании новых товаров, можно сохранять товары в качестве шаблонов товаров. Шаблоны товаров помогают ускорить процесс добавления новых товаров и способствуют единообразию данных о товарах. При регистрации нового товара появляется страница, на которой можно выбрать шаблон. После того как вы выберете шаблон, создаваемый товар заполняется его значениями параметров. Если шаблон товара только один, для новых товаров всегда используется этот шаблон. 

### <a name="save-an-item-card-as-an-item-template"></a>Сохранение карточки товара в качестве шаблона товара

1. На странице **Карточка товара** выберите действие **Сохранить как шаблон**. Открывается страница **Шаблон товара** с карточкой товара в виде шаблона.
2. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]
3. Чтобы повторно использовать измерения в шаблонах, выберите действие **Связанные**, затем выберите **Элемент**, а затем **Измерения**. Откроются **Размеры по умолчанию**  для выбранного элемента, в которых будут показаны все коды размеров, настроенные для элемента.
4. Измените или введите коды измерений, которые будут применяться к новым карточкам товаров, созданным с помощью шаблона.
5. Заполнив шаблон нового товара, нажмите кнопку **ОК**.

Шаблон товара добавляется в список шаблонов товаров, чтобы можно было использовать его для создания новых карточек товаров.

## <a name="types-of-items"></a>Типы предметов

В поле **Тип** на странице **Карточка товара** можно выбрать, для чего используется товар в вашем бизнесе, который влияет на уровень того, как вы управляете товаром в запасах.

* **Складируемый** товар означает, что товар является физической единицей, которой вы управляете и которая отслеживается в запасах.
* **Нескладируемые** товары — это физические единицы, которыми вы не управляете и которые не отслеживаются в запасах.
* **Элементы услуг** — это единицы рабочего времени, обычно используемые для регистрации продаж или покупок услуг.

Подробнее об этих типах товаров см. в статье [О типах товаров](inventory-about-item-types.md).

> [!TIP]
> Существуют также товары из каталога, которые похожи на нескладируемые товары в том смысле, что это товары, которые вы предлагаете клиентам, но не управляете ими, пока не продадите их. Подробнее см. в статье [Работа с товарами из каталога](inventory-how-work-nonstock-items.md).  

## <a name="inventory-costing"></a>Расчет себестоимости запасов

В поле **Метод учета себестоимости** вы настраиваете способ, при помощи которого вычисляется себестоимость единицы товара, делая допущения относительно физического движения товара через организацию. Предусмотрено пять методов учета себестоимости, в зависимости от типа товара. Подробнее об учете себестоимости см. в статье [Технические подробности: методы учета себестоимости](design-details-costing-methods.md).

> [!NOTE]
> Если выбран вариант **Среднее**, себестоимость единицы товара рассчитывается как средняя себестоимость единицы в каждый момент времени после покупки. Запасы переоцениваются из допущения, что все запасы продаются одновременно. При использовании этого варианта можно выбрать поле **Себестоимость единицы** на странице **Обзор расчета средней себестоимости** для просмотра транзакций, на основе которых рассчитана средняя себестоимость.

## <a name="set-up-item-substitutions"></a>Настройка замен товаров

Вы можете настроить товары, чтобы получить заменители, например, другие товары, которые можно использовать вместо исходного товара.

### <a name="to-make-an-item-substitution"></a>Создание замены товара

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Товары**, а затем выберите связанную ссылку.  
2. Найдите нужный элемент, а затем выберите **№.** чтобы открыть элемент карточка.  
3. Выберите действие **Связанный**, затем выберите **Товар**, потом **Замены**, чтобы открыть страницу **Ввод замены товара**.  
4. Выберите **Номер замены** и выберите товар замены из списка.
5. Заполните или измените другие поля на странице по мере необходимости.

Если запрошенное количество превышает количество, имеющееся на складе, появляется сообщение о наличии заменяющих товаров.

> [!NOTE]  
> Имейте в виду, что замены номенклатуры не приведет к автоматической замене номенклатуры на другую номенклатуру, например, при создании заказа на продажу или в спецификации. Вместо этого вы будете предупреждены о том, что вам доступна замена.

## <a name="prices-and-discounts"></a>Цены и скидки

Вы можете использовать специальные цены или скидки, которые вы предоставляете на товар на основе определенных критериев. Например, критерии включают клиента, минимальный объем заказа или дату окончания срока годности. Для настройки специальных цен выберите действие **Задать специальные цены** или **Задать специальные скидки**. Например, каждая строка страницы **Цены продажи** представляет специальную цену. Каждый столбец представляет критерий, который должен применяться для предоставления клиенту специальной цены, которую вы вводите в поле **Цена за единицу товара** на странице **Цены продажи**. Чтобы узнать больше о ценообразовании, перейдите в раздел [Запись цен продаж, скидок и соглашений об оплате](sales-how-record-sales-price-discount-payment-agreements.md).

## <a name="replenishment"></a>Пополнение

Вы можете указать способ поставки товаров:

* **Заказ на покупку** если вы хотите купить товары.
* **Заказ на сборку** или **заказ на производство**, если вы производите изделия собственными силами.

Существуют и другие настройки, дополняющие эти варианты.

### <a name="include-items-in-bills-of-materials"></a>Включение товаров в спецификации

Вы можете структурировать иерархии, в которых есть главный товар с входящими в его состав товарами-компонентами, в виде сборочных и производственных спецификаций. Подробнее о спецификациях см. в статье [Работа со спецификациями](inventory-how-work-BOMs.md).

### <a name="items-used-in-production-orders"></a>Товары, используемые в производственных заказах

Чтобы зарегистрировать товары, которые вы используете в производственных заказах, укажите систему пополнения как **Производственный заказ** на быстрой вкладке **Пополнение** . Дополнительные сведения см. в разделе [О производственных заказах](production-about-production-orders.md).  

### <a name="primary-and-alternate-vendors"></a>Основные и альтернативные поставщики

Если вы приобретаете один и тот же товар у нескольких поставщиков, можно связать этих поставщиков с товаром. Используйте действие **Продавцы** на странице **Карточка товара**, чтобы открыть страницу **Каталог поставщиков товара**. 

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок введите **Товары**, а затем выберите связанную ссылку.  
2. Выберите соответствующий товар, затем выберите действие **Изменить**.  
3. Выберите действие **Поставщики**.  
4. Выберите **№.** поле, а затем выберите поставщика, которого вы хотите настроить для товара.  
5. Заполнение оставшихся полей необязательно.  
6. Повторите шаги со 2 по 5 для каждого поставщика, у которого вы хотите приобретать товар.

Поставщики будут отображаться на странице **Каталог поставщиков товаров**, которая открывается с карточки товара, чтобы можно было легко выбрать другого поставщика.

Если вы покупаете один и тот же товар у нескольких поставщиков, вы можете дополнительно настроить цены и скидки.  Для получения дополнительной информации см. раздел [Запись специальных закупочных цен и скидок](purchasing-how-record-purchase-price-discount-payment-agreements.md).

## <a name="manage-inventory-in-warehouses"></a>Управление запасами на складе

При регистрации нового товара вы увидите поля, относящиеся к управлению запасами, в частности на экспресс-вкладке **Склад**. Если ваша организация не использует возможности управления запасами в [!INCLUDE [prod_short](includes/prod_short.md)], то вы можете игнорировать эти поля.  

Если ваша организация позже настроит управление складом, мы рекомендуем вам убедиться, что каждый существующий товар имеет правильную информацию в различных полях. Таким образом, складские процессы могут работать должным образом. Информация может включать такие поля, как **Код класса склада** или **Код шаблона размещения**. Дополнительные сведения см. в разделе [Настройка управления складом](warehouse-setup-warehouse.md).

Примеры использования полей на экспресс-вкладке **Планирование** прочитайте [Рекомендации по настройке. Параметры планирования](setup-best-practices-planning-parameters.md).  

## <a name="delete-item-cards"></a>Удаление карточек товаров

Если вы учли транзакцию для товара, удалить карточку нельзя, потому что операции книги могут быть необходимы для оценки стоимости запасов или аудита. Для удаления карточек товаров с операциями книги обратитесь к партнеру Майкрософт, который может сделать это посредством кода.
