# <a name="set-up-projects-prices-and-project-posting-groups"></a>Настройка проектов, цен и учетных групп проектов

Как менеджер проекта, вы можете настраивать проекты, которые определяют каждый из проектов, которыми вы управляете в [!INCLUDE[prod_short](includes/prod_short.md)]. Используйте страницу **Настройка проектов**, чтобы определить, как вы будете использовать функции, связанные с проектами.

Для каждого проекта необходимо задать различную информацию:

* Цены на товары, используемые в проекте
* Ресурсы проекта
* Счета ГК для проекта
* Учетные группы для проекта (обязательно)

## <a name="to-set-general-information-for-projects"></a>Настройка общей информации для проектов

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Настройка проекта**, а затем выберите соответствующую ссылку.
2. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

> [!NOTE]
> Переключатель **Применить связь использования по умолчанию** на странице **Настройка модуля «Проекты»** указывает, связываются ли операции книги проектов со строками планирования проектов по умолчанию. Включите переключатель, чтобы применить этот параметр ко всем новым проектам. Вы можете включить или отключить отслеживание потребления для конкретной проекта, включив или отключив переключатель **Применить связь использования** на странице **Карточка проекта**.

### <a name="specify-a-default-location-for-project-items"></a>Задание склада по умолчанию для товаров проекта

Вы можете сэкономить время на вводе данных, указав склад и ячейку для проектов на странице **Карточка проекта**. Когда вы создаете задачи проекта, строки планирования проекта и строки журнала проекта для проекта, склад и ячейка по умолчанию назначаются автоматически. Однако при необходимости вы можете изменить код склада и ячейку в задачах и строках.

Если вы зададите **Код ячейки для проекта** для склада, код ячейки будет подставляться при выборе кода этого склада. Если ваши складские процедуры предполагают складской подбор, вы также можете указать другие ячейки, из которых будут потребляться товары.

Эти поля используются по умолчанию при создании задач проекта. Существующие задачи проекта не меняются.

При использовании складов по умолчанию необходимо помнить следующее:

* Для задач проекта, если вы зададите **Код ячейки для проекта** для склада, код ячейки будет назначаться при выборе кода этого склада. Если ваши складские процедуры предполагают складской подбор, вы также можете указать другие ячейки, из которых будут потребляться товары.
* Для строк планирования проекта **Код склада** основан на значении, выбранном в строке планирования проекта при выборе товара. Если для задачи проекта не определен код ячейки, выбирается ячейка из содержимого ячейки по умолчанию. Оба эти значения можно изменить вручную.
* Для строк журнала проектов **Код склада** основан на значении, выбранном в строке журнала работ при выборе товара. Если для задачи проекта не определен код ячейки, выбирается ячейка из содержимого ячейки по умолчанию. Оба эти значения можно изменить вручную.

### <a name="invoice-multiple-customers-for-project-tasks"></a>Выставляйте счета за задачи проекта нескольким клиентам

Когда в проектах фигурирует несколько клиентов, выставление счетов нужным клиентам за нужные задачи может оказаться сложной задачей. [!INCLUDE [prod_short](includes/prod_short.md)] упрощает выставление счетов, позволяя указывать клиентов-плательщиков и клиентов-покупателей в каждой строке задач проекта, что позволяет автоматически генерировать счета для нужных клиентов. Подробнее о выставлении счетов нескольким клиентам см. в разделе [Выставление счетов одному или нескольким клиентам за задачи проекта](projects-how-create-jobs.md#invoice-one-or-more-customers-for-project-tasks).

### <a name="to-set-up-project-usage-tracking"></a>Настройка отслеживания потребления в проектах

При работе над проектом вам может потребоваться узнать, каким образом потребление сопоставляется с планом. Для просмотра потребления можно создать связь между строками планирования проекта и фактическим потреблением. Эта связь позволяет отслеживать ваши затраты и иметь представление о том, сколько работы осталось. По умолчанию тип строки планирования проекта — **Бюджет**, но использование типа строки **Бюджет и к оплате** оказывает схожий эффект.

После настройки отслеживания потребления путем включения переключателя **Применить связь использования по умолчанию** вы можете просматривать информацию в строке планирования проекта. Например, вы можете задать количество ресурса, товара или счета главной книги. Вы также можете задать количество для переноса в журнал проектов. Поле **Оставшееся количество** в строке планирования проекта показывает, какое количество остается для переноса и учета в журнале проекта.

>[!NOTE]
> Если в проекте установлен флажок **Применить связь использования**, а поле **Тип строки** в строке журнала проектов или строке покупки имеет значение **К оплате**, при учете журнала проектов или документа покупки создаются новые строки планирования проекта с типом строки **Бюджет и к оплате**.  
>
> Подробнее см. в статьях [Запись потребления для проектов](projects-how-record-job-usage.md) и [Управление запасами для проектов](projects-how-manage-project-supplies.md)

> [!IMPORTANT]
> Если значение в поле **Тип строки** в строке журнала проекта или в строке покупки не указано, то при учете журнала проектов или документа покупки строки планирования проекта не создаются.

## <a name="to-set-up-prices-for-resources-items-and-general-ledger-accounts-for-projects"></a>Настройка цен для ресурсов, товаров и счетов главной книги для проектов

> [!NOTE]
> Во 2-й волне выпуска 2020 года мы выпустили новые процессы для настройки цен и скидок и управления ими. Если вы новый клиент, вы используете новую версию. Если вы уже являетесь клиентом, то используете ли вы новый интерфейс, зависит от того, включил ли ваш администратор обновление функции **Новые возможности ценообразования для продажи** в **Управление функциями**. Для получения дополнительной информации см. [Раннее включение новых функций](/dynamics365/business-central/dev-itpro/administration/feature-management).

Вы можете настроить цены для товаров, ресурсов и счетов главной книги, связанных с проектов. 

#### [Текущая версия](#tab/current-experience)

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Проект**, а затем выберите соответствующую ссылку.  
2. Выберите проект, а затем выберите действие **Ресурс**, **Товар** или **Счет ГК**.
3. На страницах **Цены ресурсов в проекте**, **Цены товаров в проекте** или **Цены по счету ГК проекта** заполните поля необходимым образом.

Когда вы выбираете ресурс, товар или счет главной книги для проекта, [!INCLUDE [prod_short](includes/prod_short.md)] использует информацию в необязательных полях в строках планирования проекта и журналах проектов. В следующей таблице рассматривается, как это происходит.

|Столбец1  |Столбец2  |
|---------|---------|
|**Ресурсы проекта**|Поля **№ задачи проекта**, **Тип работ**, **Код валюты**, **Скидка по строке (%)** и **Коэффициент себестоимости единицы**. Значение в поле **Цена единицы** для ресурса используется в строках планирования проекта и журналах проектов при вводе ресурса или назначении ресурса группе ресурсов. Эта цена переопределяет цены, указанные на странице **Цена ресурса/Цены группы ресурсов**.|
|**Товары проекта**|Поля **№ задачи проекта**, **Код валюты** и **Скидка по строке (%)**. Значение в поле **Цена единицы** для товара будет использоваться в строках планирования проекта и журналах проектов при вводе данного товара. Эта цена переопределяет обычную цену клиента (механизм "наилучшей цены") для товаров. Чтобы использовать обычную цену клиента, не указывайте для проекта цены на товары проекта.|
|**Счета ГК**|Информация в полях **№ задачи проекта**, **Код валюты**, **Скидка по строке (%)**, **Коэффициент себестоимости единицы** и **Себестоимость единицы** используется в строках планирования проекта и журналах проектов при вводе и добавлении в проект этого счета ГК. При выборе счета главной книги в строках планирования проекта и журналах проектов значение в поле **Цена единицы** используется для расходов по проекту в главной книге.|

#### [Новая версия](#tab/new-experience)

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, затем выберите соответствующую ссылку.  
2. Выберите соответствующий проект, затем выберите действие **Прайс-листы продажи**.

## <a name="to-set-up-project-posting-groups"></a>Настройка учетных групп проектов

Один из аспектов планирования проектов заключается в выборе счетов учета, используемых для расчета себестоимости проектов. Для учета проектов необходимо настроить счета учета для каждой учетной группы проектов. Учетная группа представляет связь между проектом и тем, как он должен обрабатываться в главной книге. При создании проекта необходимо указать учетную группу, и по умолчанию каждая задача, создаваемая для проекта, будет связана с этой учетной группой. По мере создания задач вы можете переопределять учетную группу по умолчанию и выбрать более подходящую.  

> [!NOTE]  
> Перед настройкой учетных групп необходимо настроить счета в плане счетов. Дополнительные сведения см. в разделе [Настройка или изменение плана счетов](finance-setup-chart-accounts.md).  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Учетные группы проектов**, а затем выберите соответствующую ссылку.  
2. Выберите действие **Создать**, затем заполните поля, как описано в следующей таблице.  

| Поле счета | Описание | Используется в типе НЗП |
| --- | --- |  --- |
| **Код** |Идентификатор для учетной группы. Можно ввести не более 10 символов, включая пробелы. | |
| **Счет затрат НЗП** |Счет НЗП для расчетной себестоимости незавершенного производства по проекту, который представляет собой балансовый счет капитальных активов. | Примененная себестоимость, признанные затраты|
| **Счет начисленных затрат НЗП** |Счет для метода расчета НЗП "Себестоимость" или "Себестоимость продаж". Этот счет предназначен для обязательств по начисленным расходам в вашем балансе. Когда корректировка НЗП требует увеличения затрат на потребление, учитываемых в отчете о прибылях и убытках, учет производится на этом счете. | Начисленные затраты|
| **Счет примененных затрат по проекту** |Балансирующий счет для счета затрат НЗП, который является корреспондирующим счетом для счета отрицательных расходов. Используется, когда для параметра **Используемый метод учета НЗП** установлено значение *Проект*. | Примененные затраты, признанные затраты|
| **Счет примененных затрат по товару** |То же, что и **Счет примененных затрат по проекту**, однако используется, когда для параметра **Используемый метод учета НЗП** установлено значение *Операция книги проектов*.| |
| **Счет примененных затрат по ресурсу** |То же, что и **Счет примененных затрат по проекту**, однако используется, когда для параметра **Используемый метод учета НЗП** установлено значение *Операция книги проектов*.| |
| **Счет примененных затрат ГК** |То же, что и **Счет примененных затрат по проекту**, однако используется, когда для параметра **Используемый метод учета НЗП** установлено значение *Операция книги проектов*.| |
| **Счет коррекции затрат по проекту** |Балансирующий счет для счета начисленных затрат НЗП, который является счетом затрат. | Начисленные затраты|
| **Счет расходов ГК (бюджет)** |Счет продажи, используемый для расходов главной книги в задачах проекта с данной учетной группой. Если оставить это поле пустым, будет использоваться счет главной книги, введенный в строке планирования проекта. | |
| **Счет начисленных продаж НЗП** |Счет НЗП для рассчитанной стоимости продаж НЗП, который представляет собой счет начисленного дохода для вашего баланса. Когда корректировка НЗП требует увеличения признанного дохода, учет производится на этом счете. | Начисленные продажи, признанные продажи|
| **Счет ГК выставления счетов продажи НЗП** |Счет для значения продаж с накладными НЗП, которые не могут быть распознаны. Это балансовый счет доходов будущих периодов. | Признанные продажи, примененные продажи|
| **Счет примененных продаж по проекту** |Балансирующий счет для счета продаж НЗП с накладными, который является контрсчетом доходов. | Начисленные продажи, признанные продажи|
| **Счет коррекции продаж по проекту** |Балансирующий счет для счета продаж незавершенного производства по проекту, который является счетом доходов. | Начисленные продажи|
| **Счет признанных затрат** |Счет расходов, содержащий признанные затраты для проекта. Обычно это дебетовый счет затрат. | Признанные затраты|
| **Счет признанных продаж** |Счет доходов, содержащий признанный доход для проекта. Обычно это кредитовый счет дохода. | Признанные продажи|
