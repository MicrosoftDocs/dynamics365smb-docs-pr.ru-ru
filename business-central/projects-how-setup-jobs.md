# Настройка цен работ и групп учета работ

Как менеджер проекта, вы можете настроить задания, которые определяют каждый из проектов, которыми вы управляете в [!INCLUDE[prod_short](includes/prod_short.md)]. Используйте страницу **Настройка модуля "Работы"** , чтобы определить, как вы будете использовать функции, связанные с работами.

Для каждой работы необходимо задать различную информацию:

* Цены на товары, используемые для работы
* Ресурсы для работы
* Счета ГК для работы
* Учетные группы для работы (обязательно)

## Настройка общей информации для работ

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Настройка работ**, а затем выберите связанную ссылку.
2. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

> [!NOTE]
> Переключатель **Применить связь использования по умолчанию** на странице **Настройка модуля "Работы"** указывает, связываются ли операции книги работ со строками планирования работ по умолчанию. Включите переключатель, чтобы применить этот параметр ко всем новым работам. Вы можете включить или отключить отслеживание потребления для конкретной работы, включив или отключив переключатель **Применить связь использования** на странице **Карточка работы**.

### Настройка трассировки потребления работы

При выполнении работы может потребоваться узнать, каким образом потребление сопоставляется с планом. Для просмотра потребления можно создать связь между строками планирования работ и фактическим потреблением. Эта связь позволяет отслеживать ваши затраты и иметь представление о том, сколько работы осталось. По умолчанию тип строки планирования работы — **Бюджет**, но использование типа строки **Бюджет и к оплате** оказывает схожий эффект.

После настройки отслеживания потребления путем включения переключателя **Применить связь использования по умолчанию** вы можете просматривать информацию в строке планирования работы. Например, вы можете задать количество ресурса, товара или счета главной книги. Вы также можете задать количество для переноса в журнал работ. Поле **Оставшееся количество** в строке планирования работы показывает, что остается для переноса и учета в журнале работ.

>[!NOTE]
> Если в работе установлен флажок **Применить связь использования**, а поле **Тип строки** в строке журнала работ или строке покупки имеет значение **К оплате**, при учете журнала работ или документа покупки создаются новые строки планирования работ с типом строки **Бюджет и к оплате**.  
>
> Для получения дополнительной информации см. [Запись потребления для работ](projects-how-record-job-usage.md) и [Управление запасами для работы](projects-how-manage-project-supplies.md)

> [!IMPORTANT]
> Если значение в поле **Тип строки** в строке журнала работы или в строке покупки не указано, то при учете журнала работ или документа покупки строки планирования работ не создаются.

## Настройка цен для ресурсов, товаров и счетов Главной книги для заданий

> [!NOTE]
> Во 2-й волне выпуска 2020 года мы выпустили новые процессы для настройки цен и скидок и управления ими. Если вы новый клиент, вы используете новую версию. Если вы уже являетесь клиентом, то используете ли вы новый интерфейс, зависит от того, включил ли ваш администратор обновление функции **Новые возможности ценообразования для продажи** в **Управление функциями**. Для получения дополнительной информации см. [Раннее включение новых функций](/dynamics365/business-central/dev-itpro/administration/feature-management).

Можно настроить цены для товаров, ресурсов и счетов Главной книги, связанных с заданием. 

#### [Текущая версия](#tab/current-experience)

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2. Выберите задание, затем выберите действие **Ресурс**, **Номенклатура** или **Счет ГК**.
3. На страницах **Цены ресурсов работ**, **Цены товаров для работы** или **Цены по счету ГК работы** заполните поля по мере необходимости.

Когда вы выбираете ресурс, товар или счет главной книги для работы, [!INCLUDE [prod_short](includes/prod_short.md)] использует информацию в необязательных полях в строках планирования работ и журналах работ. В следующей таблице рассматривается, как это происходит.

|Столбец1  |Столбец2  |
|---------|---------|
|**Ресурсы работы**|Поля **Номер рабочего задания**, **Тип работ**, **Код валюты**, **Скидка строки (%)** и **Коэффициент себестоимости единицы**. Значение в поле **Цена единицы** для ресурса используется в строках планирования работ и журналах работ при вводе ресурса или назначения ресурса группе ресурсов. Эта цена переопределяет цены, указанные на странице **Цена ресурса/Цены группы ресурсов**.|
|**Товары работы**|Поля **Номер рабочего задания**, **Код валюты** и **Скидка строки (%)**. Значение в поле **Цена единицы** для товара будет использоваться в строках планирования работ и журналах работ при вводе данного товара. Эта цена переопределяет обычную цену клиента (механизм "наилучшей цены") для товаров. Чтобы использовать обычную цену клиента, не указывайте цены на товары работы для работы.|
|**Счета ГК**|Информация в полях **Номер рабочего задания**, **Код валюты**, **Скидка строки (%)**, **Коэффициент себестоимости единицы** и **Себестоимость единицы** используется в строках планирования работ и журналах работ, когда вводится этот счет ГК и добавляется к работе. При выборе счета главной книги в строках планирования работ и журналах работ значение в поле **Цена единицы** используется для расходов по работе в главной книге.|

#### [Новая версия](#tab/new-experience)

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Работы**, а затем выберите связанную ссылку.  
2. Выберите соответствующую работу, затем выберите действие **Прайс-листы продажи**.

## Настройка учетных групп работ

Один аспект заданий планирования заключается выборе учетных счетов для оценки работы. Для учета работ необходимо настроить счета для учета каждой учетной группы работ. Учетная группа представляет связь между заданием и тем, как оно должно быть обработано в Главной книге. При создании работы следует указать учетную группу. По умолчанию каждая задача, создаваемая для работы, будет связана с этой учетной группой. По мере создания задач вы можете переопределять учетную группу по умолчанию и выбрать более подходящую.  

> [!NOTE]  
> Перед настройкой учетных групп необходимо настроить счета в плане счетов. Дополнительные сведения см. в разделе [Настройка или изменение плана счетов](finance-setup-chart-accounts.md).  

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Группы учета работ**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**, затем заполните поля, как описано в следующей таблице.  

| Поле счета | Описанием |
| --- | --- |
| **Код** |Идентификатор для учетной группы. Можно ввести не более 10 символов, включая пробелы. |
| **Счет затрат НЗП** |Счет НЗП для расчетных затрат НЗП работ, который представляет собой балансовый счет капитальных активов. |
| **Счет начисленных затрат НЗП** |Счет для метода расчета НЗП "Себестоимость" или "Себестоимость продаж". Этот счет предназначен для обязательств по начисленным расходам в вашем балансе. Когда корректировка НЗП требует увеличения затрат на потребление, учитываемых в отчете о прибылях и убытках, учет производится на этом счете. |
| **Работы - счет ГК примен. стоим.** |Балансирующий счет для счета затрат НЗП, который является корреспондирующим счетом для счета отрицательных расходов. |
| **Счет примененных затрат по товару** |Балансирующий счет для счета затрат НЗП, который является корреспондирующим счетом для счета отрицательных расходов. |
| **Счет примененных затрат по ресурсу** |Балансирующий счет для счета затрат НЗП, который является корреспондирующим счетом для счета отрицательных расходов. |
| **Счет примененных затрат** |Балансирующий счет для счета затрат НЗП, который является корреспондирующим счетом для счета отрицательных расходов. |
| **Работы - счет ГК коррекции стоим.** |Балансирующий счет для счета начисленных затрат НЗП, который является счетом затрат. |
| **Счет расходов ГК (бюджет)** |Счет продажи, используемый для расходов главной книги в рабочих заданиях в данной учетной группе. Если оставить его пустым, будет использоваться счет ГК, введенный в строку планирования. |
| **Счет начисленных продаж НЗП** |Счет НЗП для рассчитанной стоимости продаж НЗП, который представляет собой счет начисленного дохода для вашего баланса. Когда корректировка НЗП требует увеличения признанного дохода, учет производится на этом счете. |
| **Счет ГК выставления счетов продажи НЗП** |Счет для значения продаж с накладными НЗП, которые не могут быть распознаны. Это балансовый счет доходов будущих периодов. |
| **Работа - счет для примененной себестоимости продаж** |Балансирующий счет для счета продаж НЗП с накладными, который является контрсчетом доходов. |
| **Работа - счет коррекции продаж** |Балансирующий счет для счета продаж работ НЗП, который является счетом доходов. |
| **Счет признанных затрат** |Счет затрат, содержащий признанные затраты для работы. Обычно это дебетовый счет затрат. |
| **Счет признанных продаж** |Счет доходов, содержащий признанный доход для работы. Обычно это кредитовый счет дохода. |
