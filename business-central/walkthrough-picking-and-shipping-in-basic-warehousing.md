# <a name="walkthrough-picking-and-shipping-in-basic-warehouse-configurations"></a>Пошаговое руководство. Подбор и отгрузка в основных складских конфигурациях

В [!INCLUDE[prod_short](includes/prod_short.md)] подберите и отгрузите товары с использованием одного из четырех методов, как описано в следующей таблице.

|Метод|Исходящий процесс|Требуется подбор|Требуется отгрузка|Уровень сложности (подробнее см. в разделе [Обзор Warehouse Management](design-details-warehouse-management.md))|  
|------|----------------|-----|---------|-------------------------------------------------------------------------------------|  
|А|Выполните учет подбора и отгрузки из строки заказа|||Нет отдельной складской операции.|  
|Б|Выполните учет подбора и отгрузки из складского документа подбора|Включено||База: последовательно по каждому заказу.|  
|Т|Выполните учет подбора и отгрузку из документа складской отгрузки||Включено|База: объединенная проводка получения/отправки для нескольких заказов.|  
|Д|Выполните учет подбора из складского документа подбора и учет отгрузки из документа складской отгрузки|Включено|Включено|Дополнительно|  

Подробнее см. в разделе [Исходящий складской поток](design-details-outbound-warehouse-flow.md).

В следующем пошаговом руководстве показан метод B в предыдущей таблице.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве

В основных складских конфигурациях, если настройки склада требуют обработки подбора, но не обработки отгрузки, то для регистрации и учета информации о подборе и отгрузке для исходящих документов-источников следует использовать страницу **Подбор запасов**. Исходящий документ-источник может быть заказом на продажу, возвратом покупки, исходящим заказом перемещения или производственным заказом, для которого необходимы компоненты.  

В этом пошаговом руководстве демонстрируются следующие задачи:  

- Настройка склада "ЮГ" для складских подборов.  
- Создание заказа на продажу для клиента 10000 на 30 ламп Амстердам.  
- Выпуск заказа на продажу для обработки на складе.  
- Создание складского подбора на основании выпущенного документа-источника.  
- Регистрация складского перемещения из склада с одновременным учетом расходной накладной продажи для исходного заказа на продажу.  

## <a name="roles"></a>Роли

В этом пошаговом руководстве демонстрируются задачи, выполняемые исполнителями следующих ролей:  

- Заведующий складом  
- Обработчик заказов  
- Работник склада

## <a name="story"></a>Сюжет

Элен, менеджер склада в CRONUS, настроила "ЮГ" склад для базовой обработки подбора, где работники склада обрабатывают исходящие заказы по отдельности. Светлана, обработчик заказов, создает заказ на продажу 30 единиц товара 1928-S, которые должны быть поставлены клиенту 10000 из склада "ЮГ". Вадим, работник склада, должен убедиться, что отгружаемый товар подготовлен и поставлен клиенту. Вадим управляет всеми включенными задачами на странице **Подбор запасов**, которое автоматически указывает на ячейки, где сохраняется 1928-S.

[!INCLUDE[set_up_location.md](includes/set_up_location.md)]

### <a name="setting-up-the-bin-codes"></a>Настройка кодов ячеек

После того, как вы настроили склад, вы должны добавить две ячейки.

#### <a name="to-setup-the-bin-codes"></a>Настройка кодов ячеек

1. Выберите действие **Ячейки**.
2. Создайте две ячейки с кодами *S-01-0001* и *S-01-0002*.

### <a name="making-yourself-a-warehouse-employee-at-location-south"></a>Назначения себя сотрудником склада "ЮГ"

Чтобы использовать эту функцию, вы должны добавить себя на склад в качестве складского работника. 

#### <a name="to-make-yourself-a-warehouse-employee"></a>Чтобы назначить себя работником склада

  1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать первый раз.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работники склада**, а затем выберите связанную ссылку.  
  2. Выберите поле **ИД пользователя** и укажите свою учетную запись пользователя на странице **Работники склада**.
  3. В поле **Код склада** выберите "ЮГ".  
  4. Выберите поле **По умолчанию**, затем выберите кнопку **Да**.  

### <a name="making-item-1928-s-available"></a>Делаем товар 1928-S доступным

Чтобы сделать товар 1928-S доступным на складе "ЮГ", выполните следующие действия:  

  1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать второй раз.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Журналы товаров**, а затем выберите связанную ссылку.  
  2. Откройте журнал по умолчанию, а затем создайте две строки журнала товаров со следующими сведениями о рабочей дате (23 января).  

        |Тип операции|Код товара|Код Склада|Код ячейки|количество;|  
        |----------------|-----------------|-------------------|--------------|--------------|  
        |Приход|1928-S|ЮГ|С-01-0001|20|  
        |Приход|1928-S|ЮГ|С-01-0002|20|  

        По умолчанию поле **Код ячейки** в строках продаж скрыто, поэтому необходимо включить его отображение. Для этого вам необходимо персонализировать страницу. Дополнительные сведения см. в разделе [Чтобы начать персонализацию страницы с помощью баннера персонализации](ui-personalization-user.md#to-start-personalizing-a-page-through-the-personalizing-banner).

  3. Выберите **Действия**, выберите **Учет** а затем выберите **Учет**.  
  4. Нажмите кнопку **Да**.  

## <a name="creating-the-sales-order"></a>Создание заказа на продажу

Заказы на продажу — это наиболее распространенный тип исходного исходящего документа.  

### <a name="to-create-the-sales-order"></a>Создание заказа на продажу

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать третий раз.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Заказы на продажу**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  
3. Создайте заказ на продажу для клиента 10000 на рабочую дату (23 января) со следующей строкой заказа на продажу.  

    |Товар|Код Склада|количество;|  
    |----|-------------|--------|  
    |1928-S|ЮГ|30|  

     Перейти к уведомлению склада, что заказ на продажу готов для обработки на складе.  

4. Выберите действие **Выпустить**.  

    Вадим переходит к подбору и отгрузке проданных товаров.  

## <a name="picking-and-shipping-items"></a>Товары по подбору и отгрузке

На странице **Подбор запасов** можно управлять всеми исходящими складскими операциями для конкретного документа\-источника, такого как заказ на покупку. [!INCLUDE[tooltip-inline-tip_md](includes/tooltip-inline-tip_md.md)]  

### <a name="to-pick-and-ship-items"></a>Подбор и отгрузка товаров

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать четвертый раз.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Подбор запасов**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  

    Убедитесь, что поле **Нет.** на экспресс-вкладке **Общие** заполнено.
3. Выберите поле **Документ\-источник**, а затем выберите **Заказ на продажу**.  
4. Выберите поле **Номер источника**, выберите строку для продажи клиенту 10000, а затем выберите кнопку **ОК**.  

    В качестве альтернативы выберите действие **Получить источник документа**, а затем выберите заказ на продажу.  
5. Выберите действие **Автозаполнение кол-ва для обработки**.  

    В качестве альтернативы в поле **Кол-во для обработки** введите 10 и 20 соответственно в двух строках складского подбора.  
6. Выберите действие **Учет**, выберите **Отгрузить**, затем выберите кнопку **ОК**.  

    Теперь 30 фонарей Амстердам зарегистрированы как подобранные из ячеек С-01-0001 и С-01-0002, и создана отрицательная операция товарной книги, отражающая учтенную расходную накладную продажи.
