# <a name="walkthrough-planning-supplies-manually"></a>Пошаговое руководство. Планирование поставок вручную

[!INCLUDE[complete_sample_data](includes/complete_sample_data.md)]  

В этом пошаговом руководстве демонстрируется процесс планирования заказов на продажу для выполнения нового требования. Планирование поставок можно инициировать с фиксированным интервалом, например каждое утро или каждый понедельник, или при получении уведомления от отдела продаж либо с производства, в зависимости от типа требования. В данном пошаговом руководстве будет использоваться страница **Планирование заказов** — простой инструмент планирования поставок на основе выполняемого вручную процесса принятия решений вместо автоматического планирования на основе параметров.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве  
 В этом пошаговом руководстве рассматриваются следующие задачи:  

-   планирование заказа на покупку для компонентов производства;  
-   планирование заказа на перемещение для выполнения требования продаж;  
-   планирование производственного заказа для многоуровневого товара.  

## <a name="roles"></a>Роли  
 В этом пошаговом руководстве демонстрируются задачи, выполняемые исполнителями следующих ролей:  

-   Планировщик производства  
-   Агент по закупкам  
-   обработчик заказов на продажу;  

## <a name="prerequisites"></a>Предварительные требования  
 Перед выполнением указаний этого пошагового руководства необходимо установить [!INCLUDE[d365fin](includes/d365fin_md.md)]. В эту базу данных необходимо внести следующие изменения:  

-   удалите все имеющиеся заказы на продажу велосипедов;  
-   создайте один заказ на продажу 10 велосипедов на складе «СИНИЙ»;  
-   удалите все запланированные и утвержденные производственные заказы. Не удаляйте начатые заказы с уже учтенными операциями.  

 Как правило, в этом пошаговом руководстве следует использовать предлагаемые данные, так как они содержат необходимые записи.  

## <a name="story"></a>Сюжет  
 Эдуард, планировщик производства производственной организации, собирается запланировать производственные заказы и заказы на покупку для выполнения нового требования продаж.  

 Поскольку количество уровней спецификации товаров невелико, Эдуард использует страницу **Планирование заказов** для создания заказов на поставку вручную — по очереди для каждого уровня производства.  

 В более сложной производственной среде для планирования поставок используется производственный план на основе параметров товаров, например периода перепланирования, страхового запаса времени и точки повтора заказа, а также пакетные вычисления консолидированного требования по всем уровням производства.  

## <a name="setting-up-the-sample-data"></a>Настройка образца данных  
 В стандартной демонстрационной организации CRONUS в данный момент есть множество незапланированных требований. В процессе выполнения различных задач планирования этого пошагового руководства нужно будет отклониться от реалистичных потоков деловых операций, игнорируя требования с более близкими сроками выполнения и используя вместо них более поздние сроки.  

## <a name="using-the-order-planning-page"></a>Использование страницы «Планирование заказов»  

Страница **Планирование заказов** может быть доступна из нескольких разных мест:  

-   Производство, Планирование  
-   Продажи и маркетинг, Обработка заказов  
-   Покупка, Планирование  
-   Кроме того, вы можете открыть эту страницу для конкретного производственного заказа, выбрав действие **Планирование**.

### <a name="to-use-the-order-planning-page"></a>Процедура использования страницы «Планирование заказов»  

1.  Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Планирование заказов**, затем выберите соответствующую ссылку.  

     После первого открытия страницы **Планирование заказов** должен быть вычислен план для отражения новых требований, появившихся после последнего вычисления.  

2.  Выберите действие **Вычислить план**.  

     Система планирования проанализирует появившееся новое требование, например новые либо измененные заказы на продажу или производственные заказы.  

     Количество, необходимое для каждой строки требования, вычисляется на основе общего наличия. Это вычисление выполняется последовательно для каждого заказа. То есть первым будет вычислен заказ с наиболее ранним сроком выполнения или датой поставки для строки требования. После этого дополнительные строки требования будут вычисляться в том же порядке, вне зависимости от их срока выполнения или даты отгрузки.  

3.  Убедитесь, что страница **Планирование заказов** развернута, а размеры полей столбца изменены таким образом, что в них полностью видны все имена полей по умолчанию.  

     Когда процесс вычисления закончится, на странице отобразятся все невыполненные требования, представленные в виде свернутых строк заголовков заказов, отсортированных по самой ранней дате требования.  

     Обратите внимание, что в CRONUS есть несколько заказов с невыполненными требованиями. Каждая выделенная полужирным шрифтом строка соответствует заказу, заказу на продажу или производственному заказу, содержащему по крайней мере одну строку заказа с недостаточным наличием.  

4.  В поле **Показать требования как** выберите фильтр **Все требования**.  

     С помощью поля **Тип требования** можно выбрать типы заказов, которые должны отображаться.  

     Заказы, у которых нет проблем, наличия не отображаются. Если при вычислении плана заказы отсутствуют, отобразится сообщение, а строки планирования отображаться не будут.  

## <a name="planning-a-purchase-order-to-fulfill-component-demand"></a>Планирование заказа на покупку для выполнения требования компонента  
 При выполнении этой процедуры будет создан заказ на покупку компонентов, необходимых для производства.  

### <a name="to-plan-a-purchase-order-to-fulfill-component-need-in-production"></a>Процедура планирования заказа на покупку для удовлетворения потребности в компоненте на производстве  

1.  Разверните первую строку (выберите знак "+").  
2.  Выберите первую строку требования с товаром **LSU-15**, затем выберите действие **Показать документ**.  
3.  Закройте открытый производственный заказ, чтобы вернуться на страницу **Планирование заказов**.  
4.  В поле **Метод пополнения** выберите **Покупка**.  

     По умолчанию используется значение из карточки товара или карточки единицы хранения, однако его можно изменить на одно из следующих значений:  

    -   **Покупка** — для создания заказа на покупку.  
    -   **Перемещение** — для создания заказа на перемещение.  
    -   **Произв. заказ** — для создания производственного заказа.  

5.  В поле **Поставка от** выберите один из следующих параметров в соответствии с выбранной системой пополнения.  

    -   **Поставщик** – для покупки  
    -   **Склад** — для перемещения  

     Если поле не заполнено, при попытке создания заказов на поставку отобразится сообщение об ошибке.  

    > [!NOTE]  
    >  Если для компонентов в карточках товаров задан номер поставщика по умолчанию, в строках будут указаны предварительно заданные значения.  

6.  Выберите поле **Поставка от**.  
7.  На странице **Каталог поставщиков товара** выберите действие **Создать**, затем выберите поставщика **30000**.  
8.  Нажмите кнопку **ОК**, чтобы вернуться на страницу **Планирование заказов**.  
9. Скопируйте номер поставщика **30000** в другие строки компонентов громкоговорителя для этого производственного заказа.  

     Теперь можно приступать к созданию заказа на покупку.  

10. Выберите действие **Создать заказы**. Откроется страница **Создание заказов на поставку**.  
11. В поле **Заказы на изготовление для** на экспресс-вкладке **Планирование заказов** выберите параметр **Активный заказ**.  
12. На экспресс-вкладке **Параметры** в поле **Создать заказ на покупку** выберите параметр **Создать заказы на покупку**.  
13. Нажмите кнопку **ОК**, чтобы создать заказы на покупку для всех компонентов в заказе.  

     Теперь заказы на покупку созданы, сохранены и размещены в конце списка заказов на покупку.  

## <a name="planning-a-transfer-order-to-fulfill-sales-demand"></a>Планирование заказа на перемещение для выполнения требования продаж  
 В рамках этой процедуры будет спланировано требование, в основе которого лежит заказ на продажу. Строки требования соответствуют строкам продажи, а не строкам компонента, как в случае требования производства.  

### <a name="to-plan-a-transfer-order-to-fulfill-sales-demand"></a>Процедура планирования заказа на перемещение для выполнения требования продаж  

1.  Переместите курсор в строку планирования для заказа **2008**.  
2.  Разверните строку и переместите указатель на строку требования.  

     Заказ на продажу **2008** оформлен на десять громкоговорителей типа **LS-120**, заказанных страховой компанией РОСНО.  

     Отобразится зависящий от товара метод пополнения и поставщик по умолчанию.  

    > [!NOTE]  
    >  В нижней части страницы отображаются информационные поля. В поле **Ближайшая доступная дата** указано, что необходимые десять штук будут доступны по входящему заказу на поставку через девять дней после текущего срока выполнения. Если это слишком поздно для клиента, в поле **Доступно для перемещения** указано тринадцать штук товара, находящихся на другом складе. Этот запас необходимо будет учесть в плане.  

3.  Выберите поле **Доступно для перемещения**, чтобы открыть страницу **Получение альтернативной поставки**.  
4.  Нажмите кнопку **ОК**, чтобы забронировать десять доступных товаров.  

    > [!NOTE]  
    >  В строке требования предлагаемая покупка заменена перемещением со склада "ЗЕЛЕНЫЙ". Функция **Заказы на изготовление** позволяет создать заказ на перемещения со склада "Зеленый" на требуемый склад. Принцип действия поля **Наличие заменителей** аналогичен.  

5.  Выберите действие **Создать заказы**. Откроется страница **Создание заказов на поставку**.  
6.  В поле **Заказы на изготовление для** на экспресс-вкладке **Планирование заказов** выберите параметр **Активный заказ**.  
7.  На экспресс-вкладке **Параметры** в поле **Создать заказ на перемещение** выберите параметр **Создать заказы на перемещение**.  
8.  Нажмите кнопку **ОК**, чтобы создать заказ на перемещение для поставки по заказу на продажу.  

     Теперь заказ на перемещение создан, сохранен и размещен в конце списка открытых заказов на перемещение.  

## <a name="planning-a-multilevel-production-order-to-fulfill-sales-demand"></a>Планирование многоуровневого производственного заказа для выполнения требования продажи  
 В рамках этой процедуры будет спланировано выполнение требования продажи для производимого товара с несколькими уровнями производства, для каждого из которых создается зависимое требование производства.  

### <a name="to-plan-multilevel-production-to-fulfill-sales-demand"></a>Процедура планирования многоуровневого производства для выполнения требования продаж  

1.  Выберите строку планирования со спросом по продажам для заказа **1001**, который был создан ранее в качестве предварительно требуемых данных.  

     Это требование является строкой продажи, однако для товара задан метод пополнения **Произв. заказ**. Перейдите к добавлению дополнительного колокольчика в компонент, необходимый для каждого велосипеда.  

2.  Выберите действие **Компоненты**, чтобы открыть страницу **Компоненты производственного плана**.  
3.  В строке с товаром "Колокольчик" измените заданное в поле **Кол-во в** значение **1** на **2**.  
4.  На странице **Планирование заказов** ознакомьтесь с альтернативами планирования. В данном случае альтернативные способы поставки отсутствуют — перемещение, замена или более поздняя доставка невозможны. Необходимо создать предлагаемый заказ на поставку: производственный заказ.  
5.  Выберите действие **Создать заказы**, чтобы создать производственный заказ.  

     На странице **Планирование заказов** обратите внимание на то, что строка планирования для заказа на продажу **1001** больше не существует, а первоначальное требование продажи выполнено.  

6.  Закройте страницу **Планирование заказов**.  

     Теперь при желании можно остаться в этом представлении и выполнить все задачи планирования. Но вместо этого необходимо выполнить роль планировщика производства, перейдя к созданному производственному заказу и открыв страницу **Планирование заказов**.  

 В роли планировщика производства вы теперь должны спланировать определенный производственный заказ.  

### <a name="to-plan-a-specific-production-order"></a>Процедура планирования производственного заказа  

1.  Откройте производственный заказ **101001** (для десяти велосипедов), который был только что создан с помощью функции **Заказы на изготовление**.  
2.  Откройте страницу **Компоненты производственного заказа**, чтобы убедиться, что дополнительный колокольчик отражен в производственном заказе.  
3.  Выберите действие **Планирование**.  

     Страница **Планирование заказов** открывается в таком виде, где всегда выполняется фильтрация по определенному производственному спросу. Требование продаж не отображается. Для отображения дополнительных требований необходимо вычислить план.  

4.  Выберите действие **Вычислить план**.  

     Обратите внимание, что четыре новых производственных заказа отображаются в виде незапланированного требования производства, производного от заказа **101001**. Новые строки соответствуют новому требованию производства из сборочных узлов, которые необходимо создать для производства заказа.  

5.  Выберите действие **Раскрыть все**, чтобы просмотреть все производственные требования для производственных заказов.  

     Чтобы указать дополнительные сведения о строках потребностей, в представление может потребоваться добавить поля **Необходимое количество** и **Доступное требуемое количество**.  

     Теперь необходимо поставить по десять штук каждого компонента.  

     Обратите внимание, что четыре строки спроса содержат метод пополнения "Произв. заказ". Эти четыре сборочных узла соответствуют второму уровню производства велосипеда.  

     Настройки пополнения по умолчанию уже указаны, поэтому можно приступать к оформлению заказов.  

6.  Выберите действие **Создать заказы**.  

     Перед выбором кнопки **ОК** обратите внимание на текст на экспресс-вкладке **Планирование заказов**. Этот текст важен, поскольку в структуре производства велосипеда есть несколько производимых компонентов, сборочных узлов, которые могут требоваться при создании этого производственного заказа.  

7.  На странице **Создание заказов на поставку** в поле **Заказы на изготовление для** выберите **Все строки**, а затем нажмите кнопку **ОК**, чтобы создать производственные заказы для второго уровня производства заказа.  

     Обратите внимание, что требование производства верхнего уровня для производственного заказа 101001 больше не существует. Это означает, что первоначальное требование производства для сборочных узлов уже запланировано.  

     На странице **Планирование заказов** план вычисляется повторно для планирования структуры велосипеда.  

8.  Выберите действие **Вычислить план**, чтобы повторно вычислить план, как указано в тексте встроенной справки.  

     Две новые строки соответствуют дополнительному требованию производства из сборочных узлов, запланированных на предыдущих шагах. Предлагается сделать два производственных заказа для поставки барабанов (один для 10 втулок переднего колеса и один для 10 втулок заднего колеса).  

9. Выберите действие **Раскрыть все**, чтобы просмотреть все требования для двух производственных заказов.  

     Предлагаемый план поставки указывает на то, что для компонентов в общей сложности будет создано четыре заказа на покупку. Решено сделать предлагаемые заказы.  

10. Выберите действие **Создать заказы**.  
11. В поле **Заказы на изготовление для** выберите параметр **Все строки**, а затем нажмите кнопку **ОК**. Проверьте, есть ли дополнительные требования для производства родительского товара, велосипеда, который продается по заказу на продажу 1001.  
12. Выберите действие **Вычислить план**.  

     В сообщении указывается, что теперь все необходимые товары поставлены. Проверьте запланированные производственные заказы, которые были созданы.  

13. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Утвержд. произв. заказы**, затем выберите соответствующую ссылку.  

     На странице **Утвержд. произв. заказы** просмотрите, как запланировано время начала и завершения отдельных заказов в соответствии со структурой продукта. Сначала производятся компоненты самого нижнего уровня. Поэтому необходимо запланировать многоуровневые заказы так, как описано в этом рабочем процессе планирования.
