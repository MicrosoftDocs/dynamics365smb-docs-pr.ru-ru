# Отслеживание хода выполнения проекта

С помощью функции «Незавершенное производство» (НЗП) можно оценивать финансовую стоимость выполняющихся проектов в главной книге.

По мере работы над проектом потребляются материалы и ресурсы, а также возникают расходы, которые должны учитываться в проекте. В большинстве случаев расходы учитываются в проекте до выставления счетов. Но если учитывать только расходы, финансовый отчет будет неточным. Для отслеживания фактической стоимости проекта необходимо рассчитывать НЗП и учитывать его в главной книге. Дополнительные сведения см. в разделе [Общее представление о методах учета НЗП](projects-understanding-wip.md).

НЗП можно рассчитать на основе следующих параметров:

* Себестоимость
* Стоимость продаж
* Признаваемые затраты
* Процент завершения
* Выполненный контракт

## Создание метода оценки НЗП для проекта

Можно создать метод НЗП для проекта, соответствующий потребностям вашей организации, и задать его в качестве метода по умолчанию.  

> [!NOTE]
> После использования нового метода для создания операций НЗП нельзя изменить или удалить этот метод.  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **методы оценки НЗП проекта**, а затем выберите соответствующую ссылку.  
2. Выберите действие **Создать**, затем введите в поля требуемые данные. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]  
3. Закройте страницу.   
4. Чтобы сделать этот новый метод по умолчанию, выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **настройка проектов**, а затем выберите соответствующую ссылку.  
5. В поле **Метод НЗП по умолчанию** выберите метод из списка.

## Определение метода оценки НЗП для проекта

При создании нового проекта необходимо указать, какой метод оценки НЗП будет применяться для этого проекта. В некоторых случаях метод оценки НЗП проекта, который вы используете, уже установлен по умолчанию.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, а затем выберите соответствующую ссылку.
2. Выберите действие **Создать**. Подробнее см. в статье [Создание проектов](projects-how-create-jobs.md).  
3. На странице **Карточка проекта** в поле **Метод НЗП** выберите метод оценки НЗП из списка. Если метод по умолчанию определен, вы можете выбрать другой вариант, если требуется.  

### Определение метода оценки НЗП для задачи проекта

Вы можете определить метод оценки НЗП для задачи проекта, исключить некоторые задачи проекта из расчета НЗП или сгруппировать задачи, чтобы они рассчитывались вместе. 

Если вы хотите рассчитывать НЗП для каждой задачи проекта отдельно, учет НЗП предоставляет определенные измерения для конкретных задач.

Параметр **Итоги по НЗП** определяет задачи проекта, которые должны группироваться при расчете НЗП и признанных доходов и затрат. В любой группе задач должна быть одна задача, удовлетворяющая двум условиям:
<!--But doesn't the parenthetical below contradict this -* if there is no total, the application sets the total for you, meaning the condition does not HAVE to be satisfied, right? Or am I missing something?-->

* Параметру **Итоги по НЗП** должно быть задано значение *Всего*. (В случае отсутствия задач проекта со значением параметра **Итоги по НЗП**, равным *Всего*, значение *Всего* автоматически устанавливается в последней строке задачи проекта при первом расчете НЗП.)

* Содержит **№ задачи проекта**, который является последним в группе или диапазоне задач проекта.

В следующей таблице описаны три параметра:

| Поле | Описание |
|--|--|
| **\<blank\>** | Оставьте это поле пустым, если задача проекта входит в группу задач. |
| **Итог** | Определяет диапазон или группу задач, которые будут включены в расчет НЗП и признания. В группе любая задача проекта, у которой для параметра **Тип задачи проекта** задано значение **Учет**, будет включена в итоги по НЗП, если только для параметра **Итоги по НЗП** этой задачи не задано значение **Исключено**. |
| **Исключено** | Применяется только к задаче, у которой для параметра **Тип задачи проекта** задано значение **Учет**. В этом случае задача не включается в расчет НЗП и признания. |

В следующем примере задачи проекта разделены на две группы НЗП. Этот пример демонстрирует, как работает поле **Итоги по НЗП**.

|№ задачи проекта|Описание|Тип задачи проекта|Поле **Итоги по НЗП**|  
|------------------|----------------------|----------------------|----------------------|  
|1000|Подготовка|Сумма (от)|\<blank\>|
|1010|.    Уборка|Учет|**Исключено**|
|1099|Итого подготовка|Сумма (до)|\<blank\>|
|1100|Ковровое покрытие|Сумма (от)|\<blank\>|
|1110|.    Проклейка пола|Учет|**Исключено**|
|1120|.    Укладка ковра|Учет|\<blank\>|
|1199|Итого ковровое покрытие|Сумма (до)|\<blank\>|
|1200|Готово|Сумма (от)|\<blank\>|
|1210|.    Чистка ковра пылесосом|Учет|\<blank\>|
|1299|Итого отделка|Сумма (до)|**Итого**|
|1300|Исправление ошибок|Сумма (от)|\<blank\>|
|1310|.    Исправление ошибок|Учет|\<blank\>|
|1399|Итого исправление ошибок|Сумма (до)|**Итого**|

Обратите внимание:

* С *1000* по *1299*: НЗП для этой группы задач проекта вычисляется отдельно. Обратите внимание, что две из задач — 1010 и 1110 — исключаются из расчета НЗП, поскольку их тип задачи проекта — **Учет**.

* С *1300* по *1399*: НЗП для этой группы задач проекта вычисляется отдельно.

## Рассчитать НЗП

Вы можете определить сумму НЗП, которую необходимо учесть в счетах балансового отчета по завершении периода. Для этого используется пакетное задание **Расчет НЗП по проектам**.  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **расчет НЗП по проектам**, а затем выберите соответствующую ссылку.  
2. Выберите действие **Рассчитать НЗП**.
3. На странице **Расчет НЗП по проектам** заполните требуемые поля.
4. Нажмите кнопку **ОК**.  

> [!NOTE]  
> Пакетное задание только вычисляет НЗП, но не учитывает НЗП в главной книге. Для учета после вычисления НЗП следует выполнить пакетное задание **Учет НЗП в ГК**. Подробнее см. в следующей процедуре.

## Учет НЗП

После вычисления НЗП можно выполнить учет НЗП в счетах балансового отчета в конце периода. Для этого используется пакетное задание **Учет НЗП по проектам в ГК**.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Учет НЗП по проектам в ГК**, а затем выберите соответствующую ссылку.  
2. На странице **Учет НЗП по проектам в ГК** заполните требуемые поля.  
3. Нажмите кнопку **ОК**.

## Расчет и учет операций завершения проекта

После завершения всех мероприятий по проекту, включая учет потребления и выставление счетов, необходимо изменить статус проекта на **Выполнено**. Затем необходимо обратить все НЗП, которые были учтены в главной книге.

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Проекты**, а затем выберите соответствующую ссылку.  
2. Выберите открытый проект, а затем выберите действие **Изменить**.
3. В поле **Статус** выберите **Завершено**.
4. Следуйте инструкциям, чтобы рассчитать и учесть НЗП, или выполните шаги 5 и 6, чтобы сделать это вручную.  
5. Выберите действие **Рассчитать НЗП**.
6. На странице **Расчет НЗП по проектам** заполните требуемые поля.  

     У операций НЗП по проекту, созданных пакетным заданием, появится флажок в поле **Проект выполнен**, указывающий, что они являются операциями завершения.  
7. Выберите действие **Учет НЗП в ГК**.
8. На странице **Учет НЗП по проектам в ГК** заполните требуемые поля.  

     У операций НЗП по проекту в главной книге, созданных пакетным заданием, появится флажок в поле **Проект выполнен**, указывающий, что они являются операциями завершения.

## Просмотр операций книги проектов

Все связанные с проектом операции регистрируются в регистрах проектов и нумеруются последовательно, начиная с 1. В регистре проектов содержится перечень всех учтенных операций по проектам.    

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Регистры проектов**, а затем выберите соответствующую ссылку.
2. Выберите соответствующий регистр, затем выберите действие **Книга проектов**.

На странице **Операции книги проектов** можно просмотреть операции, связанные с любым проектом.
