# <a name="monitor-project-progress-and-performance"></a>Отслеживание хода выполнения проекта

С помощью функции «Незавершенное производство» (НЗП) можно оценивать финансовую стоимость выполняющихся проектов в главной книге.

По мере работы над проектом потребляются материалы и ресурсы, а также возникают расходы, которые должны учитываться в проекте. В большинстве случаев расходы учитываются в проекте до выставления счетов. Но если вы учитываете только расходы, ваш финансовый отчет будет неточным. Для отслеживания фактической стоимости проекта необходимо рассчитывать НЗП и учитывать его в главной книге. Следующие методы WIP доступны «из коробки»:

* Себестоимость
* Стоимость продаж
* Признаваемые затраты
* Процент завершения
* Выполненный контракт

Вы также можете создать метод WIP, отвечающий потребностям вашей организации, и установить его в качестве метода по умолчанию.  

Дополнительные сведения см. в разделе [Общее представление о методах учета НЗП](projects-understanding-wip.md).

Если вы хотите просмотреть результат, используя другой метод, измените метод и рассчитайте WIP еще раз. Количество расчетов незавершенного производства не ограничено. Значение не публикуется автоматически в главная книга. После того, как вы рассчитаете незавершенную работу с использованием предпочитаемого вами метода, вы можете разместить публикацию в главная книга.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") , введите **Методы незавершенного проекта**, затем выберите соответствующую ссылку.  
2. Выберите действие **Создать**, затем введите в поля требуемые данные. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]  
3. Закройте страницу.   
4. Чтобы сделать этот новый метод по умолчанию, выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Настройка проектов**, затем выберите соответствующую ссылку.  
5. В поле **Метод НЗП по умолчанию** выберите метод из списка.

## <a name="define-a-wip-method-for-a-project"></a>Определение метода оценки НЗП для проекта

При создании нового проекта необходимо указать, какой метод оценки НЗП будет применяться для этого проекта. В некоторых случаях метод оценки НЗП проекта, который вы используете, уже установлен по умолчанию.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, а затем выберите соответствующую ссылку.
2. Выберите действие **Создать**. Подробнее см. в статье [Создание проектов](projects-how-create-jobs.md).  
3. На странице **Проект карточка** в поле **Метод WIP** на быстрой вкладке **Размещение**  выберите метод WIP из списка. Если определен метод по умолчанию, при необходимости можно выбрать другой вариант.  

### <a name="define-a-wip-method-for-a-project-task"></a>Определение метода оценки НЗП для задачи проекта

Вы можете определить метод WIP для задачи проекта, исключить задачи проекта из расчета WIP или сгруппировать задачи для совместного расчета.

Если вы хотите рассчитывать НЗП для каждой задачи проекта отдельно, учет НЗП предоставляет определенные измерения для конкретных задач.

Параметр **Итоги по НЗП** определяет задачи проекта, которые должны группироваться при расчете НЗП и признанных доходов и затрат. В любой группе задач должна быть одна задача, удовлетворяющая двум условиям:
<!--But doesn't the parenthetical below contradict this -* if there is no total, the application sets the total for you, meaning the condition does not HAVE to be satisfied, right? Or am I missing something?-->

* **WIP-Total** установлен на **Total**. Если нет задач проекта с **WIP-Total**, установленным на Total, Total автоматически устанавливается в последней строке задач проекта при первом расчете WIP.
* Номер в поле **Номер задачи проекта** является последним в группе или диапазоне задач проекта.

В следующей таблице описаны варианты:

| Поле | Описанием |
|--|--|
| **\<blank\>** | Оставьте это поле пустым, если задача проекта входит в группу задач. |
| **Итог** | Определяет диапазон или группу задач, которые будут включены в расчет НЗП и признания. В группе любая задача проекта, у которой для параметра **Тип задачи проекта** задано значение **Учет**, будет включена в итоги по НЗП, если только для параметра **Итоги по НЗП** этой задачи не задано значение **Исключено**. |
| **Исключено** | Применяется только к задаче, у которой для параметра **Тип задачи проекта** задано значение **Учет**. В этом случае задача не включается в расчет НЗП и признания. |

В следующем примере задачи проекта разделены на две группы НЗП. Этот пример демонстрирует, как работает поле **Итоги по НЗП**.

|№ задачи проекта|Описание|Тип задачи проекта|Поле **Итоги по НЗП**|  
|------------------|----------------------|----------------------|----------------------|  
|1000|Подготовка|Сумма (от)|\<blank\>|
|1010|.    Уборка|Учет|**Исключено**|
|1099|Итого подготовка|Сумма (до)|\<blank\>|
|1100|Ковровое покрытие|Сумма (от)|\<blank\>|
|1110|.    Проклейка пола|Учет|**Исключено**|
|1120|.    Укладка ковра|Учет|\<blank\>|
|1199|Итого ковровое покрытие|Сумма (до)|\<blank\>|
|1200|Готово|Сумма (от)|\<blank\>|
|1210|.    Чистка ковра пылесосом|Учет|\<blank\>|
|1299|Итого отделка|Сумма (до)|**Итого**|
|1300|Исправление ошибок|Сумма (от)|\<blank\>|
|1310|.    Исправление ошибок|Учет|\<blank\>|
|1399|Итого исправление ошибок|Сумма (до)|**Итог**|

Вы замечаете:

* Для *1000* по *1299* незавершенное производство рассчитывается отдельно для этой группы задач проекта. Однако следует отметить, что две задачи, 1010 и 1110, исключены из расчета незавершенной работы, поскольку тип их проектной задачи —  **Публикация**.
* Для *1300* по *1399* незавершенное производство рассчитывается отдельно для этой группы задач проекта.

## <a name="calculate-wip"></a>Рассчитать НЗП

Вы можете определить сумму незавершенного производства для размещения на балансовых счетах для отчетного периода с помощью пакетного задания **Расчет незавершенного производства** .  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), , введите **Расчет проекта WIP**, затем выберите соответствующую ссылку.  
2. Выберите действие **Рассчитать НЗП**.
3. На странице **Расчет НЗП по проектам** заполните требуемые поля.
4. Нажмите кнопку **ОК**.  

> [!NOTE]  
> Пакетное задание только вычисляет незавершенную работу, но не публикует ее в главная книга. Чтобы провести незавершенное производство, запустите пакетное задание **Провести незавершенное производство в главную книгу**  после того, как рассчитаете незавершенное производство. Подробнее см. в следующей процедуре.

### <a name="review-warnings"></a>Предупреждения о просмотре

Если в результате расчета WIP выводится сообщение *WIP был рассчитан с предупреждениями*, возможно, вам следует просмотреть предупреждения.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Панель НЗП по проектам**, а затем выберите соответствующую ссылку.  
2. Выберите проект, для которого вы хотите просмотреть предупреждения. Переключатель  **Предупреждения WIP** включен для проектов, имеющих предупреждения WIP.
3. Выберите действие **Показать предупреждение** .

### <a name="delete-wip-entries"></a>Удалить записи WIP

Если вы захотите попробовать другие методы WIP, вы можете получить ошибку *Невозможно изменить задачу проекта, так как у проекта есть связанные записи WIP проекта* . Чтобы иметь возможность проверить метод WIP, удалите существующие записи WIP.

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Панель НЗП по проектам**, а затем выберите соответствующую ссылку.  
2. Выберите проект, для которого вы хотите удалить записи WIP.
3. Выберите действие **Удалить записи WIP** .

## <a name="post-wip"></a>Учет НЗП

При расчете незавершенного производства его можно отразить на балансовых счетах для отчета на конец отчетного периода. Используйте пакетное задание **Проводка незавершенного производства в главную книгу** .

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") , введите **Проект: провести НЗП в ГК**, а затем выберите соответствующую ссылку.  
2. На странице **Учет НЗП по проектам в ГК** заполните требуемые поля.  
3. Нажмите кнопку **ОК**.

## <a name="calculate-and-post-project-completion-entries"></a>Расчет и учет операций завершения проекта

После завершения всех действий по проекту, включая публикацию данных об использовании и выставление счетов, необходимо обновить статус проекта на  **Завершено**. Затем вам необходимо отменить все незавершенные работы, опубликованные в главная книга.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, а затем выберите соответствующую ссылку.  
2. Выберите открытый проект, а затем выберите действие **Изменить**.
3. На вкладке **Размещение**  в поле **Статус**  выберите **Завершено**.
4. Следуйте инструкциям, чтобы рассчитать и учесть НЗП, или выполните шаги 5 и 6, чтобы сделать это вручную.  
5. Выберите действие **Рассчитать НЗП**.
6. На странице **Расчет НЗП по проектам** заполните требуемые поля.  

     Записи WIP проекта, созданные при запуске пакетного задания, имеют флажок **Проект завершен**, указывающий, что это записи о завершении.  
7. Выберите действие **Учет НЗП в ГК**.
8. На странице **Учет НЗП по проектам в ГК** заполните требуемые поля.  

     Записи проекта WIP главная книга, созданные при запуске пакетного проекта, имеют флажок **Проект завершен**, указывающий на то, что это записи о завершении.

## <a name="view-project-ledger-entries"></a>Просмотр операций книги проектов

Все связанные с проектом операции регистрируются в регистрах проектов и нумеруются последовательно, начиная с 1. В регистре проектов содержится перечень всех учтенных операций по проектам.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Регистры проектов**, а затем выберите соответствующую ссылку.
2. Выберите соответствующий регистр, затем выберите действие **Книга проектов**.

На странице **Записи в журнале проектов**  вы можете просмотреть записи, связанные с любым проектом.
