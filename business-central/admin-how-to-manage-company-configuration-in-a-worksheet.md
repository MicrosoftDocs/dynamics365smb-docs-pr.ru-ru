# <a name="manage-company-configuration-in-a-worksheet"></a>Управление конфигурацией организации в журнале
Журнал конфигураций является центральным местом, где можно планировать, выполнять трассировку и настройку. Можно создать журналы для каждой организации, с которой вы работаете, или создать стандартный журнал конфигураций, который будет использоваться для определения множества идентичных организаций.  

Первым этапом подготовки пакета конфигурации является выбор организации, которая уже настроена и адаптирована для большинства потребностей. Эта организация служит основой для выполнения конфигурации новых организаций. В журнале укажите таблицы, которые должны контролироваться и управляться конфигурацией. Так как большинство таблиц в [!INCLUDE[d365fin](includes/d365fin_md.md)] имеет связи и зависимости в других таблицах, необходимо также включить эти связанные таблицы. Вместе эти таблицы будут выступать в качестве структуры, вокруг которой будет построена новая организация. Последующие шаги обеспечивают работу с пакетом и позволяют развернуть конфигурацию.  

Чтобы упростить отслеживание и анализ проделанной работы, используйте информационную панель **Таблица пакета конфиг.** для просмотра информации о записях. Используйте информационную панель **Связанная таблица конфигурации** для отслеживания связей таблиц.  

В следующих процедурах показано, как добавлять и настраивать сведения таблицы для конфигурации.  

## <a name="to-open-the-configuration-worksheet"></a>Открытие журнала конфигураций  
1.  В [!INCLUDE[d365fin](includes/d365fin_md.md)] откройте организацию, которая является базисом конфигурации, а затем откройте начальную страницу ее ролевого центра "Внедряющий мастер служб RapidStart Services".  
2.  Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Журнал конфигураций**, затем выберите соответствующую ссылку.  

## <a name="to-add-a-table-to-the-worksheet"></a>Добавление таблицы в журнал  
1.  На странице **Журнал конфигураций** выберите действие **Изменить список**.  
2.  На первой строке в поле **Тип строки** выберите **Таблица**.  
4.  В поле **Код таблицы** выберите таблицу, куда нужно добавить конфигурацию.  
5.  В поле **Код страницы** введите код страницы, которая связанна с таблицей. Для стандартных таблиц это значение заполняется автоматически. В случае пользовательских таблиц необходимо указать код.
6.  В поле **Ссылка** введите URL-адрес страницы документации, например в Справке, которая содержит информацию о передовой рекомендации или инструкции по настройке таблицы.  
7.  Для добавления связанных таблиц выберите действие **Получить связанные таблицы**.  

    > [!NOTE]  
    > Связанные таблицы не будут добавлены с помощью действия **Получить связанные таблицы**, если выполняется какое-либо из следующих условий:
    > - Эта связь является условной.  
    > Пример. При получении связанных таблиц для таблицы **Клиент** таблица **Склад** не добавляется, поскольку она связана с таблицей **Клиент** лишь условно, то есть тогда, когда заполнено поле **Код склада** в таблице **Клиент**.  
    > - Связанная таблица фильтруется.  
    > Пример. В поле в связанной таблице содержится предложение WHERE. Причина этого в том, что соответствующая информация о связях хранится в системной таблице **Поле**, которая не полностью доступна для применения.  
    > Необходимо вручную добавить эти типы таблиц, выполнив шаг 4 этой процедуры.  

8.  Для изменения итогового списка таблиц выберите таблицу, которую необходимо удалить, затем выберите действие **Удалить**.  
9. Повторите шаги для каждой таблицы, которую необходимо добавить в конфигурацию.  
10. Чтобы удалить повторяющиеся сведения таблицы, которые могут возникнуть в результате использования действия **Получить связанные таблицы**, выберите действие **Удалить повторяющиеся строки**. При этом будут удалены повторяющиеся таблицы, которые имеют один и тот же код пакета.  

## <a name="to-add-multiple-tables-to-the-configuration-worksheet"></a>Добавление нескольких таблиц в журнал конфигураций  
1. Выберите действие **Получить таблицы**. Откроется страница пакетного задания **Получить таблицы конфигураций**.  
2. На экспресс-вкладке **Параметры** укажите типы таблиц, которые требуется добавить в конфигурацию, как описано в следующей таблице.

    |Параметр|Описание|  
    |----------------------------------|---------------------------------------|  
    |**Включить только с данными**|Установите флажок для включения только тех таблиц, которые содержат данные. Например, может понадобиться включить таблицу, в которой уже указаны типичные условия платежа, которые поддерживает решение.|  
    |**Включить связанные таблицы**|Установите флажок для включения всех связанных таблицы. Чтобы добавить подмножество связанных таблиц, см. шаг 3 в рамках этой процедуры.|  
    |**Включить таблицы измерений**|Установите флажок для включения всех таблиц измерений.|  
    |**Включить только лицензированные таблицы**|Установите флажок для включения только таблиц, у которых лицензия, по которой создается лист, позволяет получить доступ.|

3. На экспресс-вкладке **Объект** установите соответствующие фильтры для определения типов таблиц, которые необходимо включить или исключить.  
4. Нажмите кнопку **ОК**. Таблицы [!INCLUDE[d365fin](includes/d365fin_md.md)] добавлены в журнал. Каждая операция в списке имеет строку типа **Таблица**.  
5. Чтобы удалить повторяющиеся сведения таблицы, которые могут возникнуть в результате использования действия **Получить таблицы**, выберите действие **Удалить повторяющиеся строки**. При этом будут удалены повторяющиеся таблицы, которые имеют один и тот же код пакета.  
6. Можно добавить таблицы к журналу, относящиеся к выбранной таблице. Просмотрите информацию на информационной панели **Связанные таблицы**, чтобы определить, есть ли отсутствующие таблицы. Чтобы добавить связанные таблицы для определенной таблицы, выберите таблицу из списка, затем выберите действие **Получить связанные таблицы**.  

    > [!NOTE]  
    > Связанные таблицы не будут добавлены с помощью действия **Получить связанные таблицы**, если выполняется какое-либо из следующих условий:
    > - Эта связь является условной.  
    > Пример. При получении связанных таблиц для таблицы **Клиент** таблица **Склад** не добавляется, поскольку она связана с таблицей **Клиент** лишь условно, то есть тогда, когда заполнено поле **Код склада** в таблице **Клиент**.  
    > - Связанная таблица фильтруется.  
    > Пример. В поле в связанной таблице содержится предложение WHERE. Основанием для этого является то, что используемая информация об отношениях хранится в виртуальной таблице **Поле** и недоступна на страницах, таких как таблица конфигурации, в целях обеспечения производительности.  
    > Необходимо добавить связанные таблицы с такими сложными отношениями вручную, следуя шагу 4 в разделе [Добавление таблицы в журнал](admin-how-to-manage-company-configuration-in-a-worksheet.md#to-add-a-table-to-the-worksheet).

7. Для удаления таблиц из итогового списка таблиц выберите таблицу, которую требуется удалить, затем выберите действие **Удалить**.  

Воспользуйтесь следующей процедурой, чтобы определить какие поля таблиц необходимо включить. После создания этой спецификации можно экспортировать таблицу в Excel и использовать структуру таблицы в качестве шаблона для сбора данных клиента. Дополнительные сведения см. в разделе [Подготовка к миграции данных клиента](admin-use-templates-to-prepare-customer-data-for-migration.md).  

## <a name="to-specify-a-set-of-fields-and-records-for-a-configuration-table"></a>Определение набора полей и записей для таблицы конфигурации  
1. Выберите таблицу в списке таблиц конфигурации, затем выберите действие **Изменить список**.  
2. Выберите таблицу, для которой необходимо задать данные о полях, затем выберите действие **Поля**.  
3. Чтобы выбрать только поля, которые требуется включить, выберите действие **Очистить включенное**. Чтобы добавить все поля, выберите действие **Установить включенное**.  
4. Чтобы указать, что данные поля не следует проверять, снимите флажок **Проверить поле** для поля.  
5. Нажмите кнопку **ОК**.  
6. Для фильтрации определенного набора записей для включения в журнал конфигурации, выберите действие **Фильтры**, затем укажите соответствующие значения фильтров.

Можно создать зоны функций и группы таблиц в журнале для размещения, чтобы объединить сходные функции. Например, при настройке схемы счетов для своей конфигурации, можно создать группу таблиц учета. Как правило, зоны применяются для группирования набора таблиц, соответствующих функциональной области. Каждая область может содержать группы. Группа может использоваться для упорядочения таблиц, объединение которых имеет смысл.  

Следующая процедура описывает, как добавлять обозначения области и группы после создания исходного списка таблиц. После добавления этих категорий можно продолжить добавление и изменение списка таблиц.  

## <a name="to-categorize-and-group-functionality-in-the-worksheet"></a>Категоризация и группировка функций в журнале  
1. В начале области вставьте новую строку в журнал.  
2. В поле **Тип строки** выберите **Область**. В поле **Имя** введите имя области.  
3. В начале группирования таблиц вставьте новую строку в журнал.  
4. В поле **Тип строки** выберите **Группа**. В поле **Имя** введите имя области. Название группы выделяется автоматически.  
5. Чтобы переместить таблицы в нужную категорию, выберите таблицу для перемещения, затем выберите действие **Сдвинуть вверх** или **Сдвинуть вниз**. Кроме того, можно удалить строку журнала и вставить таблицу еще раз в необходимом месте.  

Некоторые таблицы [!INCLUDE[d365fin](includes/d365fin_md.md)] являются стандартными, и данные в них едва ли меняются от реализации к реализации. Соответственно, чтобы помочь клиенту сфокусироваться, можно удалить эти таблицы из журнала после включения их в пакет конфигурации. После добавления таблицы остаются частью пакета конфигурации.  

## <a name="to-remove-a-standard-table-in-the-worksheet"></a>Удаление стандартной таблицы в журнале  
После добавления всех необходимых таблиц в пакет конфигурации укажите, какие таблицы не будут требовать внимания клиента.  
1.  Выберите таблицы, а затем удалите их путем выбора действия **Удалить**.  

    > [!NOTE]  
    >  Таблицы остаются в пакете, даже если они удалены из журнала.  

## <a name="to-review-and-customize-existing-database-data"></a>Просмотр или настройка существующих данных базы данных
По мере создания пакета конфигурации для решения можно просматривать и настраивать доступные данные в базе данных в соответствии с потребностями клиента. Таблица базы данных должна иметь соответствующую страницу.  

## <a name="to-customize-data-in-the-database"></a>Настройка данных в базе данных  

1.  На странице **Журнал конфигураций** определите таблицы, данные которых необходимо просмотреть или изменить.  

    > [!NOTE]  
    >  Убедитесь, что каждая таблица имеет код страницы, присвоенный ей. Для стандартных таблиц [!INCLUDE[d365fin](includes/d365fin_md.md)] это значение заполняется автоматически. В случае пользовательских таблиц вам следует указать код.  

2.  Выберите действие **Данные базы данных**.  

     Откроется страница [!INCLUDE[d365fin](includes/d365fin_md.md)] для этой страницы.  

3.  Просмотрите имеющуюся информацию. Измените его по мере необходимости путем удаления ненужных записей или добавления новых.
