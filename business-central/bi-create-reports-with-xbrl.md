# <a name="create-reports-with-xbrl" />Создать отчеты с XBRL

> [!NOTE]
> Мы находимся в процессе удаления функций отчетности XBRL из [!INCLUDE[prod_short](includes/prod_short.md)]. Подробнее см. в статье [Изменения в выпуске 2022, волна 1](/dynamics365/business-central/dev-itpro/upgrade/deprecated-features-w1).

XBRL расшифровывается как e**X**tensible **B**usiness **R**eporting **L**anguage — расширяемый язык бизнес-отчетности. Это основанный на XML язык для разметки финансовых данных, помогающий эффективно и точно их обрабатывать, а также совместно их использовать. Инициатива XBRL позволяет множеству компаний, занимающихся разработкой программного обеспечения для планирования ресурсов предприятия (ERP), и международным бухгалтерским организациям создавать глобальную финансовую отчетность. Цель инициативы — создать стандарт для унификации учета финансовой информации для банков, инвесторов и государственных организаций. Такая отчетность может включать:  

* финансовые отчеты;  
* финансовую информацию;  
* нефинансовую информацию;  
* обязательную отчетность, такую как годовые и квартальные финансовые отчеты.  

> [!NOTE]
> Вы можете импортировать схемы, связанные с главной книгой, и создавать документы XBRL, сопоставляя данные главной книги (ГК) из плана счетов с элементами в таксономиях, которые были разработаны для финансовых отчетов, таких как балансы, отчеты о прибылях и убытках и т. д.
>
> Возможности XBRL в Business Central поддерживают таксономии для спецификации 2.1. Однако таксономии могут содержать неподдерживаемые элементы, такие как базы ссылок формул или iXBRL (встроенный XBRL), или иметь другие структурные отличия. Рекомендуем проверить функциональность XBRL, прежде чем использовать ее для отчетов.
>
> Для полной поддержки таксономий могут потребоваться сторонние теги XBRL и инструменты. У организации XBRL International есть список инструментов и услуг; в зависимости от требований к отчетности XBRL для данной таксономии вы можете изучить эти ресурсы. Подробнее см. на страницах [Getting Started for Business](https://go.microsoft.com/fwlink/?linkid=2153466) и [Tools and Services](https://go.microsoft.com/fwlink/?linkid=2153356).

## <a name="extensible-business-reporting-language" />eXtensible Business Reporting Language

Ведением таксономий XBRL занимается www.xbrl.org. Загрузить таксономии и прочитать более подробную информацию можно на сайте XBRL.  

Допустим, кому-то нужна финансовая информация от вас. Этот кто-то предоставляет вам таксономию (документ XML), содержащую одну или более схем, каждая из которых содержит одну или более строк, которые необходимо заполнить. Строки соответствуют отдельным финансовым позициям, которые требуются получателю информации. Вы импортируете эту таксономию, а затем заполняете схему (схемы), указывая, какой счет или счета соответствуют каждой строке, а также что нужно рассчитать, например оборот или сальдо на дату. В некоторых случаях можно указать постоянную величину, например количество сотрудников. Теперь можно отправить заполненный документ (документ XML) запрашивающей его стороне. Идея состоит в том, что такие действия могут повторяться, поэтому, при повторяющихся запросах, если в таксономии не было сделано никаких изменений, можно просто экспортировать новый заполненный документ для нового временного интервала.

## <a name="xbrl-comprises-the-following-components" />XBRL состоит из следующих компонентов

**Спецификация** XBRL объясняет, что такое XBRL, и как строить документы и таксономии XBRL. Спецификация XBRL содержит описание языка XBRL с технической точки зрения и рассчитана на технических специалистов.  

**Схемы** XBRL это низкоуровневые компоненты XBRL. Схема — это физический файл XSD (также называемый определением схемы XML), который выражает, как должны строиться документы и таксономии.

**Базы ссылок** XBRL — это физические файлы XML, содержащие информацию об элементах, определенных в схеме XBRL, например о метках для одного или нескольких языков, об их взаимосвязи, правилах суммирования элементов и т. д.  

**Таксономия** XBRL — это “словарь”, созданный группой людей, удовлетворяющий спецификации XBRL и предназначенный для обмена бизнес-информацией.  

XBRL **документ** это бизнес-отчет, например финансовая отчетность, подготовленная в соответствии со спецификацией XBRL. Значения величин в документе объясняются в таксономии. Вообще, документ — это нечто бесполезное, если вам неизвестна таксономия, для которой он подготовлен.  

## <a name="layered-taxonomies" />Послойные таксономии

Таксономия может состоять из базовой таксономии, например GAAP США (общепринятые принципы бухгалтерского учета США) или МСБУ (международные стандарты бухгалтерского учета), а в дополнение к ней иметь одно или несколько расширений. Чтобы отразить такую структуру, таксономия ссылается на одну или несколько схем, каждая из которых представляет собой отдельную таксономию. Когда в базу данных загружаются дополнительные таксономии, новые элементы просто добавляются в конец существующих элементов.  

## <a name="linkbases" />Базы ссылок

В XBRL спецификации 2 таксономия описывается в нескольких файлах XML. Первичный файл XML — это собственно файл схемы таксономии (файл .xsd), содержащий неупорядоченный набор элементов или фактов, которые должны быть включены в отчет. В дополнение к этому обычно имеются файлы баз ссылок (.xml). Файлы баз ссылок содержат данные, которые дополняют саму таксономию (файл .xsd). Существует шесть типов файлов баз ссылок, из которых четыре имеют отношение к [!INCLUDE[prod_short](includes/prod_short.md)]. Вот они:

* База ссылок меток: данная база ссылок содержит метки или названия элементов. Файл может содержать метки на разных языках, которые идентифицируются при помощи свойства XML, которое называется “Язык”. Идентификатор языка в XML обычно содержит двухбуквенную аббревиатуру, и, хотя довольно просто догадаться, что означает эта аббревиатура, никакой ее связи с кодом языка в Microsoft Windows или с кодами языков в демонстрационных данных, не существует. Поэтому, когда вы просматриваете языки для конкретной таксономии, вы увидите все метки для первого элемента таксономии, т. е. увидите пример для каждого языка. К таксономии может быть подключено несколько баз ссылок для меток, по одной базе для каждого языка.  
* База ссылок представления: эта база ссылок содержит информацию о структуре элементов или, точнее, о том, как создатель таксономии предлагает приложению представлять таксономию вам. База ссылок содержит серию ссылок, каждая из которых связывает по два элемента — родительский и дочерний. Если применить все эти ссылки, элементы выстроятся в иерархическом порядке. База ссылок представления отвечает исключительно за представление элементов для вашего просмотра.
* База ссылок расчетов: эта база ссылок содержит информацию о том, как выполняется свертка элементов. Структура данной базы аналогична структуре базы ссылок представления, за исключением того, что каждая ссылка или так называемая «дуга» имеет такое свойство, как вес. Вес может быть равен либо 1, либо –1, показывая, должен ли элемент прибавляться к своему родительскому элементу или вычитаться из него. Помните, что свертки не обязательно соответствуют визуальному представлению.  
* Справочная база ссылок: это база ссылок в виде файла xml, которая содержит дополнительную информацию о данных, требуемых создателем таксономии.

## <a name="set-up-xbrl-lines" />Настройка строк XBRL

После импорта или обновления таксономии строки схем должны быть заполнены всей информацией, необходимой для удовлетворения требований к финансовой отчетности. Эта информация включает в себя базовую информацию об организации, сами бухгалтерские отчеты, примечания к бухгалтерским отчетам, дополнительные таблицы и т. д.  

Строки XBRL настраиваются путем задания соответствия между данным главной книги и данными таксономии.  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Таксономии XBRL**, а затем выберите соответстввующую ссылку.  
2. На странице **Таксономии XBRL** выберите таксономию из списка.  
3. Выберите действие **Строки**.  
4. Выберите строку и заполните поля.  
5. Для получения дополнительной информации о том, что необходимо заполнять, выберите действие **Информация**.  
6. Для настройки соответствия между счетами ГК в плане счетов и строками XBRL выберите действие **Строки соответствия ГК**.  
7. Для добавления комментариев к финансовым отчетам выберите действие **Заметки**.  

   > [!TIP]
   > Чтобы исключить строки из экспортируемых данных, выберите **НЕПРИМЕНИМО** в качестве типа источника.

   > [!NOTE]  
   > Вы можете экспортировать только те данные, которые соответствуют выбору в поле **Тип источника**. Сюда входят описания и примечания.  

   > [!NOTE]  
   > Таксономии могут содержать элементы, которые [!INCLUDE[prod_short](includes/prod_short.md)] не поддерживает. Если элемент не поддерживается, в поле **Тип источника** отображается значение **Неприменимо**, а в поле **Описание** отображается сообщение об ошибке, например **Непредвиденный тип: "конкретный тип не распознан"**. Если необходимо экспортировать элемент, выберите соответствующий тип источника. Обычно это константа или описание. Это позволит вам вводить и экспортировать данные, однако такие элементы могут иметь правила проверки, которые нельзя проверить перед экспортом.

## <a name="import-an-xbrl-taxonomy" />Импорт таксономии XBRL

Первым шагом при работе с функциональностью XBRL должен быть импорт таксономии в базу данных вашей организации. Таксономия состоит из одной или нескольких схем и некоторых баз ссылок. После завершения импорта схем и баз ссылок, а также после применения баз ссылок к схемам можно настроить строки и поставить в соответствие счетам Главной книги из плана счетов соответствующие строки таксономии.  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Таксономии XBRL**, а затем выберите соответстввующую ссылку.  
2. На странице **Таксономии XBRL** создайте новую строку и введите имя и описание таксономии.  
3. Выберите действие **Схемы** и затем вставьте описание схемы.  
4. Чтобы импортировать схему, на странице **Схемы XBRL** выберите действие **Импорт**, а затем выполните одно из следующих действий для загрузки файла:

   [!INCLUDE[file-upload](includes/file-upload.md)]
5. Чтобы импортировать базу ссылок, на странице **Схемы XBRL** выберите действие **Базы ссылок**, а затем выполните одно из следующих действий для загрузки файла:

   [!INCLUDE[file-upload](includes/file-upload.md)] 
6. Теперь можно применить базу ссылок к схеме. Повторите процедуру, пока все базы ссылок не будут импортированы.  
7. Выберите действие **Применить к таксономии**, чтобы применить базу ссылок к схеме.  

> [!IMPORTANT]  
> Вместо применения баз ссылок по отдельности сразу после импорта можно дождаться, когда будут импортированы все базы ссылок, а затем применить их одновременно. Для этого нажмите кнопку **НЕТ** после отображения запроса на применение вновь импортированной базы ссылок к схеме. Затем выберите строки с базами ссылок, которые необходимо применить.  

## <a name="update-an-xbrl-taxonomy" />Обновление таксономии XBRL

Когда производится изменение таксономии, необходимо обновить текущую таксономию. Причиной для обновления может быть изменение схемы, базы ссылок либо добавление новой базы ссылок. После обновления таксономии следует лишь настроить соответствие для изменившихся или новых строк.  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Таксономии XBRL**, а затем выберите соответстввующую ссылку.  
2. На странице **Таксономии XBRL** выберите действие **Схемы**.  
3. Чтобы обновить схему, выберите схему, которую необходимо обновить, затем выберите действие **Импорт**.  
4. Чтобы обновить или добавить новую базу ссылок, выберите действие **Базы ссылок**.  
5. Выберите соответствующую базу ссылок или нажмите сочетание клавиш <kbd>Ctrl</kbd>+<kbd>N</kbd> для вставки новой строки, выберите тип базы ссылок, а затем вставьте описание.  
6. Для импорта базы ссылок выберите действие **Импорт**.  
7. Нажмите кнопку **Да**, чтобы применить базу ссылок к схеме.
