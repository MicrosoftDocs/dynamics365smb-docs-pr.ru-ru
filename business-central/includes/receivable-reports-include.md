В следующей таблице описаны некоторые ключевые отчеты по дебиторской задолженности.

| Отчет | Описанием | Идентификатор |
|--|--|--|
| [Дебиторская задолженность по срокам давности](https://businesscentral.dynamics.com?report=120) | Показывает непогашенную сумму с клиентами с разбивкой по временным интервалам для времени просрочки. В отчете также отображается часть сальдо клиента, срок оплаты которой не наступил, и может отображаться с подробными сведениями о каждом клиенте или без них. Этот отчет является основным отчетом для выверки книги клиента с главной книгой. При условии, что у вас не разрешен прямой учет на счета, используемые в счете дебиторской задолженности учетных групп клиентов, этот отчет представляет собой перечень сумм, учтенных в ГК. | 120 |
| [Выписка по клиенту](https://businesscentral.dynamics.com?report=1316) | Формирует отчет по клиенту за указанный промежуток времени. Вы можете отправить этот отчет клиентам, чтобы дать им представление о подлежащих оплате суммах, а также в качестве напоминания о просрочке платежей, если она имеет место. Вы можете выбрать отображение просроченных сумм в отдельном разделе. Вы можете включить разделение задолженности по времени в отчет **Дебиторская задолженность по срокам давности**. В качестве диапазона сроков оплаты обычно устанавливается *30Д*. 30Д означает 30-дневные интервалы, например просрочку платежа на 30, 60, 90 и 90 дней. Интервалы начинаются с конечной даты или *1М+ТМ*. 1M+ТМ — это текущий месяц в отдельном интервале, а затем месячные интервалы за предыдущие месяцы. **Примечание**. В списке клиентов этот отчет также имеет действие **Запланированные выписки**. Это действие не обеспечивает фильтрацию по выбранному клиенту. Это тот же отчет, но он используется, когда вы хотите отправить выписку всем или нескольким клиентам. Вы можете отфильтровать отчет, чтобы в нем отображались только все еще открытые операции по клиенту. Чтобы применить фильтр при настройке отчета, в поле **Стиль выписки** выберите **Открытые позиции**. | 1316 |
| [Клиент - баланс на дату](https://businesscentral.dynamics.com?report=121) | Показывает открытые записи в книге клиентов до даты окончания. В этом отчете содержится аналогичное содержимое, что и в отчете по клиенту, но без указания того, просрочена ли операция. **Примечание**. К подробным операциям книги клиентов применяется фильтр по дате. У вас могут быть платежи после конечной даты, однако примененные к счетам в пределах диапазона дат. Эти счета присутствуют в отчете, поскольку на дату окончания они не были закрыты. | 121 | 
| [Клиент - пробный баланс](https://businesscentral.dynamics.com?report=129) | Показывает обороты по клиентам за период, указанный в фильтре дат. Также отображается изменение с начала года за финансовый год, соответствующий выбранному периоду. Данные в отчете сгруппированы по учетным группам клиентов и дают иное представление о книге клиентов, чем в отчете **Дебиторская задолженность по срокам давности**. **Примечание**. Если вы не настроили отчетные периоды, [!INCLUDE [prod_short](prod_short.md)] не будет знать, какой финансовый год использовать. В отчете будут показаны данные либо с начала года за последний определенный финансовый год, либо просто будет выбран период. Этот период может быть или не быть с начала года.| 129 |
| [Клиент - подробный пробный баланс](https://businesscentral.dynamics.com?report=104) | Показывает все операции книги клиентов в пределах указанного фильтра даты. Этот отчет часто используется для проверки того, что все операции для конкретного клиента учтены, или для других внутренних проверок в книгах клиентов. | 104 |
| [Клиент - денежные поступления](https://businesscentral.dynamics.com?report=211) | Создает платежную квитанцию для каждой операции в книге клиентов типа **Оплата**. Если платеж был применен к счетам, будут указаны эти счета; в противном случае сумма платежа будет просто указана как непримененная. Вы можете отправить этот отчет клиентам, которым нужна документация, подтверждающая получение оплаты.| 211 |
| [Выверка счетов клиентов и поставщиков](https://businesscentral.dynamics.com?report=33) | Отображает операции ГК, полученные в результате разноски проводок клиента и поставщика, разделенных по счету ГК и группам разноски. Этот отчет используется для выверки сальдо в книгах клиентов и поставщиков с сальдо в главной книге. | 33 |
| [Клиент - сводка задолженности с распределением по срокам (упрощ.)](https://businesscentral.dynamics.com?report=109)| Это устаревшая версия отчета о дебиторской задолженности с распределением по срокам. Мы рекомендуем вам вместо этого использовать отчет **Дебиторская задолженность по срокам давности**. | 109 |
| [Продажи - статистика](https://businesscentral.dynamics.com?report=112) | [!INCLUDE [reports-sales-statistics](reports-sales-statistics.md)]<br>Этот отчет также можно использовать для работы с дебиторской задолженностью, так как проще быстро найти разнесенные платежи, скидки и продажи для данного клиента.| 112 |
| [Список клиентов](https://businesscentral.dynamics.com?report=101) | Содержит различную основную информацию по клиентам. Например, учетная группа клиента, скидочная группа, информация о процент-нотах и платежах и т. д. Этот отчет можно использовать, например, для ведения информации в таблице «Клиент».| 101 |
