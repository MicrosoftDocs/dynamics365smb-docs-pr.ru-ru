# <a name="understanding-wip-methods-in-project-management"></a>Понимание методов WIP в управлении проектами

По мере выполнения проекта материалы, ресурсы и т. д. потребляются и подлежат учету в проекте. Незавершенное производство (НЗП) — это функция, позволяющая в процессе выполнения проектов оценивать их финансовую стоимость в главной книге. Во многих случаях расходы на проект учитываются до выставления счетов по проекту. Если отражаются только расходы, ваш финансовый отчет является неточным.

Для трассировки стоимости в Главной книге, можно рассчитать НЗП и учесть стоимость в Главной книге. Подробнее см. в статье [Отслеживание хода выполнения проекта](projects-how-monitor-progress-performance.md).

 [!INCLUDE[prod_short](includes/prod_short.md)] Из коробки поддерживает следующие методы расчета и учета стоимости незавершенного производства.

| Метод НЗП | Описанием | Признанные затраты | Признанные продажи |
| --- | ------- |--- | --- |
| Себестоимость |Признает стоимость при выставлении счета клиенту. Признает продажи на основе выставленных счетов-фактур. |Себестоимость|Контракт (цена по счету)|
| Себестоимость продаж |Признает стоимость при выставлении счета клиенту. Признает продажи на основе выставленных счетов-фактур.|Себестоимость продаж|Контракт (цена по выставленным счетам)|
| Стоимость продаж |Признает расходы по мере их поступления в отчетность. Признает продажи пропорционально заявленным затратам.|Потребление (итоговая себестоимость)|Стоимость продаж|
| Процент завершения |Признает расходы по мере их поступления в отчетность. Признает продажи пропорционально заявленным затратам.|Потребление (итоговая себестоимость)|Процент завершения|
| Выполненный контракт |Никакие продажи или затраты не учитываются при расчете незавершенного производства. Завершенный контракт не признает доходы и расходы до тех пор, пока проект не будет завершен. Например, этот вариант полезен, когда существует высокая неопределенность в отношении оценок затрат и доходов по проекту.|По завершении|По завершении|

Точные формулы и транзакции главная книга определяются выбором в полях [**Признанная стоимость**](#recognized-cost) и [**Признанные продажи**](#recognized-sales) .

## <a name="create-a-project-wip-method"></a>Создание метода оценки НЗП для проекта

Можно создать метод НЗП для проекта, соответствующий потребностям вашей организации, и задать его в качестве метода по умолчанию.  

> [!NOTE]
> После использования метода для создания записей WIP вы не сможете изменить или удалить этот метод.  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") , введите **Методы незавершенного проекта**, а затем выберите соответствующую ссылку.  
2. Выберите действие **Новый**, выберите соответствующие значения для полей **Признанные затраты** и **Признанные продажи**, а затем заполните другие поля по мере необходимости. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]  
3. Закрытие страницы.
4. Чтобы сделать этот метод методом по умолчанию, выберите ![лампочку, которая открывает функцию Tell Me.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **настройка проектов**, а затем выберите соответствующую ссылку.  
5. В поле **Метод НЗП по умолчанию** выберите метод из списка.
