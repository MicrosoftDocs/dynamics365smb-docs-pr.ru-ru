# <a name="create-users-according-to-licenses"></a>Создание пользователей в соответствии с лицензиями
Ниже описано, как вы, как администратор, создаете пользователей и определяете, кто может войти в [!INCLUDE[d365fin](includes/d365fin_md.md)], и какие основные права имеют разные типы пользователей в соответствии с лицензиями.

Когда пользователи создаются в [!INCLUDE[d365fin](includes/d365fin_md.md)], вы можете приступить к назначению определенных разрешений пользователям с помощью наборов разрешений и организации пользователей в группы пользователей для удобного управления разрешениями. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).  

> [!NOTE]
> Процесс управления пользователями и лицензиями зависит от того, развернуто ли ваше решение в сети или локально. Например, в онлайн-развертываниях вы можете только отключить и включить пользователя после добавления в [!INCLUDE[d365fin](includes/d365fin_md.md)]. В локальных развертываниях вы можете создавать, редактировать и удалять пользователей.  

## <a name="managing-users-and-licenses-in-online-deployments"></a>Управление пользователями и лицензиями в онлайн-развертываниях
В [!INCLUDE[d365fin](includes/d365fin_md.md)] Online количество пользователей определяется подпиской и добавляется к вашему арендатору в Центре партнеров Майкрософт, обычно вашим партнером Microsoft. Для получения дополнительной информации см. разделы [Добавление нового клиента](https://docs.microsoft.com/partner-center/add-a-new-customer) и [Создание, приостановка или отмена подписок клиентов](https://docs.microsoft.com/partner-center/create-a-new-subscription) в справке Центра партнеров Майкрософт.

Чтобы определить, кто может войти в [!INCLUDE[d365fin](includes/d365fin_md.md)], лицензии на продукт должны быть назначены пользователям в соответствии с ролями, которые они будут выполнять в [!INCLUDE[d365fin](includes/d365fin_md.md)]. Это можно сделать следующими способами:
- Администратор Office 365 вашей компании может сделать это в [Центре администрирования Microsoft 365](https://admin.microsoft.com). Для получения дополнительной информации см. раздел [Добавление пользователей по отдельности или массово Office 365](https://aka.ms/CreateOffice365Users).  
- Партнер Майкрософт может назначать лицензии в Центре администрирования Microsoft 365 или в Центре партнеров Майкрософт. Для получения дополнительной информации см. раздел [Задачи управления пользователями для организаций клиентов](https://docs.microsoft.com/partner-center/assign-licenses-to-users) в справке Центра партнеров Майкрософт.

Для получения дополнительной информации см. раздел [Администрирование Business Central Online](/dynamics365/business-central/dev-itpro/administration/tenant-administration) в справке разработчика и ИТ-специалиста.

После создания пользователей с лицензией [!INCLUDE[d365fin](includes/d365fin_md.md)] в Office 365 их можно импортировать на страницу **Пользователи** в [!INCLUDE[d365fin](includes/d365fin_md.md)] с помощью действия **Получить пользователей из Office 365**.

### <a name="to-add-a-user-in-business-central"></a>Добавление пользователя в Business Central
Чтобы добавить пользователей из центра администрирования Microsoft 365 в [!INCLUDE[d365fin](includes/d365fin_md.md)] Online вы используете специальную функцию импорта.  
1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
2. Выберите действие **Получить пользователей из Office 365**.

Любой новый пользователь, который был создан для вашей подписки Office 365, будет добавлен на странице **Пользователи**. Пользователям назначаются наборы разрешений в соответствии с лицензией, назначенной пользователю в Office 365. Затем можно приступить к назначению более детальных разрешений пользователям и организации их по группам пользователей для простого управления разрешениями. Дополнительные сведения см. в разделе [Назначение наборов разрешений пользователям](ui-define-granular-permissions.md#to-assign-permission-sets-to-users).

### <a name="to-remove-a-users-access-to-the-system"></a>Закрытие доступа пользователя к системе
В сетевых развертываниях вы может закрыть пользователю доступ к системе, изменив значение поля **Статус** на **Отключено**. Все ссылки на пользователя будут сохранены, однако пользователь больше не сможет войти в систему, и активные сеансы для пользователя будут прекращены.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
2. Откройте страницу **Карточка пользователя** для соответствующего пользователя, затем в поле **Состояние** выберите **Отключено**.
3. Чтобы снова предоставить пользователю доступ, измените значение поля **Статус** на **Включено**.

В дополнение к отключению пользователя, вы можете отменить лицензию у пользователя в центре администрирования Office 365. Затем пользователь не может войти в систему. Для получения дополнительной информации см. раздел [Отмена назначении лицензий пользователям](https://docs.microsoft.com/office365/admin/manage/remove-licenses-from-users),

### <a name="to-change-the-assigned-license-for-a-user"></a>Изменение назначенной лицензии для пользователя
Иногда вам может понадобиться изменить лицензию, назначенную пользователю. Например, если вы решили использовать модуль управления сервисом и, следовательно, необходимо обновить все лицензии Essential до Premium. Или если ответственность пользователя изменилась, и вам необходимо заменить лицензию Team Member на Essential.

1. Изменение лицензии в центре администрирования Office 365. Для получения дополнительной информации см. раздел [Добавление пользователей по отдельности или массово Office 365](https://aka.ms/CreateOffice365Users).
2. Войдите в [!INCLUDE[d365fin](includes/d365fin_md.md)] в качестве администратора.
3. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
4. На странице **Пользователи** выберите действие **Обновить все группы пользователей**.
Пользователи будут перемещены в соответствующую группу пользователей, и наборы разрешений будут обновлены. Дополнительные сведения см. в разделе [Управление разрешениями с помощью групп пользователей](ui-define-granular-permissions.md#to-manage-permissions-through-user-groups).

> [!NOTE]
> Всем обычным пользователям в решении должна быть назначена одинаковая лицензия, Essential или Premium.
> Сведения о лицензировании см. в разделе [Руководство по лицензированию Microsoft Dynamics 365 Business Central](https://aka.ms/BusinessCentralLicensing).

## <a name="managing-users-and-licenses-in-online-deployments"></a>Управление пользователями и лицензиями в онлайн-развертываниях
Для локальных развертываний количество лицензированных пользователей указывается в файле лицензии (.flf). Когда администратор или партнер Майкрософт отправляет файл лицензии, администратор может указать, какие пользователи могут войти в [!INCLUDE[d365fin](includes/d365fin_md.md)].

Для локальных развертываний администратор создает, редактирует и удаляет пользователей непосредственно со страницы **Пользователи**.

### <a name="to-edit-or-delete-a-user-on-premises"></a>Редактирование или удаление пользователя в локальном развертывании
1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
2. Выберите пользователя, которого требуется отредактировать, а затем выберите действие **Изменить**.
3. На странице **Карточка пользователя** измените информацию требуемым образом.    
4. Чтобы удалить пользователя, выберите пользователя, которого требуется удалить, а затем выберите действие **Удалить**.

> [!NOTE]
> Для локальных развертываний [!INCLUDE[d365fin](includes/d365fin_md.md)] администратор может выбрать между различными механизмами авторизации учетной информации для пользователей. Затем при создании пользователя необходимо предоставить различную информацию в зависимости от используемого типа учетных данных в конкретном экземпляре [!INCLUDE[server](includes/server.md)].<br /><br />
> Дополнительные сведения, см. в разделе [Проверка подлинности и типы учетных сведений](/dynamics365/business-central/dev-itpro/administration/users-credential-types) в разделе администрирование для разработчиков и содержимом ITPro для [!INCLUDE[d365fin](includes/d365fin_md.md)].
