# <a name="create-users-according-to-licenses"></a>Создание пользователей в соответствии с лицензиями

В этой статье описано, как администраторы создают пользователей и определяют, кто может войти в [!INCLUDE[prod_short](includes/prod_short.md)], и какие разрешения предоставляются разным типам пользователей в соответствии с лицензиями.

Когда вы создаете пользователей в [!INCLUDE[prod_short](includes/prod_short.md)], вы можете назначать им определенные разрешения с помощью наборов разрешений и организовывать пользователей в группы пользователей. Группы пользователей упрощают управление разрешениями для нескольких пользователей одновременно. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).  

Для получения дополнительных сведений о различных типах лицензий и о том, как лицензирование работает в [!INCLUDE[prod_short](includes/prod_short.md)], [загрузите Руководстве по лицензированию Dynamics 365](https://go.microsoft.com/fwlink/?LinkId=866544).

> [!NOTE]
> Процесс управления пользователями и лицензиями зависит от того, развернуто ли [!INCLUDE[prod_short](includes/prod_short.md)] в сети или локально. Для [!INCLUDE [prod_short](includes/prod_short.md)] Online необходимо добавить пользователей из Microsoft 365. В локальных развертываниях вы можете создавать, редактировать и удалять пользователей напрямую.  

## <a name="managing-users-and-licenses-in-online-deployments"></a>Управление пользователями и лицензиями в онлайн-развертываниях

В онлайн-версии [!INCLUDE[prod_short](includes/prod_short.md)] количество пользователей определяется подпиской и добавляется к вашему арендатору в Центре партнеров Майкрософт, обычно вашим партнером Microsoft. Для получения дополнительной информации см. разделы [Добавление нового клиента](/partner-center/add-a-new-customer) и [Создание, приостановка или отмена подписок клиентов](/partner-center/create-a-new-subscription) в справке Центра партнеров Майкрософт.

Чтобы определить, кто может войти в [!INCLUDE[prod_short](includes/prod_short.md)], вы должны назначить лицензии на продукт пользователям в соответствии с ролями, которые они будут выполнять в [!INCLUDE[prod_short](includes/prod_short.md)]. Это можно сделать следующими способами:

- Администратор Microsoft 365 вашей компании может сделать это в [центре администрирования Microsoft 365](https://admin.microsoft.com). Для получения дополнительной информации см. раздел [Добавление пользователей по отдельности или массово Microsoft 365](/microsoft-365/admin/add-users/add-users).  
- Партнер Майкрософт может назначать лицензии в центре администрирования Microsoft 365 или в центре партнеров Майкрософт. Для получения дополнительной информации см. раздел [Задачи управления пользователями для организаций клиентов](/partner-center/assign-licenses-to-users) в справке Центра партнеров Майкрософт.

Для получения дополнительной информации см. раздел [Администрирование Business Central Online](/dynamics365/business-central/dev-itpro/administration/tenant-administration) в справке администратора.

### <a name="to-add-users-or-update-user-information-and-license-assignments-in-business-central"></a><a name="adduser"></a>Добавление пользователей или обновление данных пользователей и назначений лицензий в Business Central
После добавления пользователей или изменения информации о пользователях в центре администрирования Microsoft 365 вы можете быстро импортировать информацию о пользователях в [!INCLUDE[prod_short](includes/prod_short.md)]. Это включает в себя назначение лицензий. 

1. Войдите в [!INCLUDE[prod_short](includes/prod_short.md)], используя учетную запись администратора.
2. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Пользователи**, а затем выберите связанную ссылку.  
3. Выберите **Обновить пользователей из Microsoft 365**.

Если вы добавляете новых пользователей, следующим шагом будет назначение групп и разрешений пользователей. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md). Если вы обновляете информацию о пользователях, и обновление включает изменение лицензии, пользователи будут назначены в соответствующую группу пользователей, и их наборы разрешений будут обновлены. Дополнительные сведения см. в разделе [Управление разрешениями с помощью групп пользователей](ui-define-granular-permissions.md).  

> [!NOTE]
> Всем пользователям должна быть назначена одинаковая лицензия, Essential или Premium. Подробнее см. Руководство по лицензированию Microsoft Dynamics 365 Business Central. Руководство доступно для скачивания на веб-сайте [Business Central](https://dynamics.microsoft.com/business-central/overview/).

Для получения дополнительной информации о синхронизации пользовательской информации с Microsoft 365 см. раздел [Синхронизация с Microsoft 365](#m365).

> [!NOTE]
> Если вы используете внешнего бухгалтера для управления вашим учетом и финансовой отчетностью, вы можете пригласить его в свой экземпляр Business Central, чтобы он работал в вашими данными. Для получения дополнительной информации см. [Приглашение внешнего бухгалтера в ваш Business Central](finance-accounting.md#inviteaccountant).

### <a name="to-remove-a-users-access-to-the-system"></a>Закрытие доступа пользователя к системе

В онлайн-развертываниях вы можете удалить доступ пользователя к [!INCLUDE[prod_short](includes/prod_short.md)]. Все ссылки на пользователя сохраняются, но пользователь не может войти в систему, и активные сеансы для этого пользователя останавливаются.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Пользователи**, а затем выберите связанную ссылку.
2. Откройте страницу **Карточка пользователя** для соответствующего пользователя, затем в поле **Статус** выберите **Отключено**.
3. Чтобы снова предоставить пользователю доступ, измените значение поля **Статус** на **Включено**.

Вы также можете удалить лицензию у пользователя в центре администрирования Microsoft 365. Затем пользователь не может войти в систему. Для получения дополнительной информации см. раздел [Удаление лицензий у пользователей](/microsoft-365/admin/manage/remove-licenses-from-users),

- Администратор может добавить пользователя, выбрав действие **Обновить пользователей из Microsoft 365** на странице **Пользователи**, как описано в разделе [Добавление пользователя или обновление информации о пользователе в Business Central](#adduser).

## <a name="managing-users-and-licenses-in-on-premises-deployments"></a>Управление пользователями и лицензиями в локальных развертываниях

Для локальных развертываний количество лицензий пользователей указывается в файле лицензии (.flf). Когда администратор или партнер Майкрософт отправляет файл лицензии, администратор может указать, какие пользователи могут войти в [!INCLUDE[prod_short](includes/prod_short.md)].

Для локальных развертываний администратор создает, редактирует и удаляет пользователей непосредственно со страницы **Пользователи**.

### <a name="to-edit-or-delete-a-user-in-an-on-premises-deployment"></a>Редактирование или удаление пользователя в локальном развертывании

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Пользователи**, а затем выберите связанную ссылку.
2. Выберите пользователя, которого требуется отредактировать, а затем выберите действие **Изменить**.
3. На странице **Карточка пользователя** измените информацию требуемым образом.  
4. Чтобы удалить пользователя, выберите пользователя, которого требуется удалить, а затем выберите действие **Удалить**.

> [!NOTE]
> Для локальных развертываний администратор может указать, как проверять подлинность учетных данных пользователя в экземпляре [!INCLUDE[server](includes/server.md)]. Когда вы создаете пользователя, вы указываете тип учетных данных, который вы используете.
>
> Для получения дополнительной информации см. раздел [Типы аутентификации и учетных данных](/dynamics365/business-central/dev-itpro/administration/users-credential-types) в справке администрирования для [!INCLUDE[prod_short](includes/prod_short.md)].
