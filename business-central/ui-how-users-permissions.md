# <a name="create-users-according-to-licenses"></a>Создание пользователей в соответствии с лицензиями

В этой статье описано, как администраторы создают пользователей и определяют, кто может войти в [!INCLUDE[d365fin](includes/d365fin_md.md)], и какие разрешения предоставляются разным типам пользователей в соответствии с лицензиями.

Когда вы создаете пользователей в [!INCLUDE[d365fin](includes/d365fin_md.md)], вы можете назначать им определенные разрешения с помощью наборов разрешений и организовывать пользователей в группы пользователей. Группы пользователей упрощают управление разрешениями для нескольких пользователей одновременно. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).  

> [!NOTE]
> Процесс управления пользователями и лицензиями зависит от того, развернуто ли [!INCLUDE[d365fin](includes/d365fin_md.md)] в сети или локально. Для [!INCLUDE [prodshort](includes/prodshort.md)] (онлайн) необходимо добавить пользователей из Microsoft 365. В локальных развертываниях вы можете создавать, редактировать и удалять пользователей напрямую.  

## <a name="managing-users-and-licenses-in-online-deployments"></a>Управление пользователями и лицензиями в онлайн-развертываниях

В онлайн-версии [!INCLUDE[d365fin](includes/d365fin_md.md)] количество пользователей определяется подпиской и добавляется к вашему арендатору в Центре партнеров Майкрософт, обычно вашим партнером Microsoft. Для получения дополнительной информации см. разделы [Добавление нового клиента](/partner-center/add-a-new-customer) и [Создание, приостановка или отмена подписок клиентов](/partner-center/create-a-new-subscription) в справке Центра партнеров Майкрософт.

Чтобы определить, кто может войти в [!INCLUDE[d365fin](includes/d365fin_md.md)], вы должны назначить лицензии на продукт пользователям в соответствии с ролями, которые они будут выполнять в [!INCLUDE[d365fin](includes/d365fin_md.md)]. Это можно сделать следующими способами:

- Администратор Microsoft 365 вашей компании может сделать это в [Центре администрирования Microsoft 365](https://admin.microsoft.com). Для получения дополнительной информации см. раздел [Добавление пользователей по отдельности или массово Microsoft 365](https://aka.ms/CreateOffice365Users).  
- Партнер Майкрософт может назначать лицензии в Центре администрирования Microsoft 365 или в Центре партнеров Майкрософт. Для получения дополнительной информации см. раздел [Задачи управления пользователями для организаций клиентов](/partner-center/assign-licenses-to-users) в справке Центра партнеров Майкрософт.

Для получения дополнительной информации см. раздел [Администрирование Business Central Online](/dynamics365/business-central/dev-itpro/administration/tenant-administration) в справке администратора.

После назначения пользователям лицензии [!INCLUDE[d365fin](includes/d365fin_md.md)] в Microsoft 365 вы можете импортировать их на страницу **Пользователи** в [!INCLUDE[d365fin](includes/d365fin_md.md)] с помощью действия **Получить новых пользователей из Office 365**.

### <a name="to-add-a-user-or-update-user-information-in-business-central"></a><a name="adduser"></a>Добавление пользователя или обновление данных пользователя в Business Central

Используйте специальные функции импорта для добавления новых пользователей или обновления информации о пользователях в [!INCLUDE[d365fin](includes/d365fin_md.md)] из Центра администрирования Microsoft 365.  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.  
2. Выберите действие **Обновить пользователей из Office 365**.

    Если вы добавляете пользователей из Microsoft 365, выберите действие **Получить новых пользователей из Office 365**.  
3. Следуйте инструкциям в появившемся руководстве.

Новые пользователи и пользовательская информация в вашей подписке Microsoft 365 будет добавлена на страницу **Пользователи** в [!INCLUDE[d365fin](includes/d365fin_md.md)]. Теперь вы можете назначать группы пользователей и разрешения. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).  

Для получения дополнительной информации о синхронизации пользовательской информации с Microsoft 365 см. раздел [Синхронизация с Microsoft 365](#m365).

> [!NOTE]
> Если вы используете внешнего бухгалтера для управления вашим учетом и финансовой отчетностью, вы можете пригласить его в свой экземпляр Business Central, чтобы он работал в вашими данными. Для получения дополнительной информации см. [Приглашение внешнего бухгалтера в ваш Business Central](finance-accounting.md#inviteaccountant)

### <a name="to-change-the-assigned-license-for-a-user"></a>Изменение назначенной лицензии для пользователя

Иногда вам может понадобиться изменить лицензию, назначенную пользователю. Например, если вы решили использовать модуль управления сервисом и, следовательно, необходимо обновить все лицензии Essential до Premium. Или если ответственность пользователя изменилась, и вам необходимо заменить лицензию Team Member на Essential.

1. Изменение лицензии в центре администрирования Microsoft 365. Для получения дополнительной информации см. раздел [Добавление пользователей по отдельности или массово Microsoft 365](https://aka.ms/CreateOffice365Users).
2. Войдите в [!INCLUDE[d365fin](includes/d365fin_md.md)] в качестве администратора.
3. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
4. На странице **Пользователи** выберите действие **Восстановить группы пользователей по умолчанию**.

Пользователи будут перемещены в соответствующую группу пользователей, и наборы разрешений будут обновлены. Дополнительные сведения см. в разделе [Управление разрешениями с помощью групп пользователей](ui-define-granular-permissions.md).

> [!NOTE]
> Всем пользователям должна быть назначена одинаковая лицензия, Essential или Premium. Подробнее см. Руководство по лицензированию Microsoft Dynamics 365 Business Central. Руководство доступно для скачивания на веб-сайте [Business Central](https://dynamics.microsoft.com/business-central/overview/).

### <a name="to-remove-a-users-access-to-the-system"></a>Закрытие доступа пользователя к системе

В онлайн-развертываниях вы можете удалить доступ пользователя к [!INCLUDE[d365fin](includes/d365fin_md.md)]. Все ссылки на пользователя сохраняются, но пользователь не может войти в систему, и активные сеансы для этого пользователя останавливаются.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
2. Откройте страницу **Карточка пользователя** для соответствующего пользователя, затем в поле **Состояние** выберите **Отключено**.
3. Чтобы снова предоставить пользователю доступ, измените значение поля **Статус** на **Включено**.

Вы также можете удалить лицензию у пользователя в Центре администрирования Microsoft 365. Затем пользователь не может войти в систему. Для получения дополнительной информации см. раздел [Удаление лицензий у пользователей](https://docs.microsoft.com/office365/admin/manage/remove-licenses-from-users),

## <a name="the-device-license"></a>Лицензия на устройство

Лицензия на устройство Dynamics 365 Business Central позволяет нескольким пользователям одновременно использовать устройство, на которое распространяется лицензия. Например, это может быть устройство POS-терминала, магазина или склада. Когда вы приобрели некоторое количество лицензий на устройство, одновременно входить может это количество пользователей, назначенных группе пользователей устройства "Dynamics 365 Business Central Device Users". Подробнее см. Руководство по лицензированию Microsoft Dynamics 365 Business Central. Руководство доступно для скачивания на веб-сайте [Business Central](https://dynamics.microsoft.com/business-central/overview/).

Администратор Microsoft 365 вашей компании или партнер Microsoft может создать группу пользователей устройства "Dynamics 365 Business Central Device Users" и добавить пользователей устройства в качестве участников в [Центре администрирования Microsoft 365](https://admin.microsoft.com/) или на [портале Azure](https://portal.azure.com/).

### <a name="device-user-limitations"></a>Ограничения пользователей устройства

Пользователи с лицензией на устройство не могут выполнять следующие задачи в [!INCLUDE[d365fin](includes/d365fin_md.md)]:

- Настройка запланированных заданий в очереди заданий. Пользователи устройств являются параллельными пользователями, и поэтому мы не можем гарантировать, что нужный пользователь присутствует в системе при выполнении задачи, которая требуется.

- Пользователь устройства не может быть первым пользователем, входящим в систему. Пользователь типа "Администратор", "Полный пользователь" или "Внешний бухгалтер" должен быть первым, кто входит в систему [!INCLUDE[d365fin](includes/d365fin_md.md)], чтобы настроить ее. Для получения дополнительной информации см. раздел [Администрирование Business Central Online](/dynamics365/business-central/dev-itpro/administration/tenant-administration) в справке администратора.

### <a name="to-create-a-dynamics-365-business-central-device-users-group"></a>Чтобы создать группу "Dynamics 365 Business Central Device Users"

1. В Центре администрирования Microsoft 365 перейдите на страницу **Группы**.
2. Выберите действие **Добавить группу**.
3. На странице **Выберите тип группы** выберите действие **Безопасность**, а затем выберите действие **Добавить**.
4. На странице **Основы** введите **Dynamics 365 Business Central Device Users** в качестве названия группы.
  
   >[!NOTE]
   >Название группы должно быть написано на английском языке в точности так, как показано в шаге 4, даже если вы используете другой язык.
5. Нажмите кнопку **Закрыть**.

> [!NOTE]
> Вы также можете создать группу типа Microsoft 365. Для получения дополнительной информации см. [Сравнение групп](https://docs.microsoft.com/office365/admin/create-groups/compare-groups)

### <a name="to-add-members-to-the-group"></a>Чтобы добавить участников в группу

1. В Центре администрирования Microsoft 365 обновите страницу **Группы**, чтобы появилась ваша новая группа.
2. Выберите грппу **Dynamics 365 Business Central Device Users**, а затем выберите действие **Показать всех и управлять членами**.
3. Выберите действие **Добавить членов**.
4. Выберите пользователей, которых требуется добавить, а затем нажмите кнопку **Сохранить**.
5. Трижды нажмите кнопку **Закрыть**.

Вы можете добавить в группу "Dynamics 365 Business Central Device Users" столько пользователей, сколько нужно. Однако количество устройств, на которые пользователи могут одновременно войти, определяется количеством приобретенных лицензий на устройства.

> [!NOTE]
> Вам не нужно назначать лицензию [!INCLUDE[d365fin](includes/d365fin_md.md)] пользователям, которые являются членами группы "Dynamics 365 Business Central Device Users".

## <a name="managing-users-and-licenses-in-on-premises-deployments"></a>Управление пользователями и лицензиями в локальных развертываниях

Для локальных развертываний количество лицензий пользователей указывается в файле лицензии (.flf). Когда администратор или партнер Майкрософт отправляет файл лицензии, администратор может указать, какие пользователи могут войти в [!INCLUDE[d365fin](includes/d365fin_md.md)].

Для локальных развертываний администратор создает, редактирует и удаляет пользователей непосредственно со страницы **Пользователи**.

### <a name="to-edit-or-delete-a-user-in-an-on-premises-deployment"></a>Редактирование или удаление пользователя в локальном развертывании

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Пользователи**, затем выберите соответствующую ссылку.
2. Выберите пользователя, которого требуется отредактировать, а затем выберите действие **Изменить**.
3. На странице **Карточка пользователя** измените информацию требуемым образом.  
4. Чтобы удалить пользователя, выберите пользователя, которого требуется удалить, а затем выберите действие **Удалить**.

> [!NOTE]
> Для локальных развертываний администратор может указать, как проверять подлинность учетных данных пользователя в экземпляре [!INCLUDE[server](includes/server.md)]. Когда вы создаете пользователя, вы указываете тип учетных данных, который вы используете.
>
> Для получения дополнительной информации см. раздел [Типы аутентификации и учетных данных](/dynamics365/business-central/dev-itpro/administration/users-credential-types) в справке администрирования для [!INCLUDE[d365fin](includes/d365fin_md.md)].
