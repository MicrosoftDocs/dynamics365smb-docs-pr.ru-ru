# <a name="manage-profiles"></a>Управление профилями

Всем пользователям [!INCLUDE[d365fin](includes/d365fin_md.md)] назначаются профили, отражающие их бизнес-роль, отдел, в котором они работают, или другую классификацию. Профили позволяют администраторам централизованно определять, что различные типы пользователей могут видеть и делать в пользовательском интерфейсе. Это позволяет последним эффективно выполнять свои бизнес-задачи.

> [!NOTE]
> Как правило, профили соответствуют ролям пользователей в организации. Поэтому в пользовательском интерфейсе профиль называется *Профиль (роль)*.

Администратор создает профили и управляет ими на странице **Профили (роли)**. У каждого профиля есть карточка, на которой задаются различные параметры для соответствующей роли, например имя роли, настройки пользователя и то, какой ролевой центр используется для профиля. Дополнительные сведения о настройках пользователей и ролевых центрах см. в разделе [Изменение базовых настроек](ui-change-basic-settings.md).

Прежде чем вы сможете администрировать профили пользователей, необходимо создать и добавить пользователей через центр администрирования Microsoft 365. После этого вы сможете назначить каждому пользователю или группе пользователей разрешения, чтобы указать, какие функции они могут просматривать и/или редактировать. Дополнительные сведения см. в разделе [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).

## <a name="page-customization"></a>Настройка страниц
Вы можете настроить макеты страниц для профиля, чтобы все пользователи, которым назначен этот профиль, видели настроенные страницы. При настройке страниц администратор использует ту же функциональность, что и пользователи при персонализации. Дополнительные сведения см. в разделе [Настройка страниц для профилей](ui-personalization-manage.md).

## <a name="to-create-a-profile"></a>Создание профиля
Если вы не можете скопировать существующий профиль, вы можете создать новый вручную.

1. Выберите значок ![Поиск страницы или отчета](media/ui-search/search_small.png "Значок поиска страницы или отчета"), введите **Профили (роли)**, а затем выберите соответствующую ссылку.  
2. На странице **Профили (Роли)** выберите действие **Создать**.  
3. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

## <a name="to-copy-a-profile"></a>Копирование профиля
Чтобы сэкономить время, вы можете создать новый профиль путем копирования существующего. Копируйте профиль, который имеет настройки, аналогичные тому, который вы хотите создать.

> [!NOTE]
> При копировании профиля также копируются все задействованные настройки страницы, как созданные пользователем, так и полученные из расширений.

1. На странице **Профили (роли)** выберите строку профиля, который вы хотите скопировать, затем выберите действие **Копировать профиль**.
2. Заполните поля **ИД профиля** и **Отображаемое имя**, а затем выберите кнопку **ОК**.
3. На странице **Профили (роли)** откройте вновь созданную карточку профиля и отредактируйте другие поля, если необходимо.

## <a name="to-edit-a-profile"></a>Редактирование профиля
Профиль можно редактировать, изменяя значения в полях на странице **Профиль (роль)**. Однако изменения не будут видны пользователю, которому назначен профиль, пока они не выйдут и не вернутся.

> [!Caution]
> Не переименовывайте профиль, пока пользователи, которым назначен профиль, вошли в систему, поскольку пользователи могут столкнуться с тем, что продукт зависает и его необходимо перезапустить.

## <a name="to-assign-a-profile-to-a-user"></a>Назначение профиля пользователю
Пользователи могут назначить себе роль (представляющую профиль), выбрав поле **Роль** на странице **Мои настройки**. Администратор может сделать то же самое на странице **Профили (роли)**.

1. На странице **Профили (роли)** выберите профиль, который вы хотите назначить, затем выберите действие **Список персонализаций пользователей**.
2. На странице **Персонализации пользователей** выберите пользователя, которому вы хотите назначить профиль, затем выберите действие **Изменить**.
3. В поле **ИД профиля** выберите соответствующий профиль.

> [!NOTE]
> При назначении пользователю другого профиля все персонализации, внесенные пользователем с предыдущим профилем, сохраняются.

## <a name="to-define-user-settings-for-a-profile"></a>Задание настроек пользователя для профиля
На странице **Мои настройки** пользователи могут определить базовое поведение своей учетной записи, например ролевой центр, язык и то, какие уведомления они будут получать. Дополнительные сведения см. в разделе [Изменение базовых настроек](ui-change-basic-settings.md).

Администратор может задать эти настройки для профиля и тем самым применить настройки ко всем пользователям с соответствующей ролью.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Профили (роли)**, затем выберите соответствующую ссылку.
2. Выберите строку профиля, для которого вы хотите изменить настройки пользователя, выберите действие **Писок персонализации пользователей**.
3. На странице **Персонализации пользователей** откройте карточку для пользователя, настройки которого вы хотите изменить.
4. На странице **Карточка персонализации пользователя** отредактируйте поля требуемым образом.

## <a name="to-activate-a-profile"></a>Активация профиля
После создания профиля вы можете установить различные флажки, которые определяют, где, когда и как профиль и информация в нем становятся доступными для пользователей.

* На странице **Профиль (роль)** установите следующие флажки:
    - **Включено**, чтобы указать, видна ли соответствующая роль на странице **Доступные роли**, чтобы пользователи могли ее выбрать.  
    - **Использовать как профиль по умолчанию**, чтобы указать профиль, который применяется к пользователям, которым не назначена никакая конкретная роль.
    - **Отключить персонализацию**, чтобы указать, могут ли пользователи с соответствующей ролью персонализировать свою рабочую область.
    - **Показывать в обозревателе ролей**, чтобы указать, отображаются ли действия с бизнес-функциями, включенными в профиль, в расширенном представлении обозревателя ролей, обзоре функций. Для получения дополнительной информации см. раздел [Поиск страниц с помощью обозревателя ролей](ui-role-explorer.md),

## <a name="to-export-profiles"></a>Для экспорта профилей
Вы можете экспортировать профили из [!INCLUDE[d365fin](includes/d365fin_md.md)], например, чтобы повторно использовать их в другом арендаторе. Профили экспортируются в ZIP-файл, содержащий AL-файлы, которые можно использовать для разработки расширений. Дополнительные сведения см. в разделе [Использование клиента для создания профилей и настроек страниц](/dynamics365/business-central/dev-itpro/developer/devenv-design-profiles-using-client).

* На странице **Профили (роли)** выберите действие **Экспортировать профили**.

Экспортируется Zip-файл с AL-файлами для всех профилей.

## <a name="to-import-profiles"></a>Для импорта профилей
Вы можете импортировать профили, которые были экспортированы из [!INCLUDE[d365fin](includes/d365fin_md.md)]. Шаги более или менее противоположны шагам экспорта профилей. Дополнительные сведения см. в разделе [Для экспорта профилей](admin-users-profiles-roles.md#to-export-profiles).

1. На странице **Профили (роли)** выберите действие **Импортировать профили**.
2. Следуйте инструкциям мастера **Импортировать профили**.

    Если вы хотите импортировать только выбранные профили, используйте флажок **Выбрано**, чтобы указать, что импортировать.
3. Выберите кнопку **Импортировать выбранные**.

Импортируется ZIP-файл с AL-файлами для выбранных профилей.

## <a name="to-delete-a-profile"></a>Удаление профиля
Профиль можно удалить, выбрав действие **Удалить** на странице **Профили (роли)**. Существуют, однако, следующие ограничения:

- Вы не можете удалить профиль, назначенный пользователю или группе пользователей.
- Невозможно удалить профили, которые поступили из расширений. Необходимо сначала удалить соответствующее расширение.
- Профили можно удалять только по одному.

## <a name="to-delete-all-personalizations-made-by-a-user"></a>Удаление всех персонализаций, сделанных пользователем
Вы можете удалить все изменения, внесенные пользователем в страницы, которые составляют его рабочую область. Это имеет смысл делать, например, если сотрудник сменил роль и больше не нуждается в персонализациях. При удалении персонализация пользователя макет страницы возвращается к тому, который задан профилем.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Персонализации пользователей**, затем выберите соответствующую ссылку.

    На странице **Персонализация пользователей** перечислены все пользователи, которые сделали персонализации.

2. Откройте карточку пользователя, персонализации которого вы хотите удалить.
3. На странице **Карточка персонализации пользователя** выберите действие **Очистить персонализированные страницы** и примите появившееся сообщение.

Пользователь увидит изменения при следующем входе в систему.

Также можно удалить все настройки страниц для профиля. Дополнительные сведения см. в разделе [Удаление всех настроек для профиля](ui-personalization-manage.md#to-delete-all-customizations-for-a-profile).

## <a name="to-delete-personalizations-for-specific-pages"></a>Удаление персонализаций для конкретных страниц
Вы можете удалить персонализации, сделанные одним или несколькими пользователями на определенных страницах, составляющих их рабочую область. Это имеет смысл делать, например, если из-за изменения в бизнес-процессе какая-либо персонализация больше не должна использоваться пользователями. При удалении персонализация пользователя макет страницы возвращается к тому, который задан профилем.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Персонализации страниц пользователя**, затем выберите соответствующую ссылку.

    На странице **Персонализация страниц пользователя** перечислены все страницы, которые были персонализированы, и пользователь, которому они принадлежат.

    > [!Note]
    > Флажок в поле **Персонализация из прежних версий** указывает, что персонализация была сделана в более старой версии [!INCLUDE[d365fin](includes/d365fin_md.md)], в которой персонализации обрабатывались по другому принципу. Для пользователей, которые пытаются персонализировать эти страницы, эта операция блокируется, если они не решат разблокировать страницу. Дополнительные сведения см. в разделе [Почему заблокирована персонализация страницы?](ui-personalization-locked.md).

2. Выберите строку персонализации страницы, которую вы хотите удалить, затем выберите действие **Удалить**.

Пользователь увидит изменения при следующем входе в систему.  

Также можно удалить настройки отдельных страниц профиля. Дополнительные сведения см. в разделе [Удаление настройки для конкретных страниц профиля](ui-personalization-manage.md#to-delete-customization-for-specific-pages-for-a-profile).

## <a name="managing-user-sessions"></a>Управление сеансами пользователей

Как администратор интернет-версии [!INCLUDE[prodshort](includes/prodshort.md)], вы можете управлять сеансами пользователей в центре администрирования. Дополнительные сведения см. в [Управление сеансами](/dynamics365/business-central/dev-itpro/administration/tenant-admin-center-environments#managing-sessions) в содержимом по администрированию.  

Для [!INCLUDE[prodshort](includes/prodshort.md)] локальная версия вы можете управлять сеансами, например, с помощью SQL Server Management Studio. Дополнительные сведения см. в [Техническая документация по SQL Server](/sql/sql-server).
