# Управление профилями пользователей

Назначьте всех пользователей профилям, которые отражают:

* Их бизнес-роль
* Отдел, в котором они работают
* Другой тип классификации

Профили позволяют администраторам централизованно определять и управлять доступом различных типов пользователей к Business Central.

Как правило, профили соответствуют ролям пользователей в организации. Поэтому в пользовательском интерфейсе профиль называется *Профиль (роль)*.

Администратор создает профили и управляет ими на странице **Профили (роли)**. В каждом профиле есть карточка, где вы управляете настройками соответствующей роли. Карточка содержит следующие сведения:

* Имя роли
* Параметры пользователей
* Ролевой центр, который использует профиль

Дополнительные сведения о настройках пользователей и ролевых центрах см. в разделе [Изменение базовых настроек](ui-change-basic-settings.md).

Прежде чем вы сможете управлять профилями пользователей, необходимо создать и добавить пользователей через центр администрирования Microsoft 365. Затем вы можете назначить разрешения для каждого пользователя или группы пользователей. Разрешения определяют функции, к которым пользователи могут получить доступ. Дополнительные сведения о параметрах разрешений см. в статье [Назначение разрешений пользователям и группам](ui-define-granular-permissions.md).

## Настройка страниц

Вы можете настроить макеты страниц для профиля, чтобы все пользователи, которым назначен этот профиль, видели настроенные страницы. При настройке страниц администратор использует ту же функциональность, что и пользователи при персонализации. Дополнительные сведения о настройке макетов страниц см. в статье [Настройка страниц для профилей](ui-personalization-manage.md).

## Создать профиль

Если вы не можете скопировать существующий профиль, вы можете создать новый вручную.

1. Выберите ![Поиск страницы или отчета.](media/ui-search/search_small.png "Значок поиска страницы или отчета") , введите **Профили (роли)**, а затем выберите соответствующую ссылку.  
2. На странице **Профили (роли)** выберите действие **Новый** .  
3. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

> [!TIP]
> Если вы хотите, чтобы определенный профиль был доступен только для очень определенных пользователей, вы можете установить поле **Описание** как `Navigation menu only.`. Таким образом, профиль исключается из списка доступных ролей в **Мои настройки**.

## Копирование профиля

Чтобы сэкономить время, вы можете создать новый профиль путем копирования существующего. Копируйте профиль, который имеет настройки, аналогичные тому, который вы хотите создать.

При копировании профиля также копируются все задействованные настройки страницы, как созданные пользователем, так и полученные из расширений.

1. На странице **Профили (роли)** выделите строку для профиля, который вы хотите скопировать, а затем выберите действие **Копировать профиль** .
2. Заполните поля **Идентификатор профиля** и **Отображаемое имя**, а затем нажмите кнопку **ОК** .
3. На странице **Профили (роли)** откройте вновь созданную карточку профиля и отредактируйте другие поля, если необходимо.

## Редактировать профиль

Профиль можно редактировать, изменяя значения в полях на странице **Профили (роли)**. Однако изменения не будут видны пользователю, которому назначен профиль, пока он не выйдет из системы и не войдет в нее снова.

> [!Caution]
> Не переименовывайте профиль, пока пользователи, которым назначен профиль, вошли в систему, поскольку пользователи могут столкнуться с тем, что продукт зависает и его необходимо перезапустить.

## Назначить профиль пользователю

Пользователи могут назначить себе роль (представляющую профиль), выбрав поле **Роль** на странице **Мои настройки**. Администратор может сделать то же самое на странице **Профили (роли)**.

1. На странице **Профили (роли)** выберите профиль, который вы хотите назначить, а затем выберите действие **Список персонализации пользователей** .
2. На странице **Настройки пользователя**  выберите пользователя, которому вы хотите назначить профиль, а затем выберите действие **Изменить** .
3. В поле **ИД профиля** выберите соответствующий профиль.

При назначении пользователю другого профиля все персонализации, внесенные пользователем с предыдущим профилем, сохраняются.

## Определить пользовательские настройки для профиля

На странице **Мои настройки** пользователи могут определить базовое поведение своей учетной записи, например ролевой центр, язык и то, какие уведомления они будут получать. Дополнительные сведения о настройках пользователей см. в статье [Изменение базовых настроек](ui-change-basic-settings.md).

Как администратор, вы можете определить настройки для профиля. Настройки применяются ко всем пользователям, которым назначена роль.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать"), , введите **Профили (роли)**, а затем выберите соответствующую ссылку.
2. Выберите строку для профиля, для которого вы хотите изменить настройки пользователя, а затем выберите действие **Список персонализаций пользователя** .
3. На странице **Персонализации пользователей** откройте карточку для пользователя, настройки которого вы хотите изменить.
4. На странице **Карточка персонализации пользователя** отредактируйте поля требуемым образом.

## Активировать профиль

После создания профиля вы можете определить, если, где и как профиль и информация в нем становятся доступными для пользователей.

На странице **Профили (роли)** установите следующие флажки:

* **Включено**, чтобы указать, видна ли соответствующая роль на странице **Доступные роли**, чтобы пользователи могли ее выбрать.  
* **Использовать как профиль по умолчанию**, чтобы указать профиль, который применяется к пользователям, которым не назначена никакая конкретная роль.
* **Отключить персонализацию**, чтобы указать, могут ли пользователи с соответствующей ролью персонализировать свою рабочую область.
* **Показывать в обозревателе ролей**, чтобы указать, отображаются ли действия с бизнес-функциями, включенными в профиль, в расширенном представлении обозревателя ролей, обзоре функций. Дополнительные сведения об обозревателе ролей см. в статье [Поиск страниц с помощью обозревателя ролей](ui-role-explorer.md),

## Экспорт профилей

Вы можете экспортировать профили из [!INCLUDE[prod_short](includes/prod_short.md)] и использовать их в другом арендаторе. Профили экспортируются в ZIP-файл, который содержит файлы Application Language (AL). Вы можете повторно использовать файлы AL для разработки расширений. Дополнительные сведения об экспорте профилей см. в статье [Использование клиента для создания профилей и настроек страниц](/dynamics365/business-central/dev-itpro/developer/devenv-design-profiles-using-client).

* На странице **Профили (роли)** выберите действие **Экспортировать профили** .

    Это действие экспортирует ZIP-файл, который содержит файлы AL для всех профилей.

## Импорт профилей

Вы можете импортировать профили, которые были экспортированы из Business Central. Шаги более или менее противоположны шагам экспорта профилей.

1. На странице **Профили (роли)** выберите действие **Импортировать профили** .
1. Следуйте инструкциям мастера **Импортировать профили**.

    Если вы хотите импортировать только выбранные профили, используйте флажок **Выбрано**, чтобы указать, что импортировать.
1. Нажмите кнопку **Импортировать выбранное** .

    Это действие импортирует ZIP-файл, который содержит файлы AL для выбранных профилей.

## Удалить профиль

Профиль можно удалить, выбрав действие **Удалить** на странице **Профили (роли)**. Существуют, однако, следующие ограничения:

* Невозможно удалить профиль, назначенный пользователю или группе пользователей.
* Невозможно удалить профили, которые поступают из расширений. Необходимо сначала удалить соответствующее расширение.
* Профили можно удалять только по одному.

## Удалить персонализации

### Удалить все персонализации, сделанные определенным пользователем

Вы можете удалить все изменения, внесенные пользователем на любые страницы, что вернет страницы к исходному макету. Персонализации не связаны с профилем (ролью). Если пользователь персонализирует страницу, то он увидит персонализацию на странице независимо от того, какую роль он использует.<!--Deleting changes can be useful, for example, if an employee changes role and no longer needs them. The profile defines the page layout and deletions restore it back to that definition.--> 

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Настройки пользователя**, а затем выберите соответствующую ссылку.

    На странице **Настройки пользователя** перечислены все пользователи.<!--who make personalizations-->.

2. Выберите пользователя, персонализации которого вы хотите удалить.
3. На странице **Настройки пользователя** карточка выберите действие **Очистить персонализированные страницы**, а затем примите появившееся сообщение.

Пользователь увидит изменения при следующем входе в систему.

Также можно удалить все настройки страниц для профиля. Дополнительные сведения см. в разделе [Удаление всех настроек для профиля](ui-personalization-manage.md#delete-all-customizations-for-a-profile).

### Удалить персонализации для определенных страниц

Вы можете удалить персонализации, сделанные одним или несколькими пользователями на определенных страницах. Удаление персонализаций может быть полезно, например, если изменение в бизнес-процессе означает, что персонализация не может быть использована. При их удалении восстанавливается макет страницы, определенный профилем.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать"), , введите **Персонализированные страницы**, а затем выберите соответствующую ссылку.

    На странице **Персонализированные страницы** перечислены все персонализированные страницы и пользователи, которым они принадлежат.

    <!--A checkmark in the **Legacy Personalization** field indicates that the personalization was done in an older version of [!INCLUDE[prod_short](includes/prod_short.md)], which handled personalization differently. Users who try to personalize these pages are locked from doing so unless they choose to unlock the page.-->

2. Выберите строку для персонализации страницы, которую вы хотите удалить, а затем выберите действие **Удалить** .

Пользователь увидит изменения при следующем входе в систему.  

Также можно удалить настройки отдельных страниц профиля. Дополнительные сведения см. в разделе [Удаление настройки для конкретных страниц профиля](ui-personalization-manage.md#delete-customization-for-specific-pages-for-a-profile).

## Управление сеансами пользователей

Как администратор интернет-версии [!INCLUDE[prod_short](includes/prod_short.md)], вы можете управлять сеансами пользователей в центре администрирования. Дополнительные сведения см. в [Управление сеансами][def] в содержимом по администрированию.  

Для [!INCLUDE[prod_short](includes/prod_short.md)] локальная версия вы можете управлять сеансами, например, с помощью SQL Server Management Studio. Дополнительные сведения см. в [Техническая документация по SQL Server](/sql/sql-server).
