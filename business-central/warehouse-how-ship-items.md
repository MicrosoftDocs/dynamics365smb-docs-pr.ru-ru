# <a name="ship-items-with-a-warehouse-shipment" />Отгрузка товаров с помощью складской отгрузки

В [!INCLUDE[prod_short](includes/prod_short.md)] подберите и отгрузите товары с использованием одного из четырех методов, как описано в следующей таблице.

|Метод|Исходящий процесс|Требуется подбор|Требуется отгрузка|Уровень сложности (подробнее см. в разделе [Обзор Warehouse Management](design-details-warehouse-management.md))|  
|------|----------------|-----|---------|-------------------------------------------------------------------------------------|  
|А|Выполните учет подбора и отгрузки из строки заказа|||Нет отдельной складской операции.|  
|Б|Выполните учет подбора и отгрузки из складского документа подбора|Включено||База: последовательно по каждому заказу.|  
|Т|Выполните учет подбора и отгрузку из документа складской отгрузки||Включено|База: объединенная проводка получения/отправки для нескольких заказов.|  
|Д|Выполните учет подбора из складского документа подбора и учет отгрузки из документа складской отгрузки|Включено|Включено|Дополнительно|  

Чтобы узнать больше о отгрузке товаров, перейдите к [Исходящий складской поток](design-details-outbound-warehouse-flow.md).

Эта статья относится к методам В и Г в таблице. В обоих методах вы начинаете с создания документа отгрузки из исходного бизнес-документа. Затем следует скомплектовать товары для отгрузки.

Если склад требует складских отгрузок, вы можете отгружать товары на основе исходных документов, выпущенных на склад. Выпуск исходных документов делает товары, содержащиеся в них, готовыми к обработке на складе. Ниже приведены примеры исходных документов:

* Заказы на продажу
* Заказы на возврат покупки
* Заказы на перемещение
* Сервисные заказы

Вы можете создать складскую отгрузку одним из двух способов:

* В режиме распределения, когда работа выполняется на однократной основе. Выберите действие **Создать складскую отгрузку** в исходном документе, чтобы создать складскую отгрузку для документа.
* В режиме извлечения, когда вы используете действие **Выпуск** в исходном документе, чтобы выпустить его на склад. Сотрудник склада создает **складскую отгрузку** для одного или нескольких выпущенных исходных документов. Приведенная ниже процедура описывает, как создать складскую отгрузку в режиме извлечения.

## <a name="to-ship-items-using-a-warehouse-shipment-document" />Отгрузка товаров с помощью документа складской отгрузки

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Складские отгрузки**, затем выберите соответствующую ссылку.  
2. Выберите **Создать**.  
3. В поле **Номер** выберите серию номеров, которая будет использоваться для создания ИД отгрузки.  
4. В поле **Код склада** выберите склад, откуда вы отгружаете товар. 

    При получении строк исходного документа некоторая информация из склада копируется в каждую строку.  
5. Если для склада требуются ячейки, заполните поле **Код ячейки**. В зависимости от ваших настроек [!INCLUDE[prod_short](includes/prod_short.md)]] может добавить для вас код ячейки. Подробнее см. в разделе [Коды зон и ячеек](warehouse-how-ship-items.md#zone-and-bin-codes).  
6. Получить исходный документ можно двумя способами:

    * Выберите действие **Получить исходные документы**. Откроется страница **Исходные документы — исходящие**. Здесь вы можете выбрать один или несколько исходных документов, выпущенных на склад, которые требуют отгрузки.
    * Выберите действие **Исп. фильтры для получ. исх. документов**. Откроется страница **Фильтры для получения исходных документов**. Вы можете выбрать фильтр исходного документа и применить его. Строки всех выпущенных исходных документов, которые удовлетворяют критериям фильтрации, теперь добавляются на страницу **Складская отгрузка**. Подробнее см. в разделе [Практическое руководство. Использование фильтров при получении исходных документов](warehouse-how-ship-items.md#how-to-use-filters-to-get-source-documents).

    > [!NOTE]
    > Если на складе используются переброска и ячейки для каждой строки, можно просмотреть количество товаров, помещенных в ячейки переброски. [!INCLUDE [prod_short](includes/prod_short.md)] вычисляет эти количества автоматически при обновлении полей отгрузки. Если товары в отгрузке, которую вы готовите, можно создать подбор для всех товаров, и затем завершить отгрузку. Подробнее в разделе [Переброска товаров](warehouse-how-to-cross-dock-items.md).

7. Создать складской подбор. Если складу требуется подбор, вы можете создать операции подбора для складских отгрузок одним из двух способов:

    * В режиме распределения вы используете действие **Создать подбор**. Выберите строки для подбора и укажите информацию о подборе. Например, из каких ячеек брать и куда помещать, и сколько единиц обрабатывать. Ячейки могут быть предварительно определены для склада или ресурса.
    * В режиме распределения вы используете действие **Выпустить**. На странице **Журнал подбора** использовать действие **Получить складские документы**, чтобы получить назначенные им подборы. Если подборы на складе полностью назначены, строки в документе **Журнал подбора** удаляются. Подробнее см. в разделе [Подбор товаров для складской отгрузки](warehouse-how-to-pick-items-for-warehouse-shipment.md).

    > [!TIP]
    > Для склада, который не требует подбора, вы можете распечатать складскую отгрузку и использовать ее в качестве сборочного листа.

8. Укажите количество, которое должно быть отгружено.  

    Для склада, требующего подбора, поле **Кол-во для отгрузки** обновляется автоматически при регистрации подбора. В противном случае поле **Кол-во для отгрузки** заполняется информацией о недопоставленном количестве для каждой строки при создании строки складской отгрузки.

    Можно изменить количество, но нельзя отгрузить больше товаров, чем указано в поле **Недопоставленное кол-во** в строке исходного документа или в поле **Подобр. кол-во**, если требуется подбор.

    Чтобы заполнить поле **Кол-во для отгрузки** для всех строк нулевыми значениями, выберите действие **Удалить кол-во для отгрузки**. Например, установка количества на ноль полезна, если вы используете сканер штрих-кодов для обновления количества отгрузки. Чтобы добавить количество, доступное для отгрузки, выберите действие **Автозаполнение кол-ва для отгрузки**.

9. Выполните учет отгрузки.

## <a name="how-to-use-filters-to-get-source-documents" />Использование фильтров при получении исходных документов

Из складской отгрузки вы можете использовать страницу **Фильтры для получения исходных документов**, чтобы получить строки выпущенных исходных документов, определяющих товары, которые должно быть отгружены.

1. В складской отгрузке выберите действие **Исп. фильтры для получ. исх. документов**. 
2. Для настройки нового фильтра введите описательный код в поле **Код**, затем выберите действие **Изменить**.

    Откроется страница **Карточка фильтра исходного документа — исходящие**.

3. Используйте фильтры, чтобы определить тип строк исходного документа для извлечения. Например, вы можете выбрать типы исходных документов, такие как заказы на продажу или перемещение.
4. Выберите **Выполнить**.  

Строки всех выпущенных исходных документов, которые удовлетворяют критериям фильтрации, добавлены на страницу **Складская отгрузка**, на которой вы задали фильтры.

Количество комбинаций фильтров не ограничено. Фильтры сохраняются на странице **Фильтры для получения исходных документов** и доступны, когда они потребуются. Можно в любой момент изменить критерии, выбрав действие **Изменить**.

## <a name="zone-and-bin-codes" />Коды зон и ячеек

Если ячейки являются обязательными в складе, [!INCLUDE [prod_short](includes/prod_short.md)] предлагает код зоны и ячейки в документе складской отгрузки.

* Для расширенных конфигураций, в которых склад использует расширенные подбор и размещение, [!INCLUDE [prod_short](includes/prod_short.md)] использует ячейку, указанную в поле **Код ячейки отгрузки** в **Карточка склада**. Если **Код ячейки отгрузки** не указан, поле остается пустым. Если товар и ячейка отгрузки не совпадают, [!INCLUDE [prod_short](includes/prod_short.md)] оставляет ячейку отгрузки пустой.
* В других случаях [!INCLUDE [prod_short](includes/prod_short.md)] всегда сначала использует ячейку, указанная в поле **Код ячейки отгрузки** в **Карточка склада**. Если код ячейки отгрузки не указан, [!INCLUDE [prod_short](includes/prod_short.md)] использует код ячейки из исходного документа.

## <a name="handling-assemble-to-order-items-in-warehouse-shipments" />Обработка товаров сборки на заказ складских товаров в складских отгрузках

В сценариях сборки на заказ, поле **Кол-во для отгрузки** в строках складской отгрузки используется для записи количества собранных единиц. Количество разносится как производительность сборки при учете складской отгрузки. Для других строк складской отгрузки значение в поле **Кол-во для отгрузки** является нулевым.

Когда работники заканчивают сборку части или всего количества сборки на заказ, запишите количество в поле **Кол-во для отгрузки** в строке складской отгрузки. Затем выбрать действие **Учет отгрузки**. Выходные данные сборки разносятся, включая потребление компонентов. Расходная накладная для количества учитывается для заказа на продажу.

Из сборочного заказа можно выбрать **Строка Отгрузка со склада сборки на заказ** для доступа к строкам складской отгрузки.

После разноски расходной накладной различные поля в строке заказа на продажу обновляются, отображая продвижение работы по складу. Следующие поля также обновляются, чтобы показать объем сборки на заказ, оставшийся для сборки и отгрузки.

* **Недопоставленное количество по сборке на заказ на складе**
* **Недопоставленное количество по сборке на заказ на складе (базовое)**

> [!NOTE]
> В составных сценариях, где часть количества нужно собрать, а другую отгрузить из запасов, создаются две строки складской отгрузки. Одно значение для количества для сборки на заказ, а другое для количества запасов.
>
> Количество сборки на заказ обрабатывается, как описано в этой статье. Количество запасов обрабатывается как обычная строка складской отгрузки. Дополнительные сведения о сценариях комбинаций см. в разделе [Сборка для заказа и сборка на склад](assembly-assemble-to-order-or-assemble-to-stock.md).
