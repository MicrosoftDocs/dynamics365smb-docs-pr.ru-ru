# <a name="create-routings"></a>Создание маршрутов

Производственные организации используют маршруты, чтобы визуально отображать и направлять процесс производства.

Данный маршрут является основанием для планирования процесса, производственной мощности, планируемого задания потребности материалов и производственной документации.  

Что касается производственных спецификаций, маршруты назначаются производству конечного товара. Маршрут содержит основные данные, которые фиксируют требования процесса производства данного товара. После создания производственного заказа для товара его маршрут используется для управления графиком производственных операций, представленным на странице **Маршрутизация производственных заказов** под производственным заказом.  

Прежде чем можно будет настроить маршрут, необходимо подготовить следующее:  

- Должны быть созданы карточки товара для родительских товаров, которые участвуют в производстве. Подробнее см. в статье [Регистрация новых товаров](inventory-how-register-new-items.md).
- Производственные ресурсы настроены. Подробнее см. в статье [Настройка производственных и машинных центров](production-how-to-set-up-work-and-machine-centers.md).

## <a name="to-create-a-routing"></a>Создание маршрута

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Маршруты**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  
3. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]
4. В поле **Тип** выберите один из вариантов:
   - **Последовательный**, чтобы вычислить производственный маршрут согласно значению в поле **Номер произв. операции** .  
   - Выберите **Параллельный** для расчета операций в соответствии со значением в поле **Номер след. произв. операции** .  
5. Для редактирования маршрута задайте в поле **Статус** значение **Новый** или **В разработке**.  

    Переходите к заполнению строк маршрута.
6. В поле **Номер произв. операции** введите номер первой операции, например **10**.  
7. В поле **Тип** укажите тип используемого ресурса, например **Рабочий центр**.  
8. В поле **Номер** выберите или введите требуемый ресурс.  
9. В поле **Код связи маршрута** введите код для соединения компонента с определенной операцией. Дополнительные сведения см. в разделе [Создание связей маршрута](production-how-to-create-routings.md#to-create-routing-links).
10. В поля **Время работы** и **Время настройки** введите время, необходимое для выполнения производственной операции.

     > [!NOTE]
     > Время настройки вычисляется для каждого производственного заказа, в то время как время работы – для каждого изготавливаемого товара.  

11. В поле **Одновременные произв. мощности** укажите количество единиц выбранного ресурса, используемых для выполнения операции. Например, если два человека назначены для выполнения одной упаковочной операции, время работы делится пополам.  
12. Заполните таким же образом строки для всех операций, необходимых для создания данного товара.  
13. Строки можно скопировать из существующего маршрута, выбрав действие **Копировать маршрут**, чтобы выбрать готовые строки.  
14. Чтобы активировать маршрутизацию, в поле **Статус** выберите **Сертифицировано**.  
15. Теперь можно присоединить новый маршрут к карточке данного изготавливаемого товара, заполнив поле **Номер маршрута**. Подробнее см. в статье [Регистрация новых товаров](inventory-how-register-new-items.md).  

> [!NOTE]  
> Не забудьте пересчитать стандартную себестоимость товара из карточки **Товар**. Выберите действие **Производство**, действие **Расчет станд. себестоимости**, а затем действие **Все уровни**.  

## <a name="to-create-routing-links"></a>Создание связей маршрута

Можно создавать связи маршрутов для соединения компонентов с определенными операциями и сохранения их отношений даже при изменении производственной спецификации или маршрута. Связи маршрутов также дают возможность своевременно списывать компоненты, а именно при запуске не целого производственного заказа, а определенных связанных операций. Подробнее см. в статье [Списание компонентов в соответствии с выходом операций](production-how-to-flush-components-according-to-operation-output.md).  

Другое важное преимущество связей заключается в том, что при использовании страницы **Производственный журнал** для учета выхода и потребления эти связанные компоненты и операции отображаются в виде логической структуры процесса.  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Маршруты**, а затем выберите связанную ссылку.  
2. Откройте маршрут, содержащий операции, которые необходимо связать.  

    Убедитесь, что статус маршрутизации — **В разработке**.  

3. В нужной строке маршрута в поле **Код связи маршрута** выберите код.  
4. При необходимости добавьте другие коды связей маршрута к другим операциям в маршруте.  

    > [!NOTE]  
    > Не следует использовать один и тот же код связи маршрута в разных операциях маршрута, так как есть риск неправильно связать компонент с двумя разными операциями, из-за чего он будет потребляться дважды.  
    >
    > Чтобы по связям маршрута можно было определить операцию, с которой они связаны, рекомендуется перед именем кода связи маршрута добавлять имя операции.

5. Установите для маршрута статус **Сертифицировано**.  

    Теперь коды маршрутных ссылок назначены операции. Далее следует создать фактическую связь путем присвоения одинаковых кодов определенным компонентам в соответствующей производственной спецификации.  

6. Откройте **Производственную спецификацию**, содержащую компоненты, которые требуется связать с указанными операциями. Подробнее см. в статье [Создание производственных спецификаций](production-how-to-create-production-boms.md).
7. Убедитесь, что статус спецификации — **В разработке**.  
8. В нужной строке производственной спецификации в поле **Код связи маршрута** выберите код, назначенный операции.  
9. Добавьте коды связи маршрута к другим компонентам в соответствии с уникальными операциями, в которых они используются.  
10. Установите для производственной спецификации статус **Сертифицировано**.  

    > [!NOTE]  
    > Чтобы применить связи маршрута к существующему производственному заказу, необходимо сначала обновить этот производственный заказ. Подробнее см. в статье [Создание производственных заказов](production-how-to-create-production-orders.md).  

Теперь при создании или обновлении производственного заказа, в котором используются производственная спецификация и маршрут, выбранные компоненты будут связаны с выбранными операциями. Эта связь видна на странице **Компоненты производственного заказа** под производственным заказом. Коды связей маршрута можно добавлять и удалять в любое время.

## <a name="to-assign-personnel-tools-and-quality-measures-to-routing-operations"></a>Чтобы назначить персонал, инструменты и измерения качества операциям маршрута

При необходимости операции можно назначить персонал, обладающий требуемыми квалификациями, специальными знаниями или полномочиями. Кроме того, можно назначить инструменты и требования к качеству для операции. В этой процедуре описано, как назначить персонал. Шаги аналогичны для различных типов информации об операции.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Маршруты**, а затем выберите связанную ссылку.  
2. Откройте соответствующий маршрут.  
3. На экспресс-вкладке **Строки** выберите строку, которую требуется обработать, выберите действие **Операции**, затем выберите действие **Персонал**.  
4. Заполните поля на странице **Персонал маршрута**.  
5. Выберите кнопку **ОК**, чтобы выйти со страницы. Введенные значения копируются и присваиваются операции.  

## <a name="to-create-a-new-version-of-a-routing"></a>Создание новой версии маршрута

Принцип версии позволяет управлять несколькими версиями одного маршрута. Структура версии маршрута соответствует структуре маршрута, состоящего из заголовка версии маршрута и ее строк. Основное отличие определяется датой начала.  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Маршруты**, а затем выберите соответствующую ссылку.  
2. Выберите маршрут для копирования, затем выберите действие **Версии**.  
3. На странице **Версии маршрута** выберите действие **Создать**.
4. В поле **Код версии** введите уникальный идентификатор версии. Можно использовать любую комбинацию цифр и букв.  

    Вновь созданной версии автоматически присваивается статус **«Новая»**.  
5. Заполните соответствующим образом поля. [!INCLUDE [tooltip-inline-tip_md](../archive/SetupAndAdministration/includes/tooltip-inline-tip_md.md)]
6. Для создания строк операций выберите первую пустую строку, затем заполните **Номер операции** в соответствии с порядком операций.

    Строки операций сортируются в порядке возрастания по номерам операций. Чтобы иметь возможность вносить изменения впоследствии, рекомендуется выбирать достаточную ширину шага. Поле **Номер след. произв. операции** относится к следующей операции на маршруте.

7. Закончив настройку версии маршрута, в поле **Статус** выберите **Сертифицировано**.
