# <a name="create-service-orders"></a>Создание сервисных заказов
Страницу **Сервисный заказ** можно использовать для создания документов, в которые вводится информация о сервисе, например ремонте и обслуживании, для сервисных товаров по запросу клиента.  

При создании заказа на обслуживание необходимо заполнить только несколько полей. Некоторые поля необязательны, и многие поля заполняются автоматически при заполнении связанных с ними полей.  

## <a name="to-create-a-service-order"></a>Создание сервисного заказа    
1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Сервисные заказы**, затем выберите соответствующую ссылку.  
2. Создайте новый сервисный заказ.  
3. В поле **Номер** введите номер сервисного заказа.  

     Вместо этого, если на странице **Настройка управления сервисом** настроены серии номеров для сервисных заказов, можно нажать клавишу ВВОД для выбора следующего свободного номера сервисного заказа.  

4. В поле **Номер клиента** выберите соответствующего клиента из списка. Соответствующие поля для клиента заполняются информацией из таблицы **Клиент**.  

5. В зависимости от настроек на экспресс-вкладке **Обязательные поля**, возможно, на странице **Настройка управления сервисом** нужно будет заполнить поля **Тип сервисного заказа** и **Код менеджера**.  
6. Заполнение остальных полей необязательно.  
7. Зарегистрируйте строки сервисных товаров.  

## <a name="to-create-a-service-order-from-a-contract"></a>Создание сервисного заказа из контракта  
Можно автоматически создавать сервисные заказы на обслуживание сервисных товаров на основе сервисных контрактов.  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Создать сервисные заказы по контрактам**, затем выберите соответствующую ссылку.  
2. На экспресс-вкладке **Заголовок сервисного контракта** установите фильтры, которые нужно применить.  
3. На экспресс-вкладке **Параметры** введите в полях **Дата начала** и **Дата окончания** даты начала и окончания временного периода, для которого создаются сервисные заказы по контракту. В пакетном задании создаются сервисные заказы, включающие в себя сервисные товары в сервисных контрактах с датами следующего планового сервисного обслуживания в пределах этого периода.  

    > [!NOTE]  
    >  Существует ограничение количества дней, которые можно использовать в качестве диапазона дат при каждом использовании данного пакетного задания. Вы устанавливаете данное ограничение в поле **Сервисный заказ по контракту — макс. дней** на странице **Настройка управления сервисом**.  

4. В поле **Действие** выберите **Создать серв. заказ**.  
    > [!NOTE]  
    >  Вы не сможете создать заказ с несколькими сервисными товарами, если вы установите поле **Отдельная строка или заказ для сервисного товара** на странице **Настройка управления сервисом**. 

## <a name="to-convert-a-service-quote-to-a-service-order"></a>Преобразование сервисного предложения в сервисный заказ
Если клиент одобрил сервисное предложение, следует преобразовать его в сервисный заказ. Предложение удаляется, и настраивается новый сервисный заказ с тем же описанием, что у сервисного предложения. Дата и время отклика пересчитываются для сервисного заказа, и заказу присваивается статус **Ожидание**. Статус ремонта сервисных товаров в заказе изменяется на **Начальный**.  

В [!INCLUDE[d365fin](includes/d365fin_md.md)] выполняется поиск операций распределения для всех сервисных товаров в сервисном предложении, которые имеют статус **Активно**. Если она находит такие операции распределения, то их статус распределения изменяется на **Требуется перераспределение**. При перераспределении сервисных товаров в сервисном заказе статус операций распределения, зарегистрированных для предложения, обновляется на **Завершен**.   

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Предложения сервисных контрактов**, затем выберите соответствующую ссылку.  
2. Выберите сервисное предложение для преобразования в сервисный заказ.  
3. Выберите действие **Сделать заказ**.  

## <a name="to-check-item-availability-for-one-or-more-orders"></a>Проверка наличия товара для одного или нескольких заказов  
Можно определить, имеется ли данный товар в наличии на складе, и если нет, то можно указать, когда товар будет в наличии. Кроме того, если товар доступен для резервирования, то его можно зарезервировать, чтобы гарантировать его доступность для использования в сервисном заказе. Можно проверить наличие для определенного заказа или для всех заказов.  

1.  Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Панель диспетчера**, затем выберите соответствующую ссылку.  
2. Выполните одно из следующих действий:  

    * Для определенного заказа выберите заказ, затем выберите действие **Обзор спроса**.  
    * Для всех заказов выберите **Показать документ**. Откроется страница **Сервисный заказ**.  

3. На странице **Обзор спроса** разверните группу товара и просмотрите данные о наличии данного товара. Например можно увидеть, сколько товаров на складе. Можно также увидеть, когда будет доступен товар по невыполненному заказу (и будет ли), то есть тип источника = покупке, и были ли он зарезервирован.

## <a name="to-reserve-an-item-for-a-service-order"></a>Резервирование товара для сервисного заказа
Если необходимо обеспечить доступность товара для сервисного заказа, можно зарезервировать этот товар.

1. В поле **Поиск** введите **Сервисные заказы**, а затем выберите соответствующую ссылку.  
2. Выберите сервисный заказ, затем выберите **Изменить**.  
3. Выберите **Действия**, затем выберите **Заказ** и **Сервисные строки**.  
4. На странице **Сервисные строки** выберите товар для резервирования, затем выберите действие **Резервировать**.  
5. На странице **Резервировать** выберите **Резервировать по текущей строке**.

## <a name="to-insert-lines-based-on-standard-service-codes"></a>Чтобы вставить строки на основе стандартных кодов сервиса  
Если выполнить настройку стандартных сервисных кодов и присвоить их группам сервисных товаров, можно вставлять стандартные строки, связанные с этими стандартными сервисными кодами, в сервисные документы. Дополнительные сведения см. в разделе [Настройка стандартных сервисных кодов](service-how-setup-service-coding.md).   

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Сервисные заказы**, затем выберите соответствующую ссылку.  
2. Создайте новый сервисный заказ.  
3. Заполните соответствующим образом поля.  
4. Заполните строки сервисных товаров требуемой информацией.  
5. Выберите строку с сервисным товаром, для которого необходимо создать сервисные строки, затем выберите **Получить станд. сервисные коды**. Откроется страница **Станд. коды групп серв. товаров**, в котором отображены стандартные коды для группы сервисных товаров данной строки.  
6. Выберите соответствующий код и выберите кнопку **ОК**, чтобы ввести стандартные строки сервиса.  

> [!NOTE]  
>  Если поле **Код группы сервисных товаров** в строке сервисного товара документа пусто, что сервисный товар не принадлежит к группе сервисных товаров. В этом случае на странице **Станд. коды групп серв. товаров** будет содержаться список всех стандартных сервисных кодов. Производится выбор из списка, чтобы вставить стандартные сервисные строки в документ. Можно также выбрать из списка стандартных сервисных кодов, присвоенных определенной группе сервисных товаров. Для просмотра списка выберите соответствующий код в поле **Код группы сервисных товаров** на странице **Станд. коды групп серв. товаров**.  

## <a name="to-register-internal-or-public-comments"></a>Регистрация внутренних или публичных комментариев
Можно добавить комментарии, которые будут напечатаны в сервисных заказах и сервисных предложениях для предоставления дополнительных сведений. Можно добавить не более 80 символов, включая пробелы. Если необходимо ввести дополнительный текст, выберите следующую строку. Чтобы зарегистрировать комментарий, выберите строку, затем выберите действие **Комментарии**.  

## <a name="to-delete-invoiced-service-orders"></a>Удаление сервисных заказов, по которым выставлены счета  
Заказы обычно удаляются автоматически после выставления по ним счетов в полном объеме. Во время учета счета на странице **Учтенные сервисные счета** создается соответствующая запись. Учтенный документ можно просматривать на странице **Учтенный сервисный счет**.  

Однако сервисные заказы не удаляются автоматически, если общее количество по заказу было учтено не из самого заказа, а со страницы **Сервисный счет**. В этом случае может понадобиться удалить вручную заказы, по которым выставлены счета и которые не были удалены программой. Это можно выполнить путем запуска пакетного задания **Удалить серв. заказы, по кот. выст. сч.**.  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Удалить сервисные заказы, учтенные в счетах**, затем выберите соответствующую ссылку. Откроется страница запроса пакетного задания **Удалить сервисные заказы, учтенные в счетах**.  
2. Чтобы выбрать заказы для удаления, можно установить фильтры в полях **Номер**, **Номер клиента** и **Код плательщика**. .  
3. Выберите **ОК**.
