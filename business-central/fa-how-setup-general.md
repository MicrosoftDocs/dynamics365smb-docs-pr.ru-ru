# <a name="set-up-general-fixed-assets-information"></a>Настройка общей информации об основных средствах

Прежде чем можно будет управлять основными средствами (FA), необходимо настроить счета ГК по умолчанию, ключи распределения, шаблоны и разделы журналов для учета и переклассификации основных средств. Кроме того, определите иерархию классификации (классы и подклассы) для структурирования ваших активов и, при необходимости, определите места хранения активов.

## <a name="to-set-up-general-behavior-for-fixed-assets-functionality"></a>Настройка общего поведения для функции основных средств

Определение общее поведение функциональности основных средств, а также серии номеров документов на странице **Основные средства - настройка**.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Основные средства - настройка**, а затем выберите связанную ссылку.  
2. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

## <a name="to-set-up-fixed-asset-posting-groups"></a>Настройка учетных групп основных средств

Использовать учетные группы для определения групп основных средств. Операции для этих учетных групп учитываются на тех же счетах ГК.

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Учетные группы ОС**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.
3. На странице **Карточка учетной группы ОС** заполните поля, как требуется.

    > [!NOTE]  
    >   Чтобы балансирующие счета для различных видов учета основных средств автоматически вставлялись при выборе действия **Вставить баланс. счет ОС** в строки журнала, выполните следующий шаг на основании учета повышения стоимости.
4. На экспресс-вкладке **Балансирующий счет** в поле **Повышение стоимости - баланс. счет** выберите счет ГК, на котором необходимо учитывать балансовые операции для повышения стоимости.

Дополнительные сведения об использовании действия **Вставить баланс. счет ОС** в строках журнала ГК учета основных средств см. в разделе [Переоценка основных средств](fa-how-revalue.md).

## <a name="to-set-up-fixed-asset-journal-templates"></a>Настройка шаблонов журнала основных средств

Шаблон — это определенный заранее макет журнала. В шаблоне содержится информация о кодах трассировки, отчетах и серийных номерах. Дополнительные сведения прочитайте в разделе [Работа с финансовыми журналами](ui-work-general-journals.md).

Шаблон журнала основных средств создается программой [!INCLUDE[prod_short](includes/prod_short.md)] автоматически при первом открытии страницы **Журнал ОС**, но можно настроить дополнительные шаблоны журнала.  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Шаблоны журнала ОС**, а затем выберите связанную ссылку.  
2. Заполните соответствующим образом поля.

## <a name="to-set-up-fixed-asset-class-and-subclass-codes"></a>Настройка кодов классов и подклассов основных средств

В основных средствах можно определить иерархию классификации, которую можно использовать для группировки активов. Иерархия имеет два уровня: классы и подклассы.

### <a name="fixed-asset-class-codes"></a>Коды классов основных средств

Классы основных средств — это записи верхнего уровня в иерархии классификации, в которой группируются активы. Например, используйте классы для разделения активов на материальные и нематериальные активы. В вашей настройке необходимо создать хотя бы один класс основных средств.

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок введите **Классы ОС**, а затем выберите связанную ссылку.
2. Введите коды и наименования для классов основных средств, которые нужно создать.

### <a name="fixed-asset-subclass-codes"></a>Коды подклассов основных средств

Подклассы основных средств — это записи второго уровня в иерархии классификации, в которой группируются активы. Каждый подкласс указывает на класс верхнего уровня. Коды подклассов основных средств можно использовать для разделения основных средств на более узкие категории, например здания, транспортные средства, мебель и оборудование. В вашей настройке необходимо создать хотя бы один подкласс основных средств.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Подклассы ОС**, а затем выберите связанную ссылку.
2. Введите коды и наименования для подклассов основных средств, которые нужно создать.

## <a name="start-to-register-assets"></a>Начало регистрации активов

Если вы используете основные средства в [!INCLUDE[prod_short](includes/prod_short.md)] впервые, необходимо настроить область приложения "Главная книга". Порядок выполнения это зависит от того, интегрированы ли основные средства с Главной книгой.  

Для учета транзакций основных средств в главной книге используется следующая процедура.  

1. Выполните базовые настройки для основных средств.  
2. Заполните карточку ОС для каждого основного средства.  
3. Создайте амортизационную книгу ОС для каждого назначение амортизации (например, для формирования налоговых и финансовых отчетов). Для каждой книги амортизации определите такие термины и условия, как интеграция с главной книгой.

    Включите интеграцию главной книги, выполнив следующие шаги. В первую очередь убедитесь, что интеграция с главной книгой не включена для всех книг амортизации, затем учтите открывающие операции и, наконец, включите интеграцию с главной книгой.  
4. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Книги амортизации**, а затем выберите связанную ссылку.  
5. Выберите соответствующую карточку книги амортизации и выберите действие **Изменить**, чтобы открыть страницу **Карточка книги амортизации**.
6. На экспресс-вкладке **Интеграция** отключите все переключатели. При наличии нескольких книг амортизации, повторите этот шаг для каждой из них.  
7. В журнал основных средств введите следующие строки для каждого основного средства:
   * Строка со стоимостью покупки.
   * Строка с накопленной амортизацией до конца предыдущего финансового года.
   * Строка с накопленной амортизацией с начала текущего финансового года до даты, на которую в [!INCLUDE[prod_short](includes/prod_short.md)] задано начало расчета амортизации.

    При наличии других входящее сальдо, можно также ввести их сейчас, например "Понижение стоимости" и "Повышение стоимости".  
8. После ввода и учета строк журнала для каждого основного средства включите интеграцию главной книги в книгах амортизации.

Для учета средств, не интегрированных с главной книгой, пропустите шаги шесть и восемь.

## <a name="to-set-up-fixed-asset-location-codes-optional"></a>Настройка кодов местонахождения основных средств (необязательно)

Коды местоположения основного средства определяют идентификаторы того, где могут быть расположены активы, например в подразделениях сбыта, приема, администрации, производства или склада. С их помощью можно зарегистрировать местонахождение основного средства. Эта информация важна при учете в сферах страхования и управления товарами.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Местоположения ОС**, а затем выберите связанную ссылку.
2. Введите коды и наименования для местонахождения основных средств, которые нужно создать.

## <a name="to-set-up-fixed-asset-allocation-keys-optional"></a>Настройка ключей распределения основных средств (необязательно)

Используйте ключи распределения для распределения транзакций по различным подразделениям или проектам. Например, можно настроить ключи распределения для распределения стоимостей амортизации на транспортные средства с 35% для административного подразделения и 65% для торгового подразделения. Дополнительные сведения см. в разделе [Распределение затрат и дохода](year-allocate-costs-income.md).

Ключи распределения применяются к классам ОС, а не к отдельным основным средствам.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Учетные группы ОС**, а затем выберите связанную ссылку.  
2. На странице **Учетные группы ОС** выберите действие **Распределения**, затем выберите тип учета.
3. На странице **Распределения ОС** заполните требуемые поля.
4. Повторите шаги 2 и 3 для каждого типа учета, для которого необходимо определить ключи распределения.

## <a name="to-set-up-fixed-asset-journal-batches-optional"></a>Настройка разделов журнала ОС (необязательно)

Можно настроить несколько разделов журнала, т. е. индивидуальные журналы для каждого шаблона журнала. Например, у сотрудников могут быть собственные разделы журнала, в которых инициалы сотрудника будут использоваться как название раздела журнала. Дополнительные сведения см. в разделе [Работа с финансовыми журналами](ui-work-general-journals.md).  

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Шаблоны журнала ОС**, а затем выберите связанную ссылку.  
2. Выберите соответствующий шаблон журнала, затем выберите действие **Разделы**.
3. На странице **Разделы журнала ОС** заполните требуемые поля.
