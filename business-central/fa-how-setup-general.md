# <a name="set-up-general-fixed-assets-information"></a>Настройка общих данных основных средств
Прежде чем управлять основными средствами, необходимо настроить счета ГК по умолчанию, ключи распределения, шаблоны и разделы журналов для учета и переквалификации основных средств; можно также классифицировать основные средства по классам, таким как материальные и нематериальные.

## <a name="to-set-up-general-default-values-for-fixed-assets"></a>Настройка общих значений по умолчанию для основных средств
Общее поведение или функциональность основных средств, а также серии номеров документов настраиваются на странице **Основные средства - настройка**.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Настройка модуля "Основные средства"**, затем выберите соответствующую ссылку.  
2. Заполните соответствующим образом поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

## <a name="to-set-up-fixed-asset-posting-groups"></a>Настройка учетных групп основных средств
Для определения групп основных средств используются учетные группы. Операции для этих учетных групп учитываются на тех же счетах ГК.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Учетные группы ОС**, затем выберите соответствующую ссылку.  
2. Выберите действие **Создать**.
3. На странице **Карточка учетной группы ОС** заполните поля, как требуется.

    > [!NOTE]  
    >   Чтобы балансирующие счета для различных видов учета основных средств автоматически вставлялись при выборе действия **Вставить баланс. счет ОС** в строки журнала, выполните следующий шаг на основании учета повышения стоимости.
4. На экспресс-вкладке **Балансирующий счет** в поле **Повышение стоимости - баланс. счет** выберите счет ГК, на котором необходимо учитывать балансовые операции для повышения стоимости.

Дополнительные сведения об использовании действия **Вставить баланс. счет ОС** в строках журнала ГК учета основных средств см., например, раздел [Переоценка основных средств](fa-how-revalue.md).

## <a name="to-set-up-fixed-asset-allocation-keys"></a>Настройка ключей распределения основных средств
Транзакции могут быть распределены по различным подразделениям или проектам в соответствии с определенными пользователем ключами распределения. Например, можно настроить ключи распределения для распределения стоимостей амортизации на машины с 35% для административного подразделения и 65% для торгового подразделения. Дополнительные сведения см. в разделе [Распределение затрат и дохода](year-allocate-costs-income.md).

Ключи распределения применяются к классам ОС, а не к отдельным основным средствам.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Учетные группы ОС**, затем выберите соответствующую ссылку.  
2. На странице **Учетные группы ОС** выберите действие **Распределения**, затем выберите тип учета.
3. На странице **Распределения ОС** заполните требуемые поля.
4. Повторите шаги 2 и 3 для каждого типа учета, для которого необходимо определить ключи распределения.

## <a name="to-set-up-fixed-asset-journal-templates"></a>Настройка шаблонов журнала основных средств
Шаблон — это определенный заранее макет журнала. В шаблоне содержится информация о кодах трассировки, отчетах и серийных номерах. Дополнительные сведения см. в разделе [Работа с финансовыми журналами](ui-work-general-journals.md).

Шаблон журнала основных средств создается программой [!INCLUDE[d365fin](includes/d365fin_md.md)] автоматически при первом открытии страницы **Журнал ОС**, но можно настроить дополнительные шаблоны журнала.  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Шаблоны журнала ОС**, затем выберите соответствующую ссылку.  
2. Заполните соответствующим образом поля.

## <a name="to-set-up-fixed-asset-journal-batches"></a>Настройка разделов журнала основных средств
Можно настроить несколько разделов журнала, т. е. индивидуальные журналы для каждого шаблона журнала. Например, у сотрудников могут быть собственные разделы журнала, в которых инициалы сотрудника будут использоваться как название раздела журнала. Дополнительные сведения см. в разделе [Работа с финансовыми журналами](ui-work-general-journals.md).  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Шаблоны журнала ОС**, затем выберите соответствующую ссылку.  
2. Выберите соответствующий шаблон журнала, затем выберите действие **Разделы**.
3. На странице **Разделы журнала ОС** заполните требуемые поля.

Шаблон журнала реклассификации основных средств создается программой [!INCLUDE[d365fin](includes/d365fin_md.md)] автоматически при первом открытии страницы **Журнал реклассификации ОС**, но можно настроить дополнительные шаблоны журналов реклассификации ОС. Дополнительные сведения см. в разделе [Работа с финансовыми журналами](ui-work-general-journals.md).  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Шаблоны журнала реклассификации ОС**, затем выберите соответствующую ссылку.  
2. Заполните соответствующим образом поля.

## <a name="to-set-up-fixed-asset-reclassification-journal-batches"></a>Настройка разделов журнала реклассификации ОС
Можно настроить несколько разделов журнала, т. е. индивидуальные журналы для каждого шаблона журнала реклассификации. Например, у сотрудников могут быть собственные разделы журнала реклассификации, в которых инициалы сотрудника будут использоваться как название раздела журнала реклассификации. 2. Выберите соответствующий шаблон журнала, затем выберите действие **Разделы**.
3. На странице **Разделы журнала реклассиф. ОС** заполните требуемые поля.

## <a name="to-set-up-fixed-asset-class-codes"></a>Настройка кодов классов основных средств
Коды классов основных средств могут использоваться для разделения основных средств на группы, например, материальные и нематериальные основные средства.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Классы ОС**, затем выберите соответствующую ссылку.
2. Введите коды и наименования классов, которые нужно создать.

## <a name="to-set-up-fixed-asset-subclass-codes"></a>Настройка кодов подклассов основных средств
Коды подклассов основных средств можно использовать для разделения основных средств на категории, например здания, транспортные средства, мебель и оборудование.  

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Подклассы ОС**, затем выберите соответствующую ссылку.
2. Введите коды и наименования классов, которые нужно создать.

## <a name="to-set-up-fixed-asset-location-codes"></a>Настройка кодов местонахождения основных средств
Коды склада основного средства можно использовать для регистрации размещения основных средств, например в подразделениях сбыта, приема, администрации, производства или склада. Эта информация важна при учете в сферах страхования и управления товарами.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Местоположения ОС**, затем выберите соответствующую ссылку.
2. Введите коды и наименования для местонахождения основных средств, которые нужно создать.

## <a name="to-register-opening-entries"></a>Регистрация начальных операций
Если используете основные средства в [!INCLUDE[d365fin](includes/d365fin_md.md)] впервые, необходимо настроить область приложения "Главная книга". Порядок выполнения это зависит от того, какие основные средства интегрированы с Главной книгой.  

 Для учета транзакций основных средств в главной книге используется следующая процедура.  

1. Убедитесь в том, что были выполнены основные процедуры настройки основных средств.  
2. Создайте карточку ОС для каждого основного средства.  
3. Создайте амортизационную книгу ОС для каждого назначение амортизации (например, для формирования налоговых и финансовых отчетов). Для каждой книги амортизации следует определять такие термины и условия, как интеграция с главной книгой.  

    Включите интеграцию главной книги, выполнив следующие шаги. Убедитесь, что интеграция с главной книгой отключена для всех книг амортизации, затем учтите открывающие операции и, наконец, включите интеграцию с главной книгой.  
4. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Книги амортизации**, затем выберите соответствующую ссылку.  
5. Выберите соответствующую карточку книги амортизации и выберите действие **Изменить**, чтобы открыть страницу **Карточка книги амортизации**.
6. Убедитесь, что на экспресс-вкладке **Интеграция** все поля оставлены незаполненными (снимите все флажки). При наличии нескольких книг амортизации необходимо выключить интеграцию с ГК для каждой из них.  
7. В журнал основных средств введите следующие строки для каждого основного средства:
   * Строка со стоимостью покупки.
   * Строка с накопленной амортизацией до конца предыдущего финансового года.
   * Строка с накопленной амортизацией с начала текущего финансового года до даты, на которую в [!INCLUDE[d365fin](includes/d365fin_md.md)] задано начало расчета амортизации.

    При наличии других входящее сальдо, можно также ввести их сейчас, например "Понижение стоимости" и "Повышение стоимости".  
8. После ввода и учета строк журнала для каждого основного средства включите интеграцию главной книги в книгах амортизации.

Для учета средств, не интегрированных с главной книгой, пропустите шаги 6 и 8.
