# Пошаговое руководство. Расчет незавершенного производства для проекта

<!-- [!INCLUDE[complete_sample_data](includes/complete_sample_data.md)]   -->

С помощью проектов можно планировать использование ресурсов организации и отслеживать различные затраты, связанные с использованием ресурсов в конкретном проекте. Проекты предполагают потребление человеко-часов, станко-часов, товаров в запасах и другие типы потребления, которые необходимо отслеживать в процессе выполнения проекта. Если проект выполняется в течение длительного периода, эти затраты имеет смысл переносить на счет незавершенного производства (НЗП) в балансе по мере выполнения проекта. Затем при необходимости можно признать затраты и продажи в счетах отчета о прибыли и убытках.  

## Об этом пошаговом руководстве

 В этом пошаговом руководстве рассматриваются следующие задачи:  

-   вычисление незавершенного производства;  
-   выбор метода вычисления незавершенного производства;  
-   Исключение части проекта из незавершенного производства.  
-   Учет НЗП в Главной книге.  
-   сторнирование учета незавершенного производства.  

 На каждом этапе процесса вычисляется стоимость, и транзакции по проекту переносятся в главную книгу. Этап вычисления и учета выполняются отдельно друг от друга, что позволяет просматривать данные и вносить изменения перед учетом в главной книге. Поэтому после выполнения пакетных заданий по вычислению перед запуском пакетных заданий по учету следует убедиться в правильности всей информации.  

## Роли

 В этом пошаговом руководстве в роли действующего лица выступает член проектной группы (Татьяна).  

## Предварительные требования

 Перед выполнением заданий этого пошагового руководства необходимо установить на компьютере [!INCLUDE[prod_short](includes/prod_short.md)].  

## Сюжет

 Действия этого пошагового руководства разворачиваются вокруг организации «CRONUS Россия ЗАО», консультационно-конструкторской фирмы, занимающейся проектировкой и монтажом новых инфраструктур, например конференц-залов и офисов с мебелью, вспомогательным оснащением и камерой хранения. Работа в CRONUS организована в основном по проектному принципу, и Татьяна, член проектной группы, использует проекты для получения представления о каждом начатом CRONUS проекте, равно как и о выполненных проектах. Некоторые из проектов могут быть весьма продолжительными и выполняться месяцами. Татьяна может использовать счет НЗП для регистрации незавершенного производства и отслеживания себестоимости в процессе работы.  

## Вычисление незавершенного производства

 Организация CRONUS взялась за длительный проект, который растянется на несколько отчетных периодов. Член проектной группы Татьяна рассчитывает объем незавершенного производства (НЗП), чтобы убедиться в точности финансового отчета организации.  

 В ходе этой процедуры Татьяна выберет определенную группу задач, для которых будет вычислено незавершенное производство. На странице **Строки задач проекта** Татьяна может указать соответствующие строки в столбце **Итоги по НЗП**.  

 В следующей таблице описаны три параметра.  

|Поле|Описание|  
|-------------------------------------|---------------------------------------|  
|**\<blank\>**|Оставьте это поле пустым, если задача проекта входит в группу задач.|  
|**Итог**|Определяет диапазон или группу задач, которые будут включены в расчет НЗП и признания. В группе любая задача проекта, у которой для параметра **Тип задачи проекта** задано значение **Учет**, будет включена в итоги по НЗП, если только для параметра **Итоги по НЗП** этой задачи не задано значение **Исключено**.|  
|**Исключено**|Применяется только к задаче, у которой **Тип задачи проекта** — **Учет**. Эта задача не включается при расчете НЗП и признания.|  

 В следующем пошаговом руководстве Татьяна применяет стандартный для ее организации метод себестоимости, чтобы рассчитать НЗП. Татьяна указывает, какая часть проекта должна быть включена в расчет НЗП, присваивая значения «Итоги по НЗП» различным строкам задач проекта.  

### Расчет НЗП  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проект**, а затем выберите соответствующую ссылку.  
2. В списке **Проекты** выберите проект **Графика**, затем выберите действие **Изменить**. Откроется карточка проекта в режиме редактирования.  

     Незавершенное производство можно вычислить на основе себестоимости, суммы продаж, себестоимости проданных товаров, процента выполнения или выполненного контракта. В этом примере CRONUS использует метод себестоимости.  

3. На экспресс-вкладке **Учет** щелкните в поле **Метод НЗП** и выберите **Себестоимость**.  
4. Выберите действие **Строки задач проекта** и установите следующие значения в поле **Итоги по НЗП**.  

     В следующей таблице приводится описание значений.  

    |№ задачи проекта|Общее поле НЗП|  
    |------------------|----------------------|  
    |1130|Исключено|  
    |1190|Сумма|  
    |1210|Исключено|  
    |1310|Исключено|  

5. Выберите действие **НЗП**, затем выберите действие **Рассчитать НЗП**.  
6. На странице **Расчет НЗП по проектам** выберите проект, для которого нужно рассчитать НЗП. На экспресс-вкладке **Проект** в поле **№** выберите значение **Графика** .  
7. В поле **Дата учета** введите дату позже рабочей даты.
8. В поле **Номер документа** введите **1**. Это создает документ, на который можно в дальнейшем ссылаться для обеспечения возможности трассировки.  
9. Нажмите кнопку **ОК** для запуска пакетного задания. Отобразится сообщение. Нажмите кнопку **ОК**, чтобы продолжить. Закройте страницу **Строки задач проекта**.  

    > [!NOTE]  
    >  Сообщение указывает, что имеются предупреждения, связанные с расчетом НЗП. В следующей процедуре вы просмотрите предупреждения.  

10. На странице **Карточка проекта** разверните экспресс-вкладку **НЗП и призн. доходов и затрат**, чтобы просмотреть вычисленные значения. Можно также просмотреть **Дата учета НЗП** и значения, которые были учтены в главной книге, если таковые были.  

 Обратите внимание, что для поля **Призн. сумма затрат** в столбце **Учесть** выбрано значение 215,60. Оно отражает общую себестоимость двух товаров в группе задач проекта с 1110 по 1130. Для третьего товара выбрано значение **Исключено**, поэтому он не включен в расчет НЗП.  

### Обзор по предупреждениям НЗП  

1.  Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Панель НЗП по проектам**, а затем выберите соответствующую ссылку.  
2.  Выберите проект **Графика**, затем выберите действие **Показать предупреждения**.  
3.  На странице **Предупреждения об НЗП проекта** просмотрите предупреждение, связанное с проектом.  

 После окончания учетного периода Татьяна должна рассчитать повторно незавершенное производство, чтобы включить в расчет завершенную на данный момент работу.  

### Пересчет НЗП  

1. На странице **Карточка проекта** выберите действие **Операции НЗП** для просмотра расчета НЗП.  

     На странице **Операции НЗП проекта** отображаются последние вычисленные для проекта операции незавершенного производства, даже если незавершенное производство еще не учтено в главной книге.  

2. Выполните последовательность действий, объясняющих, как подсчитывать НЗП и пересчитывать НЗП. При каждом расчете НЗП на странице **Операции НЗП проекта** создается операция.  
3. Закрытие страницы.  

> [!NOTE]  
> Незавершенное производство и признание рассчитываются, но не учитываются в главной книге. Для учета нужно выполнить пакетное задание **Учет НЗП в ГК** после расчета НЗП и признания дохода и затрат.

## Учет НЗП в главной книге

 Теперь, когда Татьяна выполнила расчет НЗП для этого проекта, она может учесть его в главной книге.  

### Учет незавершенного производства в главной книге  

1. В списке **Проекты** выберите проект **Графика**.  
2. Выберите действие **НЗП**, затем выберите действие **Учет НЗП в ГК**.  
3. На странице **Учет НЗП по проектам в ГК** на экспресс-вкладке **Проекты** выберите в поле **№** значение **Графика** .  
4. На экспресс-вкладке **Параметры** в поле **Номер сторнированного документа** введите **1**.  
5. Нажмите кнопку **ОК** для учета НЗП в Главной книге.  
6. Нажмите кнопку **ОК**, чтобы закрыть страницу подтверждения.  

     После завершения учета можно просмотреть учтенную информацию на странице **Операции НЗП в ГК**.  

7.  В списке **Проекты** выберите проект **Графика**, затем выберите действие **Операции НЗП в ГК**.  

     На странице **Операции ГК по НЗП проекта** убедитесь, что незавершенное производство учтено в главной книге.  

8. Закрытие страницы.  
9. Откройте страницу **Карточка проекта** для проекта **Графика**.  
10. На экспресс-вкладке **НЗП и признание доходов и затрат** обратите внимание, что в столбце **Учтено** поле **Сумма признанных затрат ГК** теперь заполнена, следовательно НЗП учтено в Главной книге.  
11. Нажмите кнопку **ОК**, чтобы закрыть карточку.  

## Сторнирование учета НЗП

 Татьяна выясняет, что задачи проекта, исключенные из расчета НЗП, на самом деле должны быть засчитаны в НЗП. Татьяна может сторнировать неправильный учет, без необходимости принимать к учету новые данные по незавершенному производству.  

### Процедура сторнирования учета незавершенного производства  

1. В списке **Проекты** выберите проект **Графика**.  
2. Выберите действие **НЗП**, затем выберите действие **Учет НЗП в ГК**.  
3. На странице **Учет НЗП по проектам в ГК** на экспресс-вкладке **Проекты** выберите в поле **№** значение **Графика** .  
4. На экспресс-вкладке **Параметры** в поле **Номер сторнированного документа** введите **1**.  
5. В поле **Дата учета сторнирования** введите исходную дату учета. Это должна быть та же дата, которая использована, чтобы вычислить НЗП в первый раз.  
6. Установите флажок **Только сторнировать**. Это приведет к сторнированию учтенного ранее НЗП без учета нового НЗП в главной книге.  
7. Нажмите кнопку **ОК**, чтобы запустить пакетное задание, а затем нажмите кнопку **ОК**, чтобы закрыть страницу подтверждения.  
8. Откройте страницу **Карточка проекта** для проекта **Графика**.  
9. На экспресс-вкладке **НЗП и признание доходов и затрат** убедитесь, что нет учтенных операций НЗП.  
10. Закройте эту страницу.  
11. В списке **Проекты** выберите работу **Графика**, выберите действие **НЗП**, затем выберите действие **Операции НЗП в ГК**. Для операций НЗП установлен флажок **Сторнировано**.  
12. Закройте эту страницу.  
13. Откройте страницу **Строки задач проекта** для проекта, включите части проекта, которые должны фигурировать в расчете НЗП, а затем выполните пересчет и учет нового значения в главной книге.  

    > [!NOTE]  
    >  Предположим, Татьяна рассчитала и учла незавершенное производство для проекта с неправильными датами. Придерживаясь описанного ранее метода, Татьяна может сторнировать неправильные записи учета, исправить даты и повторно произвести учет в главной книге.  

## Дальнейшие шаги

 В этом пошаговом руководстве были рассмотрены шаги, которые необходимо выполнить для вычисления незавершенного производства в [!INCLUDE[prod_short](includes/prod_short.md)]. В больших проектах иногда удобно периодически переносить затраты на счет НЗП по мере выполнения проекта. В этом пошаговом руководстве показано, как исключать строки задания из вычисления. Здесь также указано, в каких случаях может потребоваться пересчет. И наконец, в этом пошаговом руководстве демонстрируется процесс учета незавершенного производства в Главной книге. Также включен пример процедуры сторнирования учета НЗП в Главной книге.
