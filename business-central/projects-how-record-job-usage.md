# <a name="record-consumption-or-usage-for-projects"></a>Регистрация потребления или использования в проектах

Со страницы **Карточка проекта** можно открыть страницу **Строки планирования проекта**, чтобы просмотреть и зарегистрировать потребление в различных частях проекта. Эта информация автоматически обновляется при изменении и переносе информации между проектами и журналами проектов или счетами по проектам. Для этого требуется включить переключатель **Применить связь использования по умолчанию** на странице **Настройка проектов**. Подробнее см. в статье [Настройка проектов](projects-how-setup-jobs.md).  

Например, для строк планирования с типом **Бюджет** можно ввести количество ресурса и указать, какое количество должно быть перенесено в журнал проектов. Если тип строки планирования имеет значение **К оплате**, можно ввести количество ресурса и указать, какое количество переместить в счет. Подробнее о выставлении счета клиенту см. в статье [Выставление счетов по проектам](projects-how-invoice-jobs.md). Сравнивая исходное количество, оставшееся количество или учтенное количество, вы можете быстро просмотреть информацию об использовании. Подробнее об оценке бюджетных значений при планировании см. в статье [Управление бюджетами проектов](projects-how-manage-budgets.md).  

В следующих процедурах описано, как регистрировать фактические (бюджетированные) количества и затраты с помощью журнала проектов. В качестве альтернативы можно использовать документы покупки для учета покупок для проекта. Подробнее см. в статье [Управление запасами для проекта](projects-how-manage-project-supplies.md).

## <a name="to-record-usage-for-a-project-planning-line-of-type-budget"></a>Регистрация потребления для строки планирования проекта типа «Бюджет»

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, затем выберите соответствующую ссылку.  
2. Выберите проект, а затем выберите действие **Строки планирования проекта**. 
3. Выберите строку планирования проекта типа **Бюджет** или **Бюджет и к оплате**, для которой вы хотите зарегистрировать потребление.   

    > [!NOTE]
    > Также можно регистрировать потребление для строки планирования проекта типа **К оплате**. Как правило, строки используются для создания счетов, но также можно перемещение информации в журнал. Подробнее см. в статье [Выставление счетов по проектам](projects-how-invoice-jobs.md) 

4. В поле **Количество для переноса в журнал** введите число для перемещения. По умолчанию принимается количество, введенное в поле **Количество**.

    В поле **Остаток** отображается количество, оставшееся для завершения проекта и переноса в журнал.
5. Выберите действие **Создать строки журнала проектов**.

    > [!TIP]
    > Если вы собираетесь добавить больше строк планирования проекта для этого проекта, подождите с этим шагом, пока не добавите все строки планирования проекта.
6. На странице **Перенос из проекта - строка планирования проекта** заполните поля по мере необходимости, затем нажмите кнопку **ОК**. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]
7. Выберите действие **Открыть журнал проектов**.  
8. На странице **Журнал проектов** выберите соответствующую строку, затем выберите действие **Учет**.

    [!INCLUDE [preview-posting-inventory](includes/preview-posting-inventory.md)]

9. На странице **Строки планирования проекта** просмотрите зарегистрированное потребление, проверив поля **Количество**, **Остаток** и **Количество для переноса в журнал**.  
10. Повторите шаги с 3 по 8, чтобы зарегистрировать дополнительное потребление.  

## <a name="to-create-project-journal-lines-manually"></a>Создание строк журнала проектов вручную

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Журналы проектов**, а затем выберите соответствующую ссылку.  
2. В поле **Код раздела** выберите соответствующий раздел журнала проектов.  
3. В новой строке введите номер документа, номер проекта, номер задачи проекта, тип и количество типа, который потребляется.  
4. Когда строки журнала проектов будут заполнены, выберите действие **Учет**.  

    [!INCLUDE [preview-posting-inventory](includes/preview-posting-inventory.md)]

## <a name="to-view-project-usage-estimates-and-post-updates"></a>Просмотр оценок потребления по проекту и учет обновлений

Для просмотра потребления по проекту вплоть до завершения проекта достаточно одного действия: воспользоваться пакетным заданием **Расчет оставшегося потребления по проекту** для всех задач до завершения проекта включительно.  

Это позволяет отследить и сравнить первоначальные оценки с фактическими результатами, и при необходимости внести изменения или создать новые операции. Пусть, например, расчеты показали, что для выполнения проекта необходимо 10 часов, но на сегодняшний день оно заняло 15 часов. Программа позволяет добавить дополнительные 5 часов в существующей строке журнала или создать новую строку для записи этих дополнительных 5 часов как сверхурочных, что соответствует другому типу работы. Себестоимость и цена рассчитаны правильно и их можно учесть в журнале.  

> [!NOTE]  
> Операции товара приводят к созданию операций журнала товаров и уменьшению количества запасов. Пакетное задание **Учет себест. запасов в ГК** переносит себестоимость из книги запасов в Главную книгу. Операции ресурсов приводят к созданию операций книги ресурсов.  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Журналы проектов**, а затем выберите соответствующую ссылку.  
2. Выберите соответствующий журнал проектов, затем выберите действие **Расчет остаточного потребления**.  
3. На странице **РРасчет оставшегося потребления по проекту** введите номер документа и дату учета, которые следует вставить в журнал, затем выберите кнопку **ОК**.  
4. Обновите журнал, внеся в него все изменения, которые могут требоваться.  
5. Выберите **Учесть**.

## <a name="create-inventory-and-warehouse-pick-documents-for-a-project"></a>Создание документов товарного и складского подбора для проекта

Используйте действия **Создать товарный подбор** и **Создать складской подбор** на странице **Карточка проекта**. Чтобы создать или зарегистрировать документ подбора, используйте действия **Строки перемещения/подбора/размещения** или **Строки зарегистрированного подбора**. Подробнее см. в статье [Потоки для производства, сборки и проектов](design-details-internal-warehouse-flows.md).

Вы можете использовать действия при следующих условиях:

* **Статус** проекта — **Открыто**.
* **Тип строки** строки планирования проекта — **Бюджет** или **Бюджет и к оплате**.
* **Тип** строки планирования проекта — **Товар**.
* **Требуется подбор** включено для связанного склада.
* **Расширенный подбор и размещение** отключено.

> [!NOTE] 
> Хотя параметр называется **Требуется подбор**, вы по-прежнему можете учитывать потребление непосредственно из строки журнала проектов для склада. Если склад настроен так, что требуется обработка подбора, но не требуется обработка отгрузки, то для организации и печати информации по подбору следует использовать страницу **Подбор запасов**. Вы также используете страницу для ввода и учета результатов подбора, которые, в свою очередь, учитывают потребление товаров. 
> 
> Если склад настроен так, что требуется обработка как подбора, так и отгрузки, то есть выбраны поля **Требуется подбор** и **Требуется отгрузка** на странице **Карточка склада**, для обработки подбора используется страница **Складской подбор**. Складские подборы аналогичны подборам запасов. Разница в том, что вместо того, чтобы учитывать информацию о подборе, вы регистрируете подбор. Эта регистрация не учитывает потребление, она просто делает товары доступными для учета. Как менеджер склада, вы можете использовать журнал подбора для упорядочивания информации о подборе перед созданием отдельных складских инструкций по подбору.

## <a name="to-review-planning-lines-for-a-project-ledger-entry"></a>Проверка строк планирования для операции книги проектов

После учета строк журнала проектов вы можете просмотреть строки планирования, связанные с записями журнала проектов, которые были учтены.

> [!NOTE]  
> Для этого необходимо, чтобы для проекта был установлен флажок **Применить связь использования**. Подробнее см. в статье [Настройка проектов](projects-how-setup-jobs.md).  

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Журналы проектов**, а затем выберите соответствующую ссылку.  
2. Выберите соответствующий журнал проектов, затем выберите действие **Книга операций**.  
3. На странице **Операции книги проектов** выберите действие **Показать связанные строки планирования проектов**.
