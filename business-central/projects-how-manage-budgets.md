# <a name="manage-project-budgets"></a>Управление бюджетами проектов

Для каждого проекта можно настроить бюджет. Бюджет необходим для планирования ресурсов, выделяемых для выполнения проекта. Бюджет может быть общим, с небольшим количеством операций, или же он может содержать больше операций, которые разделены по уровням деятельности. Затем можно сравнивать бюджетные суммы с фактическим потреблением, регистрируемым в журнале проектов. Мониторинг разницы между фактическим и бюджетным потреблением позволяет контролировать текущий проект и улучшить качество будущих проектов за счет снижения риска преуменьшения затрат.

Следующая процедура показывает, как оценить бюджетные затраты во время планирования. Информацию о регистрации бюджетных и фактических цен и затрат для проекта см. в статье [Регистрация потребления для проектов](projects-how-record-job-usage.md).  

## <a name="to-estimate-the-budgeted-costs-for-a-project"></a><a name="JobBudgetCosts"></a>Оценка бюджетных затрат для проекта

Когда клиент хочет знать цену проекта, счета по которому будут выставляться на основе потребления, необходимо определить бюджетные затраты на проект. Для этого используется страница **Строки задач проекта**.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, затем выберите соответствующую ссылку.  
2. Откройте соответствующий проект.
3. Выделите строку задачи типа «Учет», затем выберите действие **Строки планирования проекта**.
4. На новой строке заполните поля, как требуется. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

Для поля **Тип строки** см. следующую информацию.  

| Тип строки | Описание |
| --- | --- |
| **Бюджет и к оплате** |Суммы затрат и цен, введенные в строку планирования, представляют собой бюджетные затраты для данной строки планирования. По этой цене будет выставлен счет. |
| **Бюджет** |Клиенту не оплачивает потребление. Потребление не переносится в счет, а используется для расчета незавершенного производства. |
| **К оплате** |Клиенту оплачивает потребление. Потребление переносится в счет на основании количества, указанного в поле **Переносимое в счет кол-во**. |

> [!NOTE]  
> Поле **Плановая дата доставки** для строки планирования содержит дата, когда ожидается завершение потребления, связанного со строкой планирования. Это также дата, когда строка планирования может быть перенесена в счет по продаже и учтена. <br /><br /> В базовой задаче проекта на странице **Карточка проекта** поля **Дата начала** и **Дата окончания** содержат, соответственно, значение поля **Плановая дата доставки** на самой ранней и самой поздней строках планирования проекта на связанной странице **Строки планирования проекта**.

> [!NOTE]  
> При заполнении поля **Количество** вся информация об общих ценах и затратах будет рассчитана и заполнена для данной строки планирования. Ее можно изменить в любое время.

На странице **Карточка проекта** можно увидеть общие бюджетные затраты, бюджетную цену, оплачиваемую себестоимость и оплачиваемую цену для каждой задачи.

Информацию о регистрации бюджетных и фактических цен и затрат для проекта см. в статье [Регистрация потребления для проектов](projects-how-record-job-usage.md).
