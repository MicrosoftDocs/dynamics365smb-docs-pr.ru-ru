# Пошаговое руководство. Получение и складирование в базовых конфигурациях склада

В [!INCLUDE[prod_short](includes/prod_short.md)] получение и размещение происходит с использованием одного из четырех методов, как описано в следующей таблице.

|Метод|Входящий процесс|Требуется приемка|Требуется размещение|Уровень сложности (подробнее см. в разделе [Обзор Warehouse Management](design-details-warehouse-management.md))|  
|------------|---------------------|--------------|----------------|------------|  
|А|Выполните учет приемки и размещения из строки заказа|||Нет отдельной складской операции.|  
|Б|Выполните учет приемки и размещения из документа складского размещения||Включено|База: последовательно по каждому заказу.|  
|Т|Выполните учет приемки и размещения из документа складской приемки|Включено||База: объединенная проводка получения/отправки для нескольких заказов.|  
|Д|Выполните учет приемки из документа складской приемки и учет размещения из документа складского размещения|Включено|Включено|Дополнительно|  

Подробнее см. в разделе [Исходящий складской поток](design-details-inbound-warehouse-flow.md).

В следующем пошаговом руководстве показан метод B в предыдущей таблице.  

## Об этом пошаговом руководстве  

В базовых конфигурациях склада, если настройки склада требуют обработки размещения, но не требуют обработки приемки, то для регистрации и учета информации о размещении и приемке для входящих исходных документов следует использовать страницу **Размещение запасов**. Следующие документы являются входящими исходными документами:

* Заказ на покупку
* Возврат продажи
* Заказ на входящее перемещение
* Производственный заказ с выходом, который готов к размещению

> [!NOTE]
> Хотя эти параметры называются **Требуется подбор** и **Требуется размещение**, все равно можно разносить приемки и отгрузки непосредственно из исходных бизнес-документов на складах, для которых установлены эти флажки.  

В этом пошаговом руководстве демонстрируются следующие задачи:  

* Настройка склада SILVER для размещений запасов.  
* Настройка склада SILVER для обработки ячейки.  
* Определение стандартной ячейки для товара LS-81. (LS-75 уже настроен в CRONUS.)  
* Создать заказ на покупку для поставщика 10000 на 40 громкоговорителей.  
* Проверить соответствия ячеек складирования настройкам.  
* Выпуск заказа на покупку для обработки на складе.  
* Создание складского размещения на основании выпущенного исходного документа.  
* Удостоверьтесь, что ячейки складирования наследуются из заказа на покупку.  
* Регистрация складского перемещения в склад с учетом приходной накладной покупки для исходного заказа на покупку.

## Роли  

Следующие роли пользователей выполняют задачи, которые демонстрирует это пошаговое руководство.  

* Заведующий складом  
* Агент по закупкам  
* Работник склада  

## Предварительные требования  

Для выполнения данного пошагового руководства необходимо выполнить следующие действия:  

* Данные CRONUS International Ltd.  
* Быть сотрудником склада SILVER. Для настройки выполните следующие шаги:  

    1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Работники склада**, а затем выберите связанную ссылку.  
    2. Выберите поле **ИД пользователя** и укажите свою учетную запись пользователя на странице **Пользователи**.  
    3. В поле **Код склада** введите **SILVER**.  
    4. Установите флажок **По умолч.**.  

## Сюжет  

Элен, менеджер склада в CRONUS Россия ЗАО, создает заказ на покупку на 10 единиц товара LS-75 и 30 единиц товара LS-81 от поставщика 10000, которые должны быть доставлены на СЕРЕБРЯНЫЙ склад. Когда товар будет доставлен на склад, работник склада Вадим убирает товары в стандартные ячейки, обозначенные для товаров. Когда Вадим учитывает размещение, товары учитываются в запасах как полученные товары и доступны для продажи или других потребностей.  

## Настройка склада  

Настройка страницы **Карточка склада** определяет потоки склада организации.  

### Настройка склада  

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Склады**, а затем выберите связанную ссылку.  
2. Откройте карточку склада СЕРЕБРЯНЫЙ.  
3. Включите переключатель **Требуется размещение**.  

    Настройте стандартной ячейки для двух кодов товара с целью контроля, где они размещаются.  

4. Выберите действие **Ячейки**.  
5. Выберите первую строку, для ячейки **S-01-0001**, а затем выберите действие **Содержимое**.  

    Обратите внимание, что на странице **содержимое ячейки** товар **LS-75** уже настроен в качестве содержимого в ячейке S-01-0001.  

6. Выберите действие **Создать**.  
7. Выберите поля **Фиксир.** и **По умолчанию**.  
8. В поле **Номер товара** введите **LS-81**.  

## Создайте заказ на покупку  

Заказы на покупку — это наиболее распространенный тип входящего приходного документа.  

### Порядок создания заказа на покупку  

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Заказы на покупку**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  
3. Создайте заказ на покупку для поставщика 10000 на рабочую дату (23 января) со следующими строками заказа на покупку.  

    |Товар|Код склада|Код ячейки|Количество|  
    |----------|-------------------|--------------|--------------|  
    |LS_75|СЕРЕБР.|С-01-0001|10|  
    |LS-81|СЕРЕБР.|С-01-0001|30|  

    > [!NOTE]  
    > Код ячейки вводится автоматически в соответствии с настройками в разделе [Настройка склада](#setting-up-the-location).  

    Далее, необходимо уведомить склад, что заказ на покупку готов для обработки на складе при поступлении товара.  

4. Выберите действие **Выпустить**.  

    Громкоговорители были доставлены от поставщика 10000 на СЕРЕБРЯНЫЙ склад, и Вадим начинает их размещать.  

## Приемка и размещение товаров  

На странице **Размещение запасов** можно управлять всеми входящими складскими операциями для конкретного документа-источника, такого как заказ на покупку.  

### Приемка и размещение товаров  

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Складские размещения**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  
3. Выберите поле **Документ-источник**, а затем выберите **Заказ на покупку**.  
4. Выберите поле **Номер источника**, выберите строку для покупки у поставщика 10000, а затем нажмите кнопку **ОК**.  

    В качестве альтернативы выберите действие **Получить источник документа**, затем выберите заказ на покупку.  

5. Выберите действие **Автозаполнение кол-ва для обработки**.  

    В качестве альтернативы в поле **Кол-во для обработки** введите 10 и 30 соответственно в двух строках складского размещения.  

6. Выберите действие **Учет**, выберите параметр **Получить** и нажмите кнопку **ОК**.  

    Теперь 40 громкоговорителей зарегистрированы как размещенные в ячейке С-01-0001, и создана положительная операция товарной книги, отражающая учтенную приходную накладную покупки.
