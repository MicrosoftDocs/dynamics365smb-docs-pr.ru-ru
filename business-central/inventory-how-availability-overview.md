# <a name="view-the-availability-of-items"></a>Просмотр наличия товара
В контексте бизнес-задачи можно получить расширенную информацию о времени и месте приобретения товара, например при разговоре с клиентом о дате доставки.

Можно просмотреть наличие всех товаров на складе, а также просмотреть наличие каждого товара по событию, периоду или складу. Событие — это любая запланированная товарная транзакция, например расходная накладная или входящая накладная на перемещение.

> [!NOTE]  
>   Для просмотра наличия по складу требуется вести товары на нескольких складах. Дополнительные сведения см. в разделе [Настройка складов](inventory-how-setup-locations.md).

Если используется функция склада, наличие отличается в зависимости от распределений на уровне ячеек, если выполняются складские операции, такие как подбор и перемещение, и если система резервирования налагает ограничения, которые следует соблюдать. Довольно сложный алгоритм проверяет выполнение всех условий перед распределением количеств по подборам для расходов товаров. Дополнительные сведения см. в разделе [Сведения о проектировании: наличие на складе](design-details-availability-in-the-warehouse.md).

В разделе [!INCLUDE[d365fin](includes/d365fin_md.md)] значения доступности обычно отображаются в двух различных полях с разными определениями:

* В поле **Количество в наличии**, которое в некоторых местах называется **Запасы**, отображается фактическое количество на текущую дату в учтенных операциях книги товаров.
* Рассчитывается значение в поле **Прогнозное доступное сальдо**, и в нем отображается количество в наличии плюс запланированные поступления минус полные потребности. (В разделе [!INCLUDE[d365fin](includes/d365fin_md.md)] запланированные поступления включают количества в заказах на покупку и входящих заказах на перемещение. Полная потребность включает количества в заказах на продажу и исходящих заказах на перемещение.)

> [!TIP]  
>   Прогнозное доступное сальдо особенно релевантно на страницах **Наличие товара по периодам** и **Наличие товара по событиям**, поскольку они содержат измерение даты.  

> [!NOTE]  
>   В следующих процедурах описано, как просмотреть расширенную информацию о наличии в списках товаров и карточке товара. Доступ к этой информации также можно получить из строк документа продажи для товара в строке. Дополнительные сведения см. в разделе [Продажа продукции](sales-how-sell-products.md).

## <a name="to-view-the-availability-of-an-item-according-to-when-it-will-be-received-or-shipped"></a>Просмотр наличия товара в соответствии с временем его получения или отгрузки
Наличие товара можно просмотреть в соответствии с запланированными товарными транзакциям на странице **Наличие по событиям**.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Товары**, затем выберите соответствующую ссылку.
2. Откройте карточку товара, наличие которого требуется просмотреть.
3. Выберите действие **Наличие товара по** и выберите действие **Событие**.

    На странице **Наличие товара по событиям** отображается, как количество запасов товара будет изменяться с течением времени согласно запланированным событиям отгрузки и получения. Эта страница обеспечивает возможность сжатого просмотра, где показана одна строка накопленной информации за временной период, в котором количества запасов меняются. Временные интервалы, в течение которых не было событий, не отображаются. Можно раскрыть каждую строку для отображения сведений о событии или событиях, которые привели к накопленному количеству в строке.
4. Выберите значение в поле **Прогнозное доступное сальдо** для просмотра операций книги товаров или открытых документов, которые составляют значение.

## <a name="to-view-the-availability-of-an-item-in-different-periods"></a>Просмотр наличия товара за различные периоды
Наличие товара можно просмотреть с течением времени за определенные периоды времени на странице **Наличие товара по периодам**.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Товары**, затем выберите соответствующую ссылку.
2. Откройте карточку товара, наличие которого требуется просмотреть.
3. Выберите действие **Наличие товара по** и выберите действие **Период**.

    На странице **Наличие товара по периодам** отображается как, количество запасов товара будет меняться с течением времени за выбранный период, например день, неделю или квартал.
4. Выберите значение в поле **Прогнозное доступное сальдо** для просмотра операций книги товаров или открытых документов, которые составляют значение.

## <a name="to-view-the-availability-of-an-item-at-the-locations-where-it-is-stored"></a>Просмотр наличия товара на складе, на котором он хранится
Наличие товара можно просмотреть в разных местах, в которых он хранится, на странице **Наличие товара по складам**.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Товары**, затем выберите соответствующую ссылку.
2. Откройте карточку товара, наличие которого требуется просмотреть.
3. Выберите действие **Наличие товара по** и выберите действие **Склад**.

    На странице **Наличие товара по складам** отображается, как количество запасов товара будет меняться в будущем для каждого склада, на котором он хранится.
4. Выберите значение в поле **Кол-во в наличии** для просмотра операций книги товаров, которые составляют значение.
5. Выберите значение в поле **Прогнозное доступное сальдо** для просмотра операций книги товаров или открытых документов, которые составляют значение.

## <a name="to-view-the-availability-of-all-items-by-the-location-where-they-are-stored"></a>Просмотр наличия всех товаров по складу, на котором они хранятся
Наличие товара можно просмотреть по всем складам на странице **Товары по складам**.

1. Выберите значок ![Лампочка, которая открывает функцию "Что вы хотите сделать"](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Товары**, затем выберите соответствующую ссылку.
2. Выберите действие **Товары по складам**.

    На странице **Товары по складам** отображается, как количество доступного товара на каждом складе для всех товаров.
3. Выберите значение в поле **Кол-во в наличии** для просмотра операций книги товаров, которые составляют значение.

## <a name="to-view-the-availability-of-an-item-by-its-use-in-assembly-or-production-boms"></a>Просмотр доступности товара по использованию в сборочных или производственных спецификациях
Если товар входит в сборочные или производственные спецификации как родительский элемент или как компонент, вы можете увидеть, сколько единиц этого товара требуется, на странице **Доступность товаров по уровням спецификации**. На этой странице отображается, какое количество родительского элемента можно сделать на основании доступности дочерних элементов в строках. Товар, имеющий сборочную или производственную спецификацию, отображается на странице как разворачиваемая строка. Можно раскрыть строку для просмотра компонентов и сборочных узлов низшего уровня с их собственными спецификациями.

Например, эта страница позволяет выяснить, можно ли выполнить заказ на продажу товара в указанный срок. Для этого просмотрите текущее наличие товара и потенциальное количество, которое можно изготовить из его компонентов. Данную страницу можно использовать также для определения соответствующих узких мест в спецификациях.

В каждой строке на странице для родительских и дочерних товаров следующие ключевые поля определяют показатели наличия. Эти цифры можно использовать, чтобы узнать, сколько единиц родительской продукции вы можете предоставить, если начнете соответствующий процесс сборки.

|Поле|Описание|
|------|-----------|
|**Можно сделать родительским объектом**|Отображает, сколько можно изготовить единиц любых сборочных узлов в товаре высшего уровня. Поле определяет количество родительских единиц, которые можно сразу собрать. Это значение основано на наличии товара в строке.|
|**Можно сделать товаром верхнего уровня**|Отображает, сколько можно изготовить единиц товара высшего уровня. Поле определяет количество единиц товара верхнего уровня в спецификации, которые можно собрать. Это значение основано на наличии товара в строке.|

### <a name="to-view-the-availability-of-an-item-according-to-demand-for-its-parent"></a>Для просмотра наличия товара по требованию его родителя
На странице **Доступность товаров по уровням спецификации** отображается информация для товара в строке карточки или документа, для которого открыта страница. Товар всегда показывается в верхней строке. Можно просмотреть сведения для других товаров или для всех товаров путем изменения значения в поле **Фильтр по товару**.

> [!NOTE]  
>   По умолчанию показатели наличия в строках отображают общее наличие всех товаров под товаром верхнего уровня. Эти цифры отображаются в поле **Доступное количество**, а выделен будет товар верхнего уровня. Информация о количестве сборочных узлов может быть уклонена. Для получения реального указания на то, сколько отображаемых сборочных узлов можно включить в производство, необходимо снять флажок **Показать общую доступность** и просмотреть значение в поле **Можно сделать родительским объектом**.

Поле **Узкое место** определяет, какой товар в структуре спецификации не позволяет произвести количество, большее чем количество, указанное в поле **Можно сделать товаром верхнего уровня**. Например, "узким" товаром может быть покупаемый компонент, из-за поздней ожидаемой даты приемки которого невозможно создать дополнительные единицы товара верхнего уровня к сроку, указанному в поле **Требуется на дату**.

## <a name="to-view-the-availability-of-an-item-by-its-units-of-measure"></a>Для просмотра наличия товара по его единицам измерения
На странице **Наличие товара по единице измерения** показано наличие товара в разных единицах измерения, в которых он хранится.

> [!NOTE]  
> Чтобы эта информация оставалась точной, необходимо преобразовывать единицы измерения товара. Например, если вы покупаете товар в одной единице измерения, такой как коробки, а продаете в другой единице измерения, такой как штуки, вы должны использовать журнал товаров для преобразования единицы измерения, т. е. перевода коробок в штуки. Вы можете использовать отрицательную корректировочную строку журнала товаров, чтобы уменьшить запасы в единице измерения покупки (например, коробках), и положительную корректировочную строку, чтобы увеличить запасы в единице измерения продажи (например, штуках). 

## <a name="assembly-availability-page"></a>Страница "Доступность сборки"
Страница **Доступность сборки** отображает подробную информацию о доступности для сборочного элемента. Оно открывается:

- Автоматически из строки заказа на продажу в сценариях сборки на заказ после ввода количества, которое порождает проблему доступности компонентов.
- Автоматически из заголовка сборочного заказа после ввода значения в поле Количество, которое порождает проблему доступности компонентов.
- Вручную при открытии из сборочного заказа. На вкладке Действия в группе Функции выберите Показать доступность.

Экспресс-вкладка **Сведения** показывает подробную информацию о наличии для сборочного элемента, включая количество из сборочного заказа, которое быть собрано к сроку оплаты, исходя из наличия требуемых компонентов. Это отображается в поле Способность к сборке на экспресс-вкладке Подробности.

Значение в поле **Способность к сборке** отображается красным шрифтом, если количество меньше количества в поле **Остаток**, что указывает на нехватку компонентов для сборки полного объема.

Экспресс-вкладка **Строки** показывает подробную информацию о наличии сборочных компонентов.

Если один или несколько сборочных компонентов не доступны, это отразится в поле **Способность к сборке** в строке вопроса как количество, которое меньше количества в поле **Остаток** на экспресс-вкладке **Подробности**.
