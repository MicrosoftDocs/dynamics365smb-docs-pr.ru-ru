# <a name="pick-for-production-or-assembly-in-basic-warehouse-configurations"></a>Подбор для производства или сборки в базовых конфигурациях склада
Способ размещения подбора компонентов для заказов на производство или сборку зависит от того, как настроен склад. Дополнительные сведения см. в разделе [Настройка управления складом](warehouse-setup-warehouse.md).


## <a name="pick-for-production-in-basic-warehouse-configurations"></a>Подбор для производства в базовой конфигурации склада
Метод списания также влияет на поток компонентов при производстве. Дополнительные сведения см. в разделе [Списание компонентов в соответствии с производственным выпуском](production-how-to-flush-components-according-to-operation-output.md).

В расширенных конфигурациях склада, где требуется как подбор, так и отгрузка, вы должны использовать страницу **Складской подбор**, чтобы вывести компоненты с установленным методом списания *Вручную*, *Подбор + прямой*, *Подбор + обратный* для производственных заказов. Дополнительные сведения см. в разделе [Подбор для производства или сборки в расширенных конфигурациях склада](warehouse-how-to-pick-for-internal-operations-in-advanced-warehousing.md)

В базовых конфигурациях склада, когда на складе требуется обработка подбора, но не требуется обработка отгрузки, для организации и регистрации подбора компонентов также может использоваться страница **Подбор запасов** с методом списания, заданным на *Вручную*. При регистрации подбора запасов для внутренней операции, например производства, потребление выбранных компонентов учитывается в тот же момент. В качестве альтернативы вы можете использовать страницу **Перемещение запасов** со ссылкой на исходный документ, чтобы вывести компоненты с установленным методом списания *Вручную*, *Подбор + прямой*, *Подбор + обратный* для производственных заказов.

Когда производственные операции интегрированы со складскими процессами по ячейкам или путем непосредственного размещения и подборов, ячейка, из которой расходуются компоненты, является ячейкой, которая определена в каждой строке компонента производственного заказа. Все требуемые компоненты должны иметься в этой ячейке. В противном случае учет использованных вручную или при списании товаров для этого компонента останавливается.

**Перемещение запасов** со ссылками на исходный документ и **Складской подбор** нельзя использовать для отбора компонентов с помощью методов списания *Прямой* и *Обратный*. **Складской подбор** нельзя использовать для отбора компонентов с помощью любого метода списания, кроме *Вручную*. Для обработки оставшихся компонентов используйте **Перемещение запасов** без ссылки на исходный документ. Дополнительные сведения см. в разделе [Перемещение компонентов в производственную зону в основных конфигурациях склада](warehouse-how-to-move-components-to-an-operation-area-in-basic-warehousing.md).

> [!NOTE]  
>  Подборы запасов, перемещения запасов и складские подборы имеют описанные ниже важные различия:  
>   
>  -   При регистрации подбора запасов для внутренней операции, например производства, потребление выбранных компонентов учитывается в тот же момент. При регистрации перемещения запасов или складского подбора для внутренней операции записывается только физическое перемещение требуемых компонентов в ячейку в области операции без учета потребления.  
> -   Когда используются складские подборы, поле **Код ячейки** в строке компонента производственного заказа определяет ячейку *взять*, откуда вычитаются компоненты при учете потребления. При использовании перемещения запасов или складского подбора поле **Код ячейки** в строках компонентов производственного заказа определяет *пробел* ячейку в области операции, где работник склада должен поместить компоненты.  

Предварительное условие системы для подбора или перемещения компонентов для документов-источников заключается в том, чтобы существовал исходящий запрос склада для уведомления зоны склада о потребности в компоненте. Исходящий складской запрос создается каждый раз, когда статус производственного заказа меняется на "Выпущен" или когда создается выпущенный производственный заказ.  

## <a name="to-pick-production-components-in-basic-warehouse-configurations-using-inventory-pick"></a>Для подбора производственных компонентов в базовых конфигурациях склада с помощью подбора запасов
В базовых конфигурациях склада, где склад настроен на использование только подбора, можно выбрать компоненты для производственных действий на странице **Подбор запасов**. Дополнительные сведения см. в разделе [Подбор товаров с помощью подбора запасов](warehouse-how-to-pick-items-with-inventory-picks.md).

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Подбор запасов**, а затем выберите связанную ссылку.  
2.  Чтобы получить доступ к компонентам производственного заказа, выберите действие **Получить исходные документы**, затем выберите запущенный производственный заказ.  
3.  Выполните подбор, затем запишите текущую информацию о подборе в поле **Кол-во для обработки**.  
4.  Когда строки будут готовы для учета, выберите действие **Учет**. Во время процедуры учета создаются необходимые складские операции и учитывается потребление товаров.  

Можно также создать **Товарный подбор** непосредственно из запущенного производственного заказа. Выберите действие **Создать размещение/подбор/перемещение запасов**, установите флажок **Создать товарный подбор**, затем выберите кнопку **ОК**.

В качестве альтернативы вы можете использовать **Перемещение запасов** со ссылкой на исходный документ для перемещения товаров между ячейками. Потребление нужно будет регистрировать отдельно. Дополнительные сведения см. разделе [Учет потребления для производства в пакетном режиме](production-how-to-post-consumption.md)

## <a name="pick-for-assembly-in-basic-warehouse-configurations"></a>Подбор для сборки в базовой конфигурации склада
В расширенных настройках складов, в котором требуется подбор и отгрузки, необходимо использовать страницу **Складской подбор**, чтобы привести компоненты к заказам на сборку. Дополнительные сведения см. в разделе [Подбор для производства или сборки в расширенных конфигурациях склада](warehouse-how-to-pick-for-internal-operations-in-advanced-warehousing.md)

В рамках базовой конфигурации склада можно также делать подбор для заказов на сборку с помощью страницы **Перемещение запасов**. 

В базовых конфигурациях склада, где для местоположения требуется обработка комплектования, но не обработка отгрузки, страница **Подбор запасов** также используется для подбора, сборки и отправки заказа на продажу, где товары должны быть собраны перед отправкой. Для получения дополнительных сведений см. [Обработка собираемого на заказ товара с помощью подбора запасов](warehouse-how-to-pick-for-production.md#handling-assemble-to-order-items-with-inventory-picks).  

## <a name="handling-assemble-to-order-items-with-inventory-picks"></a>Обработка товаров сборки на заказ со складскими подборками
Страница **Подбор запасов** также используется для подбора и отгрузки для продажи, когда товары необходимо собрать до их отгрузки. Дополнительные сведения см. в разделе [Продажа товара, собранного на заказ](assembly-how-to-sell-items-assembled-to-order.md).

Товары, которые должны быть поставлены, физически не присутствуют в ячейке до тех пор, пока они не будут собраны и учтены как исходящие в ячейке в области сборки. Это означает, что подбор товаров сборки на заказ для отгрузки выполняется согласно специальной процедуре. Работники склада берут из ячейки сборочные элементы для склада отгружаемых товаров, а затем учитываются складской подбор. Учтенный складской подбор затем учитывает производительность сборки, потребление компонентов и отгрузки продаж.

Можно настроить [!INCLUDE[prod_short](includes/prod_short.md)] для автоматического создания перемещения запасов при создании складского подбора для сборочного элемента. Чтобы включить это, необходимо выбрать поле **Создавать перемещения автоматически** на странице **Настройка сборки**. Дополнительные сведения см. в разделе [Перемещение компонентов в производственную зону на основном складе](warehouse-how-to-move-components-to-an-operation-area-in-basic-warehousing.md).

Строки складского подбора для продаваемых товаров создаются разными способами в зависимости от того, какое количество строк продажи собирается на заказ.

Обычные продажи, в которых используются складские подборы, чтобы учесть отгрузку количества запасов, для каждой строки заказа на продажу создается одна строка складского подбора или несколько строк, если товар помещен в различные ячейки. Эта строка подбора основана на количестве в поле **Кол-во для отгрузки**.

В документах продаж сборки на заказ, где все количество в строке заказа на продажу собрано на заказ, создается одна строка складского подбора для этого количества товара. Это означает, что значение "Количество для сборки" совпадает со значение в поле **Кол-во для отгрузки**. В строке выбрано поле **Сборка на заказ**.

Если выход сборки настроен для данного склада, то значение в поле **Код ячейки сборки на заказ** либо значение в поле **Код исходящей сборочн. ячейки** в этом заказе вставляется в поле **Код ячейки** в строке складского подбора.

Если в строке заказа на продажу не указан код ячейки и не настроен поток сборки для склада, поле **Код ячейки** строки складского подбора останется пустым. Работник склада должен открыть страницу **Содержимое ячейки** и выбрать ячейку, где собираются сборочные элементы.

В составных сценариях, согласно которым часть количества нужно сначала собрать, а другую отобрать из запасов, создаются минимум две строки подбора запасов. Одна строка складского подбора предназначена для количества сборки на заказ. Другая строка подбора зависит от того, какие ячейки могут пополнить остаток со склада. Коды ячеек в двух строках заполняются по-разному, как описано соответственно для двух разных типов продажи. Дополнительные сведения см. в подразделе "Сценарии комбинации" в разделе [Сборка на заказ и сборка на склад](assembly-assemble-to-order-or-assemble-to-stock.md).
