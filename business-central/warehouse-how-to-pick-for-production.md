# <a name="pick-for-production-assembly-or-jobs-in-basic-warehouse-configurations"></a>Подбор для производства, сборки или заданий в базовых конфигурациях склада

Способ размещения подбора компонентов для заказов на производство или сборку зависит от того, как настроен склад. Дополнительные сведения см. в разделе [Настройка управления складом](warehouse-setup-warehouse.md).

## <a name="pick-for-production-in-basic-warehouse-configurations"></a>Подбор для производства в базовой конфигурации склада

Если на складе требуется обрабатывать подбор, но не нужно обрабатывать отгрузку, можно использовать страницу **Подбор запасов**. Страница **Подбор запасов** позволяет вам организовывать и записывать подбор товаров, для которых установлен метод сброса **Вручную**. Когда вы регистрируете подбор запасов, расход отобранных компонентов учитывается. Вы можете также использовать страницу **Перемещение запасов** со ссылкой на исходный документ, чтобы назначить компоненты с установленным методом списания **Вручную**, **Подбор + прямой**, **Подбор + обратный** для производственных заказов.

Если производство интегрировано со складскими процессами через ячейки или путем непосредственного размещения и подборов, компоненты расходуются из ячейки в строке производственного заказа. Все требуемые компоненты должны иметься в этой ячейке. В противном случае учет использованных вручную или при списании товаров для этого компонента останавливается.

> [!NOTE]  
> Ниже приведены разные важные различия между подборами запасов, перемещениями запасов и складскими подборами:  
>
> - При регистрации подбора запасов для внутренней операции, например производства, потребление выбранных компонентов учитывается в тот же момент. При регистрации перемещения запасов или складского подбора для внутренней операции записывается только физическое перемещение требуемых компонентов в ячейку в области операции без учета потребления.  
> - Когда используются складские подборы, поле **Код ячейки** в строке компонента производственного заказа определяет ячейку *взять*, откуда вычитаются компоненты при учете потребления. При использовании перемещения запасов или складского подбора поле **Код ячейки** в строках компонентов производственного заказа определяет *пробел* ячейку в области операции, где работник склада должен поместить компоненты.  

Для подбора или перемещения компонентов для документов-источников исходящий запрос склада должен уведомлять область склада о потребности в компоненте. Исходящая складская заявка создается, когда вы делаете следующее:

- Изменяете статус производственного заказа на "Выпущено".
- Создайте выпущенный производственный заказ.  

Методы списания также влияют на поток компонентов при производстве. Дополнительные сведения см. в разделе [Списание компонентов в соответствии с производственным выпуском](production-how-to-flush-components-according-to-operation-output.md). **Перемещение запасов** со ссылками на исходный документ и **Складской подбор** нельзя использовать для отбора компонентов с помощью методов списания *Прямой* и *Обратный*. **Складской подбор** нельзя использовать для отбора компонентов с помощью любого метода списания, кроме *Вручную*. Для обработки оставшихся компонентов используйте **Перемещение запасов** без ссылки на исходный документ. Дополнительные сведения см. в разделе [Перемещение компонентов в производственную зону в основных конфигурациях склада](warehouse-how-to-move-components-to-an-operation-area-in-basic-warehousing.md).

В расширенных конфигурациях склада, где требуется как подбор, так и отгрузка, вы должны использовать страницу **Складской подбор**, чтобы вывести компоненты с установленным методом списания *Вручную*, *Подбор + прямой*, *Подбор + обратный* для производственных заказов. Дополнительные сведения см. в разделе [Подбор для производства или сборки в расширенных конфигурациях склада](warehouse-how-to-pick-for-internal-operations-in-advanced-warehousing.md)

## <a name="to-pick-components-for-production-and-jobs-in-basic-warehouse-configurations"></a>Чтобы подобрать компоненты для производства и задач в базовых конфигурациях склада

В базовых конфигурациях склада, где склад настроен на использование только подбора, можно выбрать компоненты для производственных действий им строк планирования работ на странице **Подбор запасов**. Дополнительные сведения см. в разделе [Подбор товаров с помощью подбора запасов](warehouse-how-to-pick-items-with-inventory-picks.md).

> [!NOTE]
> Добавлена возможность подбора компонентов для строк планирования работы в волне 2 [!INCLUDE[d365fin](includes/d365fin_md.md)] выпуска 2022 года. Чтобы начать использовать эту возможность, администратор должен включить **Обновление функции: включение товарного и складского подбора из работ** на странице **Управление функциями**.
>
>[!INCLUDE[prod_short](includes/prod_short.md)] использует значение в поле **Оставшееся количество** в строке планирования работы, когда он создает подборки запасов. Чтобы использовать подборку запасов для работ, необходимо включить выключатель **Применить связь использования** на странице **Карточка работы** для работы. Это позволяет отслеживать использование по вашему плану. Если вы не включите этот выключатель, оставшееся количество останется на **0** и подборка запасов не будет создана. Дополнительные сведения см. в разделе [Настройка трассировки потребления работы](projects-how-setup-jobs.md?tabs=current-experience#to-set-up-job-usage-tracking).

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), значок, введите **Подбор запасов**, а затем выберите связанную ссылку.  
2. Чтобы получить доступ к компонентам производственного заказа, выберите действие **Получить исходные документы**, затем выберите запущенный производственный заказ.  
3. Выполните подбор компонента, затем запишите информацию о подборе в поле **Кол-во для обработки**.  
4. Когда строки будут готовы для учета, выберите действие **Учет**. Во время процедуры учета создаются необходимые складские операции и учитывается потребление товаров.  

Можно также создать **Товарный подбор** непосредственно из запущенного производственного заказа. Выберите действие **Создать размещение/подбор/перемещение запасов**, установите флажок **Создать товарный подбор**, затем выберите кнопку **ОК**.

В качестве альтернативы используйте **Перемещение запасов** со ссылкой на исходный документ для перемещения товаров между ячейками. Потребление нужно будет регистрировать отдельно. Дополнительные сведения см. разделе [Учет потребления для производства в пакетном режиме](production-how-to-post-consumption.md)

## <a name="pick-for-assembly-in-basic-warehouse-configurations"></a>Подбор для сборки в базовой конфигурации склада

Используйте следующие страницы для подбора заказов на сборку:

- Страница **Перемещение запасов**.
- Если для склада требуется подборка, но не отгрузка, используйте страницу **Подбор запасов** для подбора, сборки и отгрузки заказов на продажу, когда товары необходимо собрать до их отгрузки. Для получения дополнительных сведений см. [Обработка собираемого на заказ товара с помощью подбора запасов](warehouse-how-to-pick-for-production.md#handling-assemble-to-order-items-with-inventory-picks).  

В расширенных настройках складов, в котором требуется подбор и отгрузки, необходимо использовать страницу **Складской подбор**, чтобы привести компоненты к заказам на сборку. Дополнительные сведения см. в разделе [Подбор для производства или сборки в расширенных конфигурациях склада](warehouse-how-to-pick-for-internal-operations-in-advanced-warehousing.md)

## <a name="handling-assemble-to-order-items-with-inventory-picks"></a>Обработка товаров сборки на заказ со складскими подборками

Страница **Подбор запасов** также используется для подбора и отгрузки для продажи, когда товары необходимо собрать до их отгрузки. Дополнительные сведения см. в разделе [Продажа товара, собранного на заказ](assembly-how-to-sell-items-assembled-to-order.md).

Товары для отгрузки, которые собираются на заказ, не считаются имеющимися в ячейке до тех пор, пока они не будут собраны и учтены как исходящие в ячейке в области сборки. Таким образом, подбор этих товаров для отгрузки осуществляется специальным потоком. Работники склада берут из ячейки сборочные элементы для склада отгружаемых товаров, а затем учитываются складской подбор. Учтенный складской подбор затем учитывает производительность сборки, потребление компонентов и отгрузки продаж.

Можно настроить [!INCLUDE[prod_short](includes/prod_short.md)] для автоматического создания перемещения запасов при создании складского подбора для сборочного элемента. Чтобы создавать движения автоматически, выберите поле **Создавать перемещения автоматически** на странице **Настройка сборки**. Дополнительные сведения см. в разделе [Перемещение компонентов в производственную зону на основном складе](warehouse-how-to-move-components-to-an-operation-area-in-basic-warehousing.md).

[!INCLUDE[prod_short](includes/prod_short.md)] создает строки складского подбора для продаваемых товаров по другому, в зависимости от того, какое количество строк продажи собирается на заказ (нет, некоторые или все).

- В продажах, где вы используете подборку запасов для учета отгрузки запасов, [!INCLUDE[prod_short](includes/prod_short.md)] создает одну строку подбора запасов для каждой строки заказа на продажу. Если товар помещен в разные ячейки, будет создано более одной строки. Строки подбора основана на количестве в поле **Кол-во для отгрузки**.
- В документах продаж, где все количество в строке заказа на продажу собрано на заказ, создается одна строка складского подбора для этого количества товара. Значение "Количество для сборки" совпадает со значение в поле **Кол-во для отгрузки**. В строке выбрано поле **Сборка на заказ**.

Если выход сборки настроен для данного склада, то значение в полях **Код ячейки сборки на заказ** или **Код исходящей сборочн. ячейки** в этом заказе вставляется в поле **Код ячейки** в строке складского подбора.

Если в строке заказа на продажу не указан код ячейки и не настроен поток сборки для склада, поле **Код ячейки** строки складского подбора останется пустым. Работник склада должен открыть страницу **Содержимое ячейки** и выбрать ячейку, где собираются сборочные элементы.

Когда часть количества нужно сначала собрать, а другую отобрать из запасов, [!INCLUDE[prod_short](includes/prod_short.md)] создает как минимум две строки подбора запасов. Одна строка складского подбора предназначена для количества сборки на заказ. Другая строка подбора зависит от того, какие ячейки могут пополнить остаток со склада. Коды ячеек в двух строках заполняются по-разному, как описано соответственно для двух разных типов продажи. Дополнительные сведения см. в подразделе "Сценарии комбинации" в разделе [Сборка на заказ и сборка на склад](assembly-assemble-to-order-or-assemble-to-stock.md).
