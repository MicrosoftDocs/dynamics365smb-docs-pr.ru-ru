# <a name="walkthrough-managing-projects"></a>Пошаговое руководство. Управление проектами с помощью работ

<!-- [!INCLUDE[complete_sample_data](includes/complete_sample_data.md)]   -->

В этом пошаговом руководстве даются общие сведения о функциях управления проектами в заданиях. Работы используются для планирования использования ресурсов организации и отслеживания различных затрат, связанных с ресурсами в конкретном проекте. В приложении "Работы" учитывается использование человеко-часов, станко-часов, товаров в запасах и других типов потребления, которые может понадобиться отследить в процессе выполнения работы.  

 В данном пошаговом руководстве описана процедура настройки новой работы в дополнение к заданиям более общего характера, таких как работа с фиксированными ценами, оплата в рассрочку, учет счетов из работ и копирование работ.  

## <a name="about-this-walkthrough"></a>Об этом пошаговом руководстве

 В этом пошаговом руководстве демонстрируются следующие задачи:  

### <a name="setting-up-a-project"></a>Настройка работы

 Если структура бюджета настроена для работ, создание работы будет простым процессом. В этом пошаговом руководстве рассматриваются следующие процедуры:  

- Настройка строк рабочего задания и строк планирования.  
- Создание цен для товаров, ресурсов и счетов Главной книги, характерных для конкретной работы.  
- выставление счетов из работы.  

### <a name="handling-fixed-prices"></a>Работа с фиксированными ценами

 В области работ можно применять фиксированные цены и цены на услуги или товары, которые были заранее оговорены клиентами. В этом пошаговом руководстве выполняются следующие задачи:  

- Узнать о процедуре определения стоимости по контракту и по счету.  
- Разрешить включение в план работы, по которой не выставлен счет.  

### <a name="copying-a-job"></a>Копирование работы

 Данная часть пошагового руководства посвящена процедуре полного или частичного копирования работы для сокращения количества вводимых вручную данных и повышения точности. Это включает следующее:  

- копирование части работы в новую работу;  
- копирование характерных для работы цен;  
- копирование строк планирования.  

### <a name="making-payment-by-installment"></a>Оплата в рассрочку

 Когда работа над большим и дорогостоящим проектом продолжается в течение длительного периода времени, клиенты часто договариваются с организацией про оплату в рассрочку. В этом сценарии демонстрируется процедура оформления оплаты в рассрочку, в том числе:  

- Создание оплаты в рассрочку для работы.  
- выставление счетов клиентам;  
- Учет для использования в работе, настроенной для оплаты в рассрочку.  

## <a name="roles"></a>Роли

 В этом пошаговом руководстве описываются задачи для следующих ролей:  

- Менеджер проекта  
- Член проектной группы  

## <a name="prerequisites"></a>Предварительные требования

 Перед выполнением заданий этого пошагового руководства необходимо выполнить следующие действия.  

- Установите демонстрационную базу данных CRONUS.
- Создать образец данных, выполнив шаги следующего раздела.  

## <a name="story"></a>Сюжет

В этом пошаговом руководстве рассматривается CRONUS, консультационно-конструкторская фирма, занимающаяся проектированием и монтажом новых инфраструктур, например конференц-залов и офисов с мебелью, вспомогательным оснащением и камерами хранения. Большая часть деятельности этой организации ориентирована на проекты. Станислав, менеджер проекта в CRONUS, использует работы для получения обзора по каждой выполняемой работе, начатой CRONUS, а также по уже завершенным работам. Именно Станислав обычно оформляет сделки с клиентами и вводит основные компоненты работы, которые включают строки задания и планирования, а также цены, в [!INCLUDE[prod_short](includes/prod_short.md)]. Станислав считает, что создание, поддержка и извлечение сведений не вызывают трудностей. Станиславу также нравится, как [!INCLUDE[prod_short](includes/prod_short.md)] позволяет копировать работы и оплату в рассрочку.

 Татьяна, член проектной группы, подотчетной Станиславу, отвечает за ежедневный мониторинг работы. По каждому заданию Татьяна вводит данные о своей работе в дополнение к данным о работе, выполняемой техническими специалистами, а также регистрирует использованные ими товары и затраты, связанные с их работой.

В следующей процедуре вы создадите раздел журнала работ, в котором будет производиться учет потребления, связанного с работой Татьяны.

2.  На странице **Журнал работ** выберите поле **Код раздела**. Откроется страница **Разделы журнала работ**.  
3.  Чтобы создать новую строку со следующей информацией, выберите действие **Создать**.  

    - **Имя**: **Татьяна**  
    - **Описание**: **Татьяна**  
    - **Серия номеров**: **ЖУРР-ОБЩ**  

4.  Нажмите кнопку **ОК**, чтобы сохранить изменения.

## <a name="setting-up-a-project-1"></a>Настройка работы

 В этом сценарии CRONUS удалось заключить контракт с клиентом, Progressive Home Furnishings, на проектирование конференц\-зала и столовой. Клиент находится в США, и для проекта потребуется специальное программное обеспечение. Менеджер проекта должен договориться с клиентом и создать работу, соответствующую условиям этого договора.  

### <a name="to-set-up-a-project"></a>Настройка работы

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите действие **Создать**, чтобы создать новую карточку.  
3.  На экспресс-вкладке **Общее** введите следующие сведения:  

    - **Описание**: **Рекомендации по оборудованию конференц-зала**  
    - **Код плательщика**: **01445544**  

4.  На экспресс-вкладке **Учет** введите следующие сведения:  

    - **Состояние**: **Планирование**  
    - **Работа - учетная группа**: **Настройка**  
    - **Метод НЗП**: **Себестоимость**  

5.  На экспресс-вкладке **Длительность** введите сегодняшнюю дату в поля **Дата начала** и **Дата окончания**. Эти даты помогут с конвертацией валют при выставлении счета за работу.  
6.  На экспресс-вкладке **Внешняя торговля** задайте в качестве кода валюты значение **USD**. Если в поле **Код валюты счета** выбрано значение USD, счета за работу будут выставляться в долларах США, а планироваться только в локальной валюте CRONUS.  

 Можно корректировать цены для клиента для каждого задания в зависимости от соглашений. В следующей процедуре менеджер проекта указывается стоимость времени Татьяны, устанавливает цену для обязательного программного обеспечения и добавляет командировочные расходы, которые клиент согласился оплатить.  

### <a name="to-customize-pricing"></a>Настройка цен

1.  Из карточки работы выберите действие **Ресурс**.  
2.  На странице **Работа - цены ресурсов** введите следующие сведения:  

    - **Код**: **Татьяна**  
    - **Цена единицы**: **20**  

3.  Закройте страницу.  
4.  Выберите действие **Товар**.  
5.  На странице **Работа - цены товаров** введите следующие сведения и настроенную цену:  

    1.  **Код товара**: **80201 (программа обработки графики)**  
    2.  **Цена единицы**: **200**  

6.  Закройте страницу.  
7.  Выберите действие **Счет ГК**.  
8.  На странице **Работа — цены по счету ГК** введите следующую информацию и себестоимость поездки, которую клиент согласился оплатить, + 25 процентов:  

    1.  **Счет ГК**: **8430 (дорога)**  
    2.  **Коэффициент себестоимости единицы**: **1,25**  

9. Закройте страницу.  

 Заключительным этапом настройки задания является добавление рабочих заданий и строк планирования, являющихся частью каждой задачи. От этих строк планирования зависят компоненты, по которым клиенту будет выставлен счет.  

### <a name="to-add-project-tasks"></a>Добавление рабочих заданий

1.  На карте **Работа** для новой работы выберите действие **Строки рабочего задания**.  
2.  В следующей таблице даны сведения, которые требуется ввести в поля.  

    |Номер рабочего задания|Описание|Тип рабочего задания|  
    |------------------|---------------------------------------|-------------------|  
    |1000|Консультации по оборудованию зала|Сумма (от)|  
    |1010|Консультационная встреча с клиентом|Учет|  
    |1020|Разработка|Учет|  
    |1090|Всего за консультации|Сумма (до)|  

3.  Чтобы показать, что некоторые задания являются подкатегориями других заданий, выберите действие **Вложенность раб. зад.**.  

 Строка планирования может быть одного из следующих типов:  

- **Бюджет**: добавлено в плане без выставления счета.  
- **К оплате**: счет выставлен, но не добавлен в плане.  
- **Бюджет и к оплате**: выставлен счет и добавлено в план.  

 В этом пошаговом руководстве менеджер проекта использует **Бюджет и к оплате**. Менеджер создает три строки планирования для задачи 1010 и две строки планирования для задачи 1020.

## <a name="calculating-remaining-usage"></a>Расчет остаточного потребления

 Член проектной группы Татьяна какое-то время выполняла определенную работу и желает зарегистрировать отработанные часы и потребление. Татьяна не превысила количество часов работы, заранее оговоренное с клиентом. Татьяна использует пакетное задание **Расчет оставшегося потребления** для вычисления оставшегося потребления для работы в журнале работ. Для каждой задачи при выполнении этого пакетного задания вычисляется разница между запланированным потреблением товаров, ресурсов и ГК расходов и фактическим потреблением, учтенным в операциях книги работ. Оставшееся потребление затем отображается в журнале работ, из которого его можно учесть.  

### <a name="to-calculate-remaining-usage"></a>Процедура расчета остаточного потребления

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Журналы работы**, а затем выберите связанную ссылку.  
2.  На странице **Журнал работ** откройте в поле **Код раздела** список **Журнал работ - разделы**. Выберите раздел журнала работ **Татьяна**.  
3.  Выберите действие **Расчет остаточного потребления**.  
4.  На странице **Работа — расчет остаточного потребления** на экспресс-вкладке **Рабочее задание** выберите поле **Код работы**, а затем выберите нужный номер работы, обычно работу J00010.  
5.  На экспресс-вкладке **Параметры** введите в поле **Номер документа** значение **J00001**. Это упростит дальнейшую трассировку учета.  
6.  Введите в качестве даты учета сегодняшнюю дату.  
7.  Нажмите кнопку **ОК**. В результате будут созданы строки журнала работ, которые являются производными от строк планирования и созданы Станиславом для работы.  
8.  Нажмите кнопку **ОК** на странице подтверждения. Созданные строки добавлены в журнал работ.  
9. Убедитесь, что все номера документов равны J00001, затем выберите действие **Учет**. Выберите **Да**, чтобы подтвердить учет.  

Теперь строки учтены.  

## <a name="creating-and-posting-a-job-sales-invoice"></a>Создание и учет счета продажи для работы

 Далее Татьяна может создать новый счет для работы в целом или части работы. Кроме того, Татьяна может присоединять друг к другу счета одного клиента по одной и той же работе. В этом случае Татьяна выставляет счета за всю работу, поскольку проект теперь завершен.  

### <a name="to-create-a-job-sales-invoice"></a>Процедура создания счета продажи для работы

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите созданную ранее работу, затем выберите действие **Работа — создание счета продажи**.  
3.  На экспресс-вкладке **Рабочее задание** снимите фильтр **Номер рабочего задания**, чтобы выставить счет по работе. В поле **Номер работы** выберите соответствующую работу.  
4.  На экспресс-вкладке **Параметры** введите дату учета и укажите, требуется ли создать счет по каждой задаче или один счет для всех задач.  
5.  Нажмите кнопку **ОК**, чтобы создать счет, а затем нажмите кнопку **ОК** на странице подтверждения.  

 Создав счет-фактуру, Татьяна может получить к ней доступ, например, из ролевого центра **Обработчик заказов на продажу**. 

### <a name="to-post-a-new-sales-invoice"></a>Процедура учета нового счета продажи для работы

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Счета продажи**, а затем выберите связанную ссылку.  
2.  Откройте счет для клиента с номером 01445544. Теперь можно просмотреть сведения, введенные из строк планирования.  
3.  Выберите действие **Учет**. Выберите **Да**, чтобы подтвердить учет.  

### <a name="to-view-the-posted-invoice"></a>Процедура просмотра учтенного счета

1.  Откройте работу, затем выберите действие **Строки планирования работ**.  
2.  Выберите любые строки планирования, по которым выставлен счет, затем выберите действие **Счет продажи/кредит-нота**.
3. На странице **Счета работ** выберите действие **Открыть счет продажи/кредит-ноту**.  

 У Татьяны возник вопрос относительно цен, затрат и прибыли в связи с данной конкретной работой, поэтому Татьяна открывает эти сведения на странице **Статистика**.  

### <a name="to-open-the-statistics-page"></a>Открытие страницы статистики

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2.  Выберите действие **Статистика**. Можно просмотреть подробные сведения о цене, себестоимости, и прибыли задачи в местной и в иностранной валюте.  
3.  Нажмите кнопку **Закрыть**, чтобы закрыть страницу **Работа - статистика**.  

## <a name="handling-fixed-prices-1"></a>Работа с фиксированными ценами

 CRONUS получила контракт на оборудование конференц\-зала. Как менеджеру проекта, Станиславу необходимо получить полное представление о заданиях, необходимых для выполнения работы, вместе с бюджетируемой себестоимостью и понесенными издержками по каждому заданию. Кроме того, Станиславу необходимо знать общую цену работы по контракту и сумму, на которую к данному моменту были выставлены счета. Он договорился с клиентом о фиксированной цене работы.  

### <a name="to-manage-fixed-pricing-in-jobs"></a>Управление фиксированной ценой в работах

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2. Выберите номер работы **Водоканал**, затем выберите действие **Строки рабочего задания**.  
3. Выделите строку 1120, затем в поле **Бюджет (итог. себест.)** щелкните правой кнопкой мыши сумму и нажмите **DrillDown**.  

     Просмотрев строки планирования работ, Станислав определил, что на этом этапе проекта также необходимо 30 часов работы Татьяны. Станислав договаривается с клиентом о фиксированной цене.  

4. На странице **Строки рабочего задания** выберите строку 1120, затем выберите действие **Строки планирования работ**. Создайте строку планирования с следующими сведениями:  

    | Строка | Тип строки | Тип        | №   | количество; |
    |------|-----------|-------------|-------|----------|
    | 1    | Бюджет и к оплате  | Ресурс | Татьяна | 30 |

     Закройте страницу.  
5. В поле **Бюджет (итог. себест.)** щелкните правой кнопкой мыши поле, а затем еще раз выберите **Раскрытие** на странице **Рабочее задание — строки**. Просмотр изменений в расписании. Как можно увидеть, в план было добавлено 30 часов.  
6. Закройте страницы.  

Будучи добавленной в график для данной строки задания, Татьяна выполняет эту работу в течение 25 часов и вводит эти часы в журнал работ.  

### <a name="to-enter-hours-in-a-project-journal"></a>Процедура ввода часов в журнал работ

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Журналы работы**, а затем выберите связанную ссылку.  
2. В новой строке введите следующие сведения:  

    - **Тип строки**: **(пусто)**  
    - **Дата учета**: **(сегодняшняя дата)**  
    - **Номер документа**: **J00002**  
    - **Код работы:**: **водоканал**  
    - **Номер рабочего задания**: **1120**  
    - **Тип**: **Ресурс**  
    - **Номер**: **Татьяна**  
    - **Кол-во**: **25**;  

3. Выберите действие **Учет**.  

     Спустя несколько дней Татьяна проработала еще 10 часов, и теперь общее отработанное ею время составляет 35 часов. Поскольку договор с клиентом был заключен на 30 часов, ему будет выставлен счет только на 5 из этих часов. Татьяна вручную добавит в график пять дополнительных часов работы.  

4. На странице **Журнал работ** выберите действие **Расчет остаточного потребления**.  
5. На странице **Работа - расчет остаточного потребления** на экспресс-вкладке **Параметры** введите следующие сведения:  

    - **Номер документа**: **J00003**  
    - **Дата учета**: **(сегодняшняя дата)**  

6. На экспресс-вкладке **Рабочее задание** введите следующие сведения:  

    - **Код работы:**: **водоканал**  
    - **Номер рабочего задания**: **1120**  

7. Нажмите кнопку **ОК** для запуска расчета.

    У Татьяны осталось 5 часов работы. Поле **Тип строки** пустое, что указывает на то, что остается учесть только потребление, поскольку задание уже запланировано.  

8. В окне **Журнал работ** создайте новую строку со следующими данными. Убедитесь, что оба номера работ следуют непосредственно за уже использованными номерами:  

    - **Тип строки**: **Бюджет**  
    - **Код работы:**: **водоканал**  
    - **Номер рабочего задания**: **1120**  
    - **Тип**: **Ресурс**  
    - **Номер**: **Татьяна**  
    - **Кол-во**: **5**  

     При использовании типа строки **Бюджет** обновляются запланированные себестоимость и цены, но себестоимость по контракту и цены, по которым клиенту выставляется счет, остаются прежними.  

9. Выберите действие **Учет**. Выберите кнопку **ОК**, чтобы закрыть страницу.  
10. Откройте список **Работы**.  
11. Выберите задание GUILDFORD, затем в разделе **Строки рабочего задания** выберите строку 1120 и в поле **Бюджет (итог. себест.)** щелкните правой кнопкой мыши на сумме. Выберите **DrillDown**, чтобы просмотреть сведения.  

     Изменения автоматически вводятся в строку номера рабочего задания 1120. В итоговой себестоимости плановой работы были добавлены в план пять дополнительных часов работы Татьяны.  

12. Выберите кнопку **Закрыть**, чтобы закрыть страницу.  
13. Щелкните правой кнопкой мыши сумму в поле **Контракт (итоговая себест.)** и выберите **DrillDown**, чтобы просмотреть сведения.  

В общую цену по контракту включены только изначально предусмотренные контрактом 30 часов, поскольку именно такое время было оговорено с клиентом.  

## <a name="copying-projects"></a>Копирование работ

Станислав договорился с клиентом, ЗАО «Нефтераз», об оборудовании 10 конференц-залов. Соглашение похоже на более раннее задание. Соответственно, копирование этой более ранней работы позволит сэкономить время.  

На странице **Копирование работы** можно выбрать работу и строки задания, которые необходимо скопировать. Кроме того, можно выбрать копирование операций книги исходной работы, что приведет к созданию строк планирования на основе фактического потребления, или копирование строк планирования исходной работы, что приведет к копированию в новую работу исходных строк планирования. Затем можно выбрать строку планирования или тип строки книги операций, которые необходимо добавить, выбрав только те данные, которые касаются этой новой работы. И, наконец, можно выбрать работу, в которую необходимо скопировать данные, и указать, следует ли также копировать цены и количества.  

### <a name="to-copy-a-project"></a>Процедура копирования работы

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Работы**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**, чтобы создать новую работу. Введите следующие сведения:  

    - **Описание**: **Оборудование 10 конференц-залов**  
    - **Код плательщика**: **20000**  

3. Выберите действие **Копировать рабочие задания из**,  
4. На странице **Копировать рабочие задания** введите следующие сведения:  

    - **Код работы:**: **водоканал**  
    - **С номера рабочего задания**: **1000**  
    - **Источник**: **Планирование работ - строки**  
    - **Включ. строки планирования типа**: **Бюджет + К оплате**  
    - **В работу с кодом**: **Водоканал Оборудование 10 конференц\-залов**  
    - Выберите поля **Копировать измерения** и **Копировать количество**.  

5. Нажмите кнопку **ОК**, чтобы скопировать работу, а затем нажмите кнопку **ОК**, чтобы закрыть страницу подтверждения.  

Путем сравнения цен, строк рабочих заданий и строк планирования работ для двух работ можно увидеть, что информация была успешно скопирована.  

## <a name="making-payments-by-installments"></a>Оплата в рассрочку

CRONUS только что получила крупный проект, на осуществление которого уйдет год. Поскольку для него требуется выделить много ресурсов, менеджер проекта настраивает контракт таким образом, чтобы клиент выплатил часть стоимости авансом, еще часть, когда проект будет наполовину завершен, и окончательно рассчитался после завершения проекта.  

### <a name="to-set-up-a-new-account"></a>Настройка нового счета

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **План счетов**, а затем выберите связанную ссылку.  
2. На странице **План счетов** выберите действие **Создать**, чтобы создать новую карточку.  
3. В карточке **нового счета ГК** введите следующие сведения:  

    - **Номер**: **40255**  
    - **Название**: **Оплата работы**  

4. На экспресс-вкладке **Учет** в поле **Общая товарная группа** выберите **Услуги**. Закройте страницу.  
5. На странице **План счетов** выберите **Оплата работы № 40255**, затем выберите действие **Создать отступы в плане счетов**. Нажмите **Да** для подтверждения.  

Следующие процедуры описывают создание нового задания, установки цены и последующей настройки оплаты в рассрочку. В строках рабочего задания можно создать особые строки, предназначенные для оплаты в рассрочку. Все задания, которые будут выполняться в связи с работой, добавленной в план, будет вводиться в строках потребления. Для каждой строки задания оплаты в строках планирования можно ввести в качестве типа строки значение **К оплате**, которое означает, что клиенту будет выставлен счет. Введите новую строку для авансового платежа. В строке задания потребления можно ввести сведения об используемых в этом проекте товарах и ресурсах, расширив с их помощью план, например, указав для работы количество часов сотрудника и число используемых товаров.  

### <a name="to-make-a-payment-by-installment"></a>Процедура оплаты в рассрочку

1. Создать новую работу.  
2. В карточке **Работа** укажите следующие сведения:  

    - **Описание**: **Отделка приемной**  
    - **Код плательщика**: **30000**  
    - **Работа - учетная группа**: **Настройка**  
    - **Метод НЗП**: **Себестоимость**  

3. На карточке работы выберите действие **Цены**, затем выберите действие **Ресурс**. Введите следующие сведения:  

    - **Код**: **Татьяна**  
    - **Цена единицы**: **10**  

     Закройте страницу.  

4. На карточке **Работа** в разделе **Задачи** добавьте строки рабочего задания, как описано в следующей таблице:  

    | Строка | Номер рабочего задания | Описание          | Тип рабочего задания |
    |------|--------------|----------------------|---------------|
    | 1    | 1000         | Оплата — авансовая оплата | Учет       |
    | 2    | 2000         | Потреблено                | Учет       |
    | 3    | 3000         | Платеж — промежуточный     | Учет       |
    | 4    | 4000         | Платеж — завершенный | Учет       |

5. Выберите задачу 1000, затем выберите действие **Строки планирования работ**.  

6. Создайте строку планирования со следующими сведениями:  

    | Строка | Тип строки | Дата планирования  | Тип        | №   | количество; | Цена единицы |
    |------|-----------|----------------|-------------|-------|----------|------------|
    | 1    | К оплате  | (текущая дата) | Счет ГК | 40255 | 1        | 5000       |

     Закройте страницу.  

7. Выберите задачу 2000, затем выберите действие **Строки планирования работ**.  

8. Создайте строку планирования со следующими сведениями:

    | Строка | Тип строки | Дата планирования  | Тип     | №    | количество; |
    |------|-----------|----------------|----------|--------|----------|
    | 1    | Бюджет    | (текущая дата) | Ресурс | Татьяна | 120      |
    | 2    | Бюджет    | (текущая дата) | Товар     | 70104  | 10       |

     Закройте страницу. На странице **Строки рабочего задания** можно просмотреть обновленные запланированные суммы.  

9. Выберите задачу 32000, затем выберите действие **Строки планирования работ**.  

10. Создайте строку планирования со следующими сведениями:

    | Строка | Тип строки | Дата планирования   | Тип        | №   | количество; | Цена единицы |
    |------|-----------|-----------------|-------------|-------|----------|------------|
    | 1    | К оплате  | (дата в будущем) | Счет ГК | 40255 | 1        | 5000       |

     Закройте страницу.  

11. Создайте подобную запись строки планирования для рабочего задания 4000.  

 Теперь, после ввода строк рабочего задания и планирования, Станислав создаст счет для первого платежа. Чтобы счет содержал только строки для первого платежа, Станислав использует для его создания строки рабочего задания. Заказ на продажу можно открыть из строк планирования или строк задачи.  

### <a name="to-create-an-invoice"></a>Создание счета

1.  На странице **Строки рабочего задания** выберите строку 1000, затем выберите действие **Создать счет продажи**.  
2.  На странице **Создание счета продажи** установить в качестве даты учета сегодняшнюю дату, затем укажите **В задачу** и нажмите кнопку **ОК**, чтобы создать счет со сведениями по умолчанию. Нажмите кнопку **ОК**, чтобы закрыть страницу подтверждения.  
3.  Выберите действие **Счет/кредит-нота продажи**. Как можно увидеть в счете продажи, в счет включен только аванс. Теперь его можно отправить клиенту, как и было оговорено.
