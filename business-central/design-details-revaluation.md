# <a name="design-details-revaluation"></a>Сведения о проектировании: переоценка

Можно переоценить запасы с учетом базы оценки, которая наиболее точно отражает стоимость запасов. Можно записать переоценку задним числом, чтобы правильно обновить себестоимость проданных товаров для товаров, которые вы уже продали. Товары с методом учета себестоимости "По стандартной", по которым счета не выставлены (или выставлены не полностью), также можно переоценить.  

В [!INCLUDE[prod_short](includes/prod_short.md)] поддерживается следующая гибкость при переоценке:  

- Переоцениваемое количество можно вычислить для любой даты, даже в прошлом.  
- В случае товаров, для которых используется метод учета себестоимости "По стандартной", ожидаемые операции себестоимости включены в переоценку.  
- Обнаруживаются расходы склада, на которые повлияла переоценка.  

## <a name="calculate-the-revaluable-quantity"></a>Расчет количества, доступного для переоценки

Количество, которое можно переоценить, — это остаток запасов, доступный на заданную дату. Количество представляет собой итог операций книги товаров, по которым полностью выставлены счета, учтенных в день переоценки или ранее.  

> [!NOTE]  
>  Товары с методом учета себестоимости "По стандартной" обрабатываются по-разному при расчете переоцениваемого количества по товару, складу и варианту. Количества и стоимость операций журнала товаров, счета за которые выставлены не полностью, включаются в переоцениваемое количество.  

После учета переоценки можно учитывать расход или приход склада с датами учета до даты учета переоценки. Однако переоценка не повлияет на это количество. Чтобы уравновесить запасы, учитывается только исходное переоцениваемое количество.  

Поскольку переоценку можно выполнить в любой день, необходимы соглашения, определяющие, когда товар считается частью запасов. Например, если товар хранится в запасах и если товар является незавершенным производством (НЗП).

## <a name="expected-cost-in-revaluation"></a>Ожидаемая себестоимость при переоценке

Количество, которое может быть переоценено, рассчитывается как сумма количеств по операциям журнала товаров, по которым полностью выставлены счета, учтенных в день переоценки или ранее. Когда товары получены или отгружены, но по счета по ним не выставлены, рассчитать их стоимость как запасов невозможно. Товары, для которых используется метод учета себестоимости "По стандартной", не ограничены в этом отношении.  

> [!NOTE]  
>  Еще одним типом ожидаемой себестоимости, которую можно переоценить, являются запасы НЗП при соблюдении некоторых правил. Дополнительные сведения см. в разделе [Переоценка запасов НЗП](design-details-revaluation.md#wip-inventory-revaluation).  

При расчете доступного для переоценки количества для товаров, для которых метод учета себестоимости "По стандартной", в расчет включаются операции книги товаров, счета по которым не выставлены (или выставлены не полностью). Эти операции переоцениваются при учете переоценки. При выставлении счета по переоцененной операции создаются следующие операции стоимости:  

- Обычная учтенная операция стоимости с типом **Прямые затраты**. Сумма себестоимости этого товара — это прямые затраты из исходной строки.  
- Операция стоимости с типом **Отклонение**. Эта операция записывает разницу между стоимостью, на которую выставлен счет, и переоцененной стандартной стоимостью.  
- Операция стоимости с типом **Переоценка**. Эта операция записывает сторнирование переоценки ожидаемой стоимости.

### <a name="example-1"></a>Пример

Следующий пример основан на производстве цепи из предыдущего примера. Этот пример иллюстрирует создание трех типов операций на основе следующего сценария:  

1.  Вы учитываете приобретенные звенья как полученные со стоимостью единицы 2,00 МВ.  
2.  Вы учитываете переоценку звеньев с новой стоимостью единицы 3,00 МВ, тем самым изменяя стандартную стоимость на 3,00 МВ.  
3.  Вы учитываете исходную покупку звеньев цепи как такую, по которой выставлен счет, в результате чего создаются следующие операции стоимости:  

    1.  Учтенная операция стоимости с типом **Прямые затраты**.  
    2.  Операция стоимости с типом **Переоценка** для записи сторнирования переоценки ожидаемой себестоимости.  
    3.  Операция стоимости с типом "Отклонение", записывающая разницу между стоимостью по счету и переоцененной стандартной себестоимостью.

|2.|01-20-20|Переоценка|01-20-20|150,00|0,00|1|2|

## <a name="wip-inventory-revaluation"></a>Переоценка запасов НЗП

Переоценка запасов НЗП подразумевает переоценку компонентов, зарегистрированных как запасы НЗП.  

Важно иметь соглашения, которые определяют, когда товар относится к запасам НЗП с финансовой точки зрения. В [!INCLUDE[prod_short](includes/prod_short.md)] существуют следующие соглашения:  

- Приобретенный компонент становится частью запасов сырья с момента учета покупки как такой, по которой выставлен счет.  
- Приобретенный/собранный субподрядчиком подкомпонент становится частью запасов НЗП с момента учета его потребления в связи с производственным заказом.  
- Приобретенный/собранный субподрядчиком компонент остается частью запасов НЗП до момента выставления счета по производственному заказу (произведенному товару).  

Способ задания даты оценки для операции стоимости потребления следует тем же правилам, что и для запасов НЗП. Подробнее см. в разделе [Определение влияния переоценки на расход склада](#determine-whether-revaluation-affects-an-inventory-decrease).  

Переоценивать запасы НЗП можно при соблюдении следующих условий:

- Дата переоценки предшествует датам учета соответствующих операций книги товаров типа "Потребление".
- Вы не выставили счет по производственному заказу.  

> [!CAUTION]  
> Отчет **Оценка стоимости запасов — НЗП** отражает стоимость учтенных операций производственного заказа, что может несколько осложнять работу с переоцененными товарами НЗП.  

## <a name="revaluate-items-with-the-average-costing-method"></a>Переоценка товаров с методом учета себестоимости "По средней"

Вы можете переоценивать товары, для которых используется метод учета себестоимости "По средней", только если параметр **Вычислить для** имеет значение *Товар*.

Выполнить переоценку можно только в конце периода, выбранного в поле **Период расчета средней себестоимости** на странице **Настройка модуля "Запасы"**.

Переоценка не повлияет на отрицательные транзакции в текущем месяце, поэтому полностью примененные входящие операции также не включаются.
