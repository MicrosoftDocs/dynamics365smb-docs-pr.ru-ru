# <a name="design-details-revaluation" />Сведения о проектировании: переоценка
Можно переоценить запасы с учетом базы оценки, которая наиболее точно отражает стоимость запасов. Можно записать переоценку задним числом, чтобы себестоимость проданных товаров правильно обновлялась для уже проданных элементов. Товары с методом учета себестоимости "По стандартной", по которым счет выставлен не полностью, также можно переоценить.  

В [!INCLUDE[prod_short](includes/prod_short.md)] поддерживается следующая гибкость при переоценке:  

-   Переоцениваемое количество можно вычислить для любой даты, даже в прошлом.  
-   В случае товаров, для которых используется метод учета себестоимости "По стандартной", ожидаемые операции себестоимости включены в переоценку.  
-   Обнаруживаются расходы склада, на которые повлияла переоценка.  

## <a name="calculating-the-revaluable-quantity" />Расчет количества, доступного для переоценки
 Переоцениваемое количество — это остаток запасов, доступный переоценке на заданную дату. Оно вычисляется как общая сумма количеств операций книги товаров, по которым полностью выставлен счет и дата учета которых совпадает или предшествует дате учета переоценки.  

> [!NOTE]  
>  Товары с методом учета себестоимости "По стандартной" обрабатываются по-разному при расчете переоцениваемого количества по товару, складу и варианту. Количества и стоимость операций журнала товаров, счета за которые выставлены не полностью, включаются в переоцениваемое количество.  

После учета переоценки можно учитывать расход или приход склада с датами учета до даты учета переоценки. Однако переоценка не повлияет на это количество. Чтобы уравновесить запасы, учитывается только исходное переоцениваемое количество.  

Поскольку переоценку можно выполнить в любой день, необходимы соглашения, определяющие, когда товар считается частью запасов с финансовый точки зрения. Например, если товар хранится в запасах и если товар является незавершенным производством (НЗП).

В следующем примере показано, когда перенесенные товары НЗП становятся частью запасов. Пример основан на создании цепочки из 150 звеньев.

**1К**. Пользователь учитывает приобретенные звенья как полученные. В следующей таблице показана результирующая операция журнала товаров.  

|Дата учета|Товар|Тип операции|Количество|Номер операции|  
|------------------|----------|----------------|--------------|---------------|  
|01-01-20|СВЯЗЬ|Покупка|150|1|  

> [!NOTE]  
>  Теперь товар с методом учета себестоимости "По стандартной" доступен для переоценки.  

**1C**. Пользователь учитывает приобретенные звенья как звенья, по которым выставлен счет, и звенья становятся частью запасов с финансовой точки зрения. В следующей таблице показаны результирующие операции стоимости.  

|Дата учета|Тип операции|Дата оценки|Сумма себестоимости (факт.)|Номер товарной операции|Номер операции|  
|------------------|----------------|--------------------|----------------------------|---------------------------|---------------|  
|01-15-20|Прямая себестоимость|01-01-20|150,00|1|1|  

 **2К + 2С**. Пользователь учитывает приобретенные звенья как использованные в производстве железной цепи. С финансовый точки зрения, связи становятся частью запасов НЗП.  В следующей таблице показана результирующая операция журнала товаров.  

|Дата учета|Товар|Тип операции|Количество|Номер операции|  
|------------------|----------|----------------|--------------|---------------|  
|02-01-20|СВЯЗЬ|Потребление|-150|1|  

В следующей таблице показана результирующая операция стоимости.  

|Дата учета|Тип операции|Дата оценки|Сумма себестоимости (факт.)|Номер товарной операции|Номер операции|  
|------------------|----------------|--------------------|----------------------------|---------------------------|---------------|  
|02-01-20|Прямая себестоимость|02-01-20|-150,00|2|2|  

Дата оценки устанавливается на дату учета потребления (02-01-20) в качестве обычного складского расхода.  

**3К**. Пользователь учитывает цепь как выход и завершает производственный заказ. В следующей таблице показана результирующая операция журнала товаров.  

|Дата учета|Товар|Тип операции|Количество|Номер операции|  
|------------------|----------|----------------|--------------|---------------|  
|02-15-20|ЦЕПЬ|Выход|1|3|  

**3C**. Пользователь выполняет пакетное задание **Коррекция себест. запасов**, в ходе которого цепь учитывается как цепь, по которой выставлен счет, чтобы указать, что по всем потребленным материалам полностью выставлен счет. С финансовый точки зрения связи больше не являются частью запасов НЗП, когда по выходу полностью выставлен счет и выход откорректирован. |01-15-20|Прямая себестоимость|01-01-20|150,00|2|2|

|02-15-20|Прямая себестоимость|02-15-20|150,00|3|3|  

## <a name="expected-cost-in-revaluation" />Ожидаемая себестоимость в переоценке
Переоцениваемое количество вычисляется как сумма количеств по операциями журнала товаров, за которые полностью выставлен счет, с датой учета равной или предшествующей дате переоценки. Это означает, что если некоторые товары получены или отгружены, но счет за них не выставлен, их инвентарную стоимость вычислить невозможно. Товары, для которых используется метод учета себестоимости "По стандартной", не ограничены в этом отношении.  

> [!NOTE]  
>  Еще одним типом ожидаемой себестоимости, которую можно переоценить, являются запасы НЗП при соблюдении некоторых правил. Дополнительные сведения см. в разделе [Переоценка запасов НЗП](design-details-revaluation.md#wip-inventory-revaluation).  

При вычислении переоцениваемого количества товаров с использованием стандартного метода учета затрат включаются в вычисление операций журнала товаров, счета за которые выставлены не полностью. Эти операции затем переоцениваются при учете переоценки. При выставлении счета для переоцененной операции создаются следующие операции стоимости.  

-   Обычная учтенная операция стоимости с типом **Прямые затраты**. Сумма себестоимости этого товара — это прямые затраты из исходной строки.  
-   Операция стоимости с типом **Отклонение**. Эта операция записывает разницу между стоимостью, на которую выставлен счет, и переоцененной стандартной стоимостью.  
-   Операция стоимости с типом **Переоценка**. Эта операция записывает сторнирование переоценки ожидаемой стоимости.  

### <a name="example-1" />Пример
В следующем примере демонстрируется создание трех типов операций при производстве цепи из предыдущего примера. Он основан на следующем сценарии:  

1.  Пользователь учитывает приобретенные звенья как полученные со стоимостью единицы 2,00 МВ.  
2.  Затем пользователь учитывает переоценку цепей с новой стоимостью единицы 3,00 МВ, обновляя стандартную стоимость до 3,00 МВ.  
3.  Пользователь учитывает исходную покупку звеньев как с выставленным счетом, в результате чего получается следующее.  

    1.  Учтенная операция стоимости с типом **Прямые затраты**.  
    2.  Операция стоимости с типом **Переоценка** для записи сторнирования переоценки ожидаемой себестоимости.  
    3.  Операция стоимости с типом "Отклонение", записывающая разницу между стоимостью по счету и переоцененной стандартной себестоимостью.  
В следующей таблице показаны результирующие операции стоимости.  

|Шаг|Дата учета|Тип операции|Дата оценки|Сумма себестоимости (ожид.)|Сумма себестоимости (факт.)|Номер товарной операции|Номер операции|  
|----------|------------------|----------------|--------------------|------------------------------|----------------------------|---------------------------|---------------|  
|1.|01-15-20|Прямая себестоимость|01-15-20|300,00|0,00|1|1|  
|2.|01-20-20|Переоценка|01-20-20|150,00|0,00|1|2|  
|3.a.|01-15-20|Прямая себестоимость|01-15-20|-300,00|0,00|1|3|  
|3.b.|01-15-20|Переоценка|01-20-20|-150,00|0,00|1|4|  
|3.c.|01-15-20|Отклонение|01-15-20|0.00|450,00|1|5|

## <a name="wip-inventory-revaluation" />Переоценка запасов НЗП
Переоценка запасов НЗП подразумевает переоценку компонентов, которые зарегистрированы как часть запасов НЗП на момент переоценки.  

Помня это, важно установить правила, когда товар считается частью НЗП на складе с финансовой точки зрения. В [!INCLUDE[prod_short](includes/prod_short.md)] существуют следующие соглашения:  

-   Приобретенный компонент становится частью запасов сырья с момента учета покупки как покупки, по которой выставлен счет.  
-   Приобретенный компонент/подкомпонент становится частью запасов НЗП с момента учета его потребления в соответствии с производственным заказом.  
-   Приобретенный компонент/подкомпонент остается частью запасов НЗП до момента выставления счета по производственному заказу (произведенному товару).  

Способ задания даты оценки для операции стоимости потребления следует тем же правилам, что и для запасов НЗП. Дополнительные сведения см. в разделе [Определение влияния переоценки на расход склада](design-details-revaluation.md#determining-whether-an-inventory-decrease-is-affected-by-revaluation).  

Запасы НЗП можно переоценить при условии, что дата переоценки — не позже, чем дата учета соответствующих операций журнала товаров типа "Потребление", а счет за соответствующий производственный заказ еще не выставлен.  

> [!CAUTION]  
>  Отчет **Оценка стоимости запасов — НЗП** позволяет сравнить стоимость операций производственного заказа, что может несколько осложнять работу с товарами НЗП, которые были переоценены.
