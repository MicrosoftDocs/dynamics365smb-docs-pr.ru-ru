# <a name="the-vat-group-management-extension"></a>Расширение для управления группой НДС

Вы можете присоединиться к одному или нескольким предприятиям в вашей стране, чтобы объединить отчетность по НДС под одним регистрационным номером. Такой тип расположения известен как *Группа НДС*. Вы можете взаимодействовать с группой в качестве участника или представителя группы.

## <a name="forming-a-vat-group"></a>Формирование группы НДС
Участники группы НДС и представитель группы могут использовать мастер настройки **Настройка управления группой НДС**, чтобы определить их взаимодействие с группой и установить связь между их клиентами [!INCLUDE[prod_short](includes/prod_short.md)]. Участники группы будут использовать соединение для подачи декларации по НДС представителю группы. Представитель будет подавать НДС в налоговые органы от имени группы, используя единую декларацию по НДС. 

## <a name="vat-group-constellations"></a>Созвездия групп НДС
Общение происходит от участников группы к представителю. Представитель группы предоставляет API, который позволяет участникам подавать свои декларации по НДС представителю группы НДС. 
[!INCLUDE[prod_short](includes/prod_short.md)] поддерживает подачу межгрупповых деклараций по НДС для компаний, использующих [!INCLUDE[prod_short](includes/prod_short.md)] локально или в Интернете и в любой комбинации. Настройка связи зависит от созвездия. В следующих разделах описывается, как создавать различные созвездия групп.

В следующем списке показан рекомендуемый порядок шагов для создания группы НДС:

1. Создайте настройку в Azure Active Directory. Дополнительные сведения см. в разделе [Требования к аутентификации](ui-extensions-vat-group.md#requirements-for-authentication).
2. Поделитесь технической информацией, которая нужна участникам группы НДС и представителю для подключения их клиентов [!INCLUDE[prod_short](includes/prod_short.md)]. Обычно представитель группы НДС имеет эту информацию, такую как название среды представителя группы НДС, в которую участники группы НДС будут подавать НДС.
3. Создать пользователей в [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС, которые участники группы НДС могут использовать для аутентификации и подключения.
4. Запустите мастер настройки **Настройка управления группой зачета НДС** для подключения участников группы НДС.

  Для руководства требуется некоторая информация от представителя группы НДС. Для получения дополнительной информации см. раздел [Настройка участника группы НДС](#vat-group-member-setup). Обратите внимание на **ИД участника группы** для каждого участника группы НДС. Эти идентификаторы нужны представителю, чтобы добавить компании в группу НДС.
5. Настройте расширение управления группой НДС в [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС, используя мастер настройки **Настройка управления группой НДС**. 

> [!NOTE]
> Чтобы подключиться к представителю группы НДС, участникам группы необходим пользователь с доступом к [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС. Представитель группы НДС должен создать для этого хотя бы одного пользователя, однако из соображений безопасности мы рекомендуем вам создать пользователя для каждого участника группы НДС. Убедитесь, что учетные данные этих пользователей распространяются среди участников группы НДС безопасным способом.

## <a name="understanding-the-vat-group-management-setup"></a>Общие сведения о настройке управления группой НДС

Представитель группы НДС представляет API, который участники группы НДС могут использовать для подключения к своему клиенту [!INCLUDE[prod_short](includes/prod_short.md)] и подать декларацию по НДС. Участники группы НДС часто используют [!INCLUDE[prod_short](includes/prod_short.md)] в отдельных клиентах Azure Active Directory. Следовательно, требуется дополнительная настройка, чтобы установить связь между участником группы НДС и [!INCLUDE[prod_short](includes/prod_short.md)] представителя . 
  
Участники группы НДС подключаются к представителю, позвонив в веб-службу клиента представителя группы НДС. Вызывающий должен быть аутентифицирован с помощью OAuth. При настройке расширения для управления группами НДС вам будет предложено пройти аутентификацию у представителя группы НДС, чтобы получить и сохранить токен доступа. Этот токен доступа используется при подаче декларации по НДС представителю группы НДС. 

Аутентификацией занимается Azure Active Directory, поэтому настройки должны быть выполнены в Azure Active Directory представителя группы НДС, чтобы разрешить соединения. Регистрация приложения Azure Active Directory должна быть настроена с доступом к представителю группы НДС [!INCLUDE[prod_short](includes/prod_short.md)]. Это также верно, если представитель группы НДС использует [!INCLUDE[prod_short](includes/prod_short.md)] локально. Кроме того, необходимо настроить систему единого входа, если представитель группы НДС использует [!INCLUDE[prod_short](includes/prod_short.md)] локально.

> [!NOTE]
> В течение ограниченного времени также поддерживается аутентификация с использованием ключа доступа к веб-сервису. Для получения дополнительной информации см. [Устаревшие функции выпуска 2021, волна 1](/dynamics365/business-central/dev-itpro/upgrade/deprecated-features-w1#deprecated-features-in-2021-release-wave-1).

## <a name="requirements-for-authentication"></a>Требования к аутентификации 
Когда представитель группы НДС использует [!INCLUDE[prod_short](includes/prod_short.md)] онлайн или локально, участники группы НДС используют Azure Active Directory для аутентификации при подаче декларации по НДС представителю группы НДС. Если участники группы НДС также используют [!INCLUDE[prod_short](includes/prod_short.md)] онлайн, участник группы НДС может пройти аутентификацию, используя указанные учетные данные пользователя и информацию о конечной точке. Для получения дополнительной информации см. [Настройка участника группы НДС](ui-extensions-vat-group.md#vat-group-member-setup).

Участники группы НДС с локальным [!INCLUDE[prod_short](includes/prod_short.md)] должны настроить регистрацию приложения в Azure Active Directory для клиента [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС. Регистрация в приложении позволит [!INCLUDE[prod_short](includes/prod_short.md)] (онлайн) представителя группы НДС аутентифицировать участника группы. Дополнительные сведения см. в [Быстрый старт: зарегистрируйте приложение на платформе идентификации Microsoft](/azure/active-directory/develop/quickstart-register-app).

При создании регистрации приложения в Azure Active Directory необходимо указать следующее.

* В разделе **Аутентификация** добавьте **Интернет** в качестве платформы и используйте следующий **URL-адрес перенаправления**: https://businesscentral.dynamics.com/OAuthLanding.htm.
* В разделе **Аутентификация** раздел для **Поддерживаемые типы учетных записей** выберите **Учетные записи в любом каталоге организации (любой каталог Azure AD — мультитенатность)**.
* В разделе **Сертификаты и секреты** создайте новый секрет клиента и обратите внимание на значение. Участникам группы НДС нужен секрет для настройки подключения к представителю группы.
* В разделе **Разрешения API** добавьте разрешения для [!INCLUDE[prod_short](includes/prod_short.md)]. Разрешите делегированный доступ к **Financials.ReadWrite.All** и **user_impersonation**.
* В разделе **Обзор** обратите внимание на **ИД приложения (клиента)**. Участникам группы НДС нужен ИД для настройки подключения к представителю группы. 

## <a name="vat-group-member-setup"></a>Настройка участника группы НДС
Настройте участника группы НДС, запустив мастер настройки **Настройка управления группой зачета НДС**. 

> [!NOTE]
> Прежде чем начать, свяжитесь с представителем группы НДС, чтобы получить следующую информацию об их клиенте [!INCLUDE[prod_short](includes/prod_short.md)]:
>
> * URL-адрес API
> * Название организации 
> * ИД клиента
> * Секрет клиента
> * Войдите в учетные данные для выделенного пользователя 

1. Чтобы определить роль группы НДС компании, выберите **Участник**, а затем выберите **Далее**.
2. Скопируйте значение поля **ИД участника группы**, а затем отправьте его представителю группы НДС, чтобы он мог добавить вашу компанию в качестве утвержденного участника группы.
3. Добавьте **URL-адрес API** от представителя группы НДС. Обычно URL-адрес имеет следующий формат: `https://api.businesscentral.dynamics.com/v2.0/[TENANT-ID]/[ENVIRONMENTNAME]`. Например, `https://api.businesscentral.dynamics.com/v2.0/907869c3-b252-4aca-b9cb-17a15d25477b/UKRepresentative`. 
4. Добавьте название компании [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС, например *CRONUS UK Ltd*.
5. Выберите **Тип аутентификации**, выберите **OAuth2**, а затем выберите **Далее**.
6. В поле **ИД клиента** введите идентификатор, предоставленный представителем группы НДС.
7. В поле **Секрет клиента, предоставленный представителем группы НДС** введите секрет, предоставленный представителем группы НДС.
8. В поле **Конечная точка авторизации OAuth 2.0** введите `https://login.microsoftonline.com/common/oauth2`.
9. В поле **URL-адрес ресурса OAuth 2.0** введите `https://api.businesscentral.dynamics.com/`.
10. В поле **URL-адрес перенаправления OAuth 2.0** введите `https://businesscentral.dynamics.com/OAuthLanding.htm`. 
11. Когда вы указали различные поля, выберите **Далее**, а затем введите учетные данные пользователя, предоставленные представителем группы НДС.
12. Выберите конфигурацию отчета по НДС, которую вы используете для сообщения НДС налоговому органу в вашей стране.

  Например, в Соединенном Королевстве конфигурация отчета по НДС будет настроена так, чтобы сообщать НДС в HMRC. Расширение управления группой НДС копирует эту настройку, но заменяет codeunit отправки кодом, который поддерживает отправку представителю группы НДС, а не налоговым органам. codeunit предоставляется Microsoft. По завершении нажмите **Далее**.

## <a name="using-the-vat-group-management-features"></a>Использование функций управления группой НДС

Участники группы НДС используют стандартные процессы для подготовки деклараций по НДС. Единственная разница — выбрать версию отчета **VATGROUP**, которая отправляет декларацию по НДС представителю группы НДС, а не налоговым органам. Дополнительные сведения см. в разделе [Об отчете о возврате НДС](finance-how-report-vat.md#vatreturn).

В следующих разделах описываются задачи, которые должны выполнять представители группы НДС.

### <a name="vat-group-submissions"></a>Подачи отчетов в группу зачета НДС

На странице **Подачи отчетов в группу зачета НДС** перечислены декларации по НДС, представленные участниками. Эта страница служит местом для черновиков отправки, пока представитель группы НДС не включит их в декларацию по НДС для группы. Вы можете открыть заявки, чтобы просмотреть НДС для отдельных полей, указанных участником группы НДС.

### <a name="creating-a-group-vat-return"></a>Создание групповой декларации по НДС

Чтобы сообщить НДС от имени группы, на странице **Декларации по НДС** создайте декларацию по НДС только для вашей компании. После этого включите самые последние представления НДС от участников группы НДС, выбрав действие **Включить групповой НДС**.  

Когда декларация по НДС представителя группы НДС подана властям от имени всей группы, вы обычно запускаете действие **Вычисление и учет зачета НДС**. Это действие закрывает открытые операции НДС и переводит суммы на расчетный счет НДС. В настоящее время это действие не учитывает отправления групп. Это означает, что будут проводиться только записи НДС компании-представителя группы НДС. Суммы, представленные участником группы НДС, должны быть разнесены на сумму расчета НДС вручную, чтобы на счету расчетов по НДС представителя группы НДС отражалось обязательство по тому, о чем было сообщено властям. Это поведение изменится в следующем обновлении [!INCLUDE[prod_short](includes/prod_short.md)], поэтому будет рассчитан весь групповой НДС (Общая сумма в строках отчета в декларации НДС). 

> [!NOTE]
> Участники группы НДС могут исправлять поданные декларации по НДС до тех пор, пока представитель группы не выпустил декларацию по НДС для группы. Чтобы внести исправление, участник группы НДС должен создать новую декларацию по НДС за период декларации НДС и отправить ее представителю группы НДС. Со стороны представителя группы НДС последняя декларация по НДС будет включена на страницу **Декларации по НДС**. 

> [!IMPORTANT]
> Функциональность группы НДС поддерживается только на тех рынках, где [!INCLUDE[prod_short](includes/prod_short.md)] использует структуру НДС, которая состоит из налоговых деклараций и периодов возврата НДС. Вы не можете использовать группы НДС на других рынках, где есть другие реализации местной отчетности по НДС, например в Австрии, Германии, Италии, Испании и Швейцарии.
