# <a name="the-vat-group-management-extension-for-the-united-kingdom"></a>Расширение управления группой НДС для Соединенного Королевства

Вы можете присоединить одно или несколько предприятий в вашей стране, чтобы объединить отчетность по НДС под одним регистрационным номером. Такой тип расположения известен как *Группа НДС*. Вы можете взаимодействовать с группой в качестве участника или представителя группы.

## <a name="forming-a-vat-group"></a>Формирование группы НДС
Участники группы НДС и представитель группы могут использовать мастер настройки **Настройка управления группой НДС**, чтобы определить их взаимодействие с группой и установить связь между их клиентами [!INCLUDE[prod_short](includes/prod_short.md)]. Участники группы будут использовать соединение для подачи декларации по НДС представителю группы. Представитель будет использовать единую декларацию по НДС для подачи в налоговые органы от имени всей группы. 

## <a name="license-requirements"></a>Требования к лицензиям
Участники группы должны иметь лицензию на использование [!INCLUDE[prod_short](includes/prod_short.md)]. Вы не можете использовать гостевые учетные записи в группах НДС. 

* Для расчета и подачи деклараций по НДС пользователь должен быть полноправным пользователем Business Central.
* Для входа в систему и выполнения основных задач, таких как создание учетных записей, вы можете иметь лицензию участника команды Dynamics 365 Business Central.

## <a name="vat-group-constellations"></a>Созвездия групп НДС
Общение происходит от участников группы к представителю. Представитель группы предоставляет URL-адрес API, который позволяет участникам подавать свои декларации по НДС представителю группы НДС. 
[!INCLUDE[prod_short](includes/prod_short.md)] поддерживает подачу межгрупповых деклараций по НДС для компаний, использующих [!INCLUDE[prod_short](includes/prod_short.md)] локально или в Интернете и в любой комбинации. Настройка связи зависит от созвездия. В этой статье описаны различные созвездия групп.

В следующем списке показан рекомендуемый порядок шагов для создания группы НДС:

1. Создайте настройку в Azure Active Directory. Дополнительные сведения см. в разделе [Требования к аутентификации](ui-extensions-vat-group.md#requirements-for-authentication).
2. Поделитесь технической информацией, которая нужна участникам группы НДС и представителю для подключения их клиентов [!INCLUDE[prod_short](includes/prod_short.md)]. Обычно представитель группы НДС имеет эту информацию, такую как название среды представителя группы НДС, в которую участники группы НДС будут подавать НДС.
3. Создайте пользователей, которых участники группы НДС могут использовать для аутентификации при подключении к [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС. У пользователей должна быть полная лицензия пользователя на [!INCLUDE[prod_short](includes/prod_short.md)].
4. Запустите мастер настройки **Настройка управления группой зачета НДС** для подключения участников группы НДС.

  Для руководства требуется некоторая информация от представителя группы НДС. Для получения дополнительной информации см. [Настройка участников группы НДС](#set-up-vat-group-members). Обратите внимание на **ИД участника группы** для каждого участника группы НДС. Эти идентификаторы нужны представителю, чтобы добавить компании в группу НДС.
5. Настройте расширение управления группой НДС в [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС, используя мастер настройки **Настройка управления группой НДС**. 

> [!NOTE]
> Чтобы подключиться к представителю группы НДС, участники группы должны иметь учетную запись пользователя с возможностью доступа к [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС. Представитель группы НДС должен создать для этого хотя бы одного пользователя, однако из соображений безопасности мы рекомендуем вам создать пользователя для каждого участника группы НДС. Убедитесь, что учетные данные этих пользователей распространяются среди участников группы НДС безопасным способом. Это может быть системная учетная запись пользователя, которая не связана с реальным человеком.

## <a name="about-the-vat-group-management-setup"></a>О настройке управления группой НДС

Представитель группы НДС предоставляет API участникам группы. Участники используют API для подключения к клиенту [!INCLUDE[prod_short](includes/prod_short.md)] представителя и подачи деклараций по НДС. Участники группы НДС часто используют [!INCLUDE[prod_short](includes/prod_short.md)] в отдельных клиентах Azure Active Directory. Следовательно, требуется настройка, чтобы установить связь между участником группы НДС и [!INCLUDE[prod_short](includes/prod_short.md)] представителя. 
  
Участники группы НДС подключаются к представителю, вызывая веб-службу клиента представителя группы НДС. Вызывающий должен быть аутентифицирован с помощью OAuth2. При настройке расширения для управления группами НДС вам будет предложено пройти аутентификацию у представителя группы НДС, чтобы получить и сохранить токен доступа. Этот токен доступа используется при подаче декларации по НДС представителю группы НДС. 

## <a name="construct-the-api-url"></a>Создание URL-адреса API
1. В центре администрирования Business Central для клиента представителя выберите вкладку **Среды**.
2. В разделе **Сведения** скопируйте **URL-адрес**.
3. Откройте Блокнот и вставьте URL-адрес. Замените **https://businesscentral.dynamics.com** на **https://api.businesscentral.dynamics.com/v2.0**.

## <a name="requirements-for-authentication"></a>Требования к аутентификации 
> [!NOTE]
> Для этого требуются учетные данные для учетной записи администратора, которая имеет полную лицензию пользователя для [!INCLUDE[prod_short](includes/prod_short.md)].

Когда представитель группы НДС использует [!INCLUDE[prod_short](includes/prod_short.md)] Online или локальную версию, участники группы НДС должны использовать Azure Active Directory для аутентификации пользователей при подаче декларации по НДС представителю группы НДС. Для локальной версии [!INCLUDE[prod_short](includes/prod_short.md)] участники должны настроить единый вход. Для получения дополнительной информации см. раздел [Настройка аутентификации Azure Active Directory с WS-Federation](/dynamics365/business-central/dev-itpro/administration/authenticating-users-with-azure-active-directory?tabs=singletenant%2Cadmintool). Если участники группы НДС также используют [!INCLUDE[prod_short](includes/prod_short.md)] онлайн, участник группы НДС может пройти аутентификацию, используя указанные учетные данные пользователя и информацию о конечной точке. Для получения дополнительной информации см. [Настройка участников группы НДС](ui-extensions-vat-group.md#set-up-vat-group-members).

Участники группы НДС с локальным [!INCLUDE[prod_short](includes/prod_short.md)] должны настроить регистрацию приложения в Azure Active Directory для клиента [!INCLUDE[prod_short](includes/prod_short.md)] представителя группы НДС. Регистрация в приложении позволяет [!INCLUDE[prod_short](includes/prod_short.md)] Online представителя группы НДС аутентифицировать участника группы. Дополнительные сведения см. в [Быстрый старт: зарегистрируйте приложение на платформе идентификации Microsoft](/azure/active-directory/develop/quickstart-register-app).

При создании регистрации приложения в Azure Active Directory необходимо указать следующую информацию.

* В разделе **Аутентификация** добавьте **Интернет** в качестве платформы и используйте следующий **URL-адрес перенаправления**: https://businesscentral.dynamics.com/OAuthLanding.htm.
* В разделе **Аутентификация** раздел для **Поддерживаемые типы учетных записей** выберите **Учетные записи в любом каталоге организации (любой каталог Azure AD — мультитенатность)**.
* В разделе **Сертификаты и секреты** создайте новый секрет клиента и обратите внимание на значение. Участникам группы НДС нужен секрет для настройки подключения к представителю группы.
* В разделе **Разрешения API** добавьте разрешения для [!INCLUDE[prod_short](includes/prod_short.md)]. Разрешите делегированный доступ к **Financials.ReadWrite.All** и **user_impersonation**.
* В разделе **Обзор** обратите внимание на **ИД приложения (клиента)**. Участникам группы НДС нужен ИД для настройки подключения к представителю группы. 

## <a name="set-up-vat-group-members"></a>Настройка участников группы НДС
> [!IMPORTANT]
> Компаниям-участникам группы НДС не нужно подключаться к HMRC, поскольку они отчитываются через представителя группы.

Прежде чем начать, свяжитесь с представителем группы НДС, чтобы получить следующую информацию об их клиенте [!INCLUDE[prod_short](includes/prod_short.md)]:

* URL-адрес API
* Название организации 
* ИД клиента
* Секрет клиента
* Войдите в учетные данные для выделенного пользователя 

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Настройка управления группами НДС**, затем выберите соответствующую ссылку.
2. В поле **Роль группы НДС** укажите, являетесь ли вы участником группы или представителем группы. Выберите **Участник**, чтобы выступать в качестве участника группы и подавать отчеты по НДС представителю группы, а не в налоговый орган, затем выберите **Далее**.
3. Скопируйте значение поля **ИД участника группы**, а затем отправьте его представителю группы НДС, чтобы он мог добавить вашу компанию в качестве утвержденного участника группы.
4. В поле **Версия продукта представителя группы** укажите версию [!INCLUDE[prod_short](includes/prod_short.md)], которой пользуется представитель.
5. В поле **Компания-представитель группы** введите наименование компании представителя группы НДС. Например, **CRONUS UK Ltd**.
6. В поле **Тип аутентификации** выберите **OAuth2**. Если представитель группы НДС использует [!INCLUDE[prod_short](includes/prod_short.md)] Online, укажите, что **Представитель группы использует Business Central Online**, затем выберите **Далее**. В зависимости от того, использует ли представитель [!INCLUDE[prod_short](includes/prod_short.md)] Online или локальную версию, следуйте инструкциям в разделе [Представитель группы НДС использует Business Central Online](ui-extensions-vat-group.md#vat-group-representative-uses-business-central-online) или [Представитель группы НДС использует локальную версию Business Central On-Premesis](ui-extensions-vat-group.md#vat-group-representative-uses-business-central-on-premesis).

### <a name="vat-group-representative-uses-business-central-online"></a>Представитель группы НДС использует Business Central Online
1. Введите учетные данные пользователя, предоставленные представителем группы НДС, и добавьте необходимые разрешения.
2. Выберите конфигурацию отчета по НДС, которая в настоящее время используется для подачи деклараций по НДС в налоговые органы. После завершения настройки [!INCLUDE[prod_short](includes/prod_short.md)] создаст новую конфигурацию на основе этого выбора и будет использовать ее для отправки деклараций по НДС представителю группы НДС.  
3. Добавьте **URL-адрес API** от представителя группы НДС. Обычно URL-адрес имеет следующий формат: `https://api.businesscentral.dynamics.com/v2.0/[TENANT-ID]/[ENVIRONMENTNAME]`. Например, `https://api.businesscentral.dynamics.com/v2.0/907869c3-b252-4aca-b9cb-17a15d25477b/UKRepresentative`.

### <a name="vat-group-representative-uses-business-central-on-premesis"></a>Представитель группы НДС использует локальную версию Business Central On-Premesis
1. В поле **ИД клиента** укажите идентификатор клиента из регистрации приложения в Azure Active Directory.
2. В поле **Секрет клиента** укажите секрет клиента из регистрации приложения в Azure Active Directory.
3. В поле **Конечная точка авторизации OAuth 2.0** введите `https://login.microsoftonline.com/common/oauth2`.
4. В поле **URL-адрес ресурса OAuth 2.0** введите `https://api.businesscentral.dynamics.com/`.
5. В поле **URL-адрес перенаправления OAuth 2.0** введите `https://businesscentral.dynamics.com/OAuthLanding.htm`.
6. Когда вы указали различные поля, выберите **Далее**, а затем введите учетные данные пользователя, предоставленные представителем группы НДС.
7. Выберите конфигурацию отчета по НДС, которую вы используете для сообщения НДС налоговому органу в вашей стране.

## <a name="set-up-the-vat-group-representative"></a>Настройка представителя группы НДС
> [!NOTE]
> Для локальной версии поддерживается только один экземпляр клиента представителя группы.

> [!IMPORTANT]
> Компания-представитель должна включить подключение к службе **Настройка НДС HMRC** на странице **Подключения к службам**. Представители также должны загрузить периоды подачи деклараций по НДС из HMRC.

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Настройка управления группами НДС**, затем выберите соответствующую ссылку.
2. В поле **Роль группы НДС** укажите, являетесь ли вы участником группы или представителем группы. Выберите **Представитель**, чтобы выступать в качестве представителя группы НДС и подавать отчеты по НДС за группу в налоговый орган, затем выберите **Далее**.
3. В поле **Счет зачета НДС** укажите счет, который вы используете для зачета НДС.
4. В поле **№ поля НДС к уплате** укажите поле, представляющее итоговую сумму НДС к уплате из отчета, поданного группой НДС.
5. В поле **Шаблон финансового журнала зачета по группе** укажите шаблон общего журнала, используемый для документа проводки НДС группы по счету зачета.
6. Поле **Утвержденные участники** показывает количество участников группы, настроенных для подачи декларации по НДС представителю. Чтобы добавить новых участников, нажмите это количество, чтобы открыть страницу **Утвержденные участники**.
7. Чтобы добавить новых участников, добавьте следующую информацию на страницу **Утвержденные участники группы зачета НДС**:
    1. В поле **ИД участника группы** введите идентификатор для участника группы.
    2. В поле **Название участника группы** укажите имя участника группы. 
    3. В поле **Организация** укажите компанию, из которой участник группы будет отправлять отчеты по НДС в [!INCLUDE[prod_short](includes/prod_short.md)]. Например, **CRONUS UK Ltd**.
    4. Укажите контактные данные для организации.

## <a name="use-the-vat-group-management-features"></a>Использование функций управления группой НДС
Участники группы НДС используют стандартные процессы для подготовки деклараций по НДС. Единственная разница — выбрать версию отчета **VATGROUP**, которая отправляет декларацию по НДС представителю группы НДС, а не налоговым органам. Дополнительные сведения см. в разделе [Об отчете о возврате НДС](finance-how-report-vat.md#vatreturn).

В следующих разделах описываются задачи, которые должны выполнять представители группы НДС.

### <a name="vat-group-submissions"></a>Подачи отчетов в группу зачета НДС

На странице **Подачи отчетов в группу зачета НДС** перечислены декларации по НДС, представленные участниками. Эта страница служит местом для черновиков отправки, пока представитель группы НДС не включит их в декларацию по НДС для группы. Вы можете открыть заявки, чтобы просмотреть НДС для отдельных полей, указанных участником группы НДС. 

> [!TIP]
> На странице **Периоды возврата НДС** поле **Подачи отчетов участников группы** показывает, сколько отчетов подали участники. Чтобы убедиться, что этот номер актуален, выберите действие **Получить периоды возврата НДС**.

### <a name="creating-a-group-vat-return"></a>Создание групповой декларации по НДС

Чтобы подать декларацию по НДС для группы, на странице **Декларации по НДС** создайте декларацию по НДС только для вашей компании. После этого включите самые последние представления НДС от участников группы НДС, выбрав действие **Включить групповой НДС**.  

Когда декларация по НДС представителя группы НДС подана в налоговый орган для всей группы, вы обычно запускаете действие **Вычисление и учет зачета НДС**. Это действие закрывает открытые операции НДС и переводит суммы на расчетный счет НДС. В настоящее время это действие не учитывает отправления групп. <!--Has this changed?--> Будут проводиться только записи НДС компании-представителя группы НДС. Суммы, представленные участником группы НДС, должны быть разнесены на сумму расчета НДС вручную, чтобы на счету расчетов по НДС представителя группы НДС отражалось обязательство по тому, о чем было сообщено властям. Это поведение изменится в следующем обновлении [!INCLUDE[prod_short](includes/prod_short.md)], поэтому будет рассчитан весь групповой НДС (Общая сумма в строках отчета в декларации НДС). <!--Has this behavior changed?-->

> [!NOTE]
> Участники группы НДС могут исправлять поданные декларации по НДС до тех пор, пока представитель группы не выпустил декларацию по НДС для группы. Чтобы внести исправление, участник группы НДС должен создать новую декларацию по НДС за период декларации НДС и отправить ее представителю группы НДС. Со стороны представителя группы НДС последняя декларация по НДС будет включена на страницу **Декларации по НДС**. 

> [!IMPORTANT]
> Функциональность группы НДС поддерживается только на тех рынках, где [!INCLUDE[prod_short](includes/prod_short.md)] использует структуру НДС, которая состоит из налоговых деклараций и периодов возврата НДС. Вы не можете использовать группы НДС на других рынках, где есть другие реализации местной отчетности по НДС, например в Австрии, Германии, Италии, Испании и Швейцарии.
