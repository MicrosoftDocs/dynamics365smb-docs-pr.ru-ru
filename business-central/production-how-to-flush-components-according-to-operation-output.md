# <a name="flush-components-according-to-operation-output" />Списание компонентов в соответствии с производственным выпуском
Вы можете определить различные стратегии списания, чтобы автоматизировать регистрацию расхода компонентов. 

Эта функция полезна по следующим причинам:  

- **Оценка стоимости товаров**

    Операции стоимости для выхода и потребления создаются параллельно по мере выполнения производственного заказа. Без кодов связи маршрута инвентарная стоимость увеличивается по мере учета выхода, а затем одномоментно уменьшается, учитывается потребление компонентов вместе с завершенным производственным заказом.  
- **Запасы - наличие**

    При постепенном учете потребления сведения о наличии компонентов более точные, что важно для поддержания внутреннего баланса между спросом и предложением. Без кодов связи маршрута при проч. запросах компонента может считаться, что он доступен, при том, что он находится на удержании задержанного учета потребления.  
- **Точно вовремя**

    Благодаря возможности настроить продукты согласно запросам клиента можно сократить потери за счет того, что изменения работы и системы происходят только при необходимости.  

- **Уменьшить ввод данных**

    Возможность автоматически списывать операцию позволяет автоматизировать весь процесс регистрации потребления и выпуска продукции. Минус использования автоматического списания заключается в том, что регистрация отходов и брака может оказаться неточной, либо будет отсутствовать вообще.

## <a name="automatic-consumption-posting-flushing-methods" />Методы автоматического учета (списание) потребления

- Прямое списание всего заказа  
- Прямое списание по одной операции  
- Обратное списание по одной операции  
- Обратное списание всего заказа  

### <a name="automatic-reporting---forward-flush-the-entire-order" />Автоматическая отчетность — прямое списание всего заказа
При прямом списании производственного заказа в начале выполнения задания поведение приложения очень похоже на ручное потребление. Основное отличие в том, что потребление происходит автоматически.  

- Все содержимое производственной спецификации потребляется и списывается со склада в момент обновления запущенного производственного заказа.  
- Количество потребления равно количеству по каждой сборке, указанному в производственной спецификации, умноженному на количество изготавливаемых родительских товаров.  
- Нет необходимости регистрировать какую-либо информацию в журнале потребления, если списываются все товары.  
- При потреблении товаров со склада не важно, когда делаются записи в журнале выпуска продукции, поскольку журнал выпуска продукции не влияет на этот режим учета потребления.  
- Коды маршрутных ссылок задать невозможно.  

Прямое списание всего заказа в производственных средах обеспечивает:  

-   Низкий уровень брака  
-   Малое число операций  
-   Высокое потребление компонентов на ранних операциях  

### <a name="automatic-reporting---forward-flushing-by-operation" />Автоматическая отчетность — прямое списание по одной операции
Списание по одной операции позволяет выполнять списание со склада во время конкретной операции на маршруте родительского товара. "Материал" привязан к маршруту с помощью кодов маршрутных ссылок, которые соответствуют кодам маршрутных ссылок, применяемых к компонентам в производственной спецификации.  

Списание происходит в момент начала операции с таким же кодом маршрутной ссылки. Начало означает регистрацию некоторых действий в журнале выпуска для данной операции. Таким действием может оказаться ввод времени настройки.  

Количество списания равно количеству по сборке, указанному в производственной спецификации, умноженному на количество создаваемых родительских товаров (ожидаемое количество).  

Этот метод предпочтителен при наличии множества операций, а некоторые компоненты потребуются только на поздних этапах сборки. Фактически, в настройке JIT могут даже отсутствовать в наличии товары, когда начинается выполнение RPO.  

Потребление материала может происходить во время операций путем использования кодов маршрутных ссылок. Некоторые компоненты могут использоваться только в ходе заключительных операций сборки, поэтому не должны списываться со склада до этого момента.  

### <a name="automatic-reporting---back-flushing-by-operation" />Автоматическая отчетность — обратное списание по одной операции
Обратное списание по одной операции регистрирует потребление после учета операции в журнале выпуска.  

Преимущество этого метода в том, что известно количество родительских товаров, завершенных в ходе операции.  

Материал в производственной спецификации связан с маршрутными записями посредством кодов маршрутных ссылок. Обратное списание происходит в момент учета операции с конкретным кодом маршрутной ссылки, для которой зарегистрировано завершенное количество.  

Объем списания соответствует количеству по одной сборке, указанному в производственной спецификации, умноженному на количество родительских товаров, учтенных для данной операции как количество выхода. Это значение может отличаться от ожидаемого количества.  

### <a name="automatic-reporting---back-flushing-the-entire-order" />Автоматическая отчетность — обратное списание всего заказа
Этот метод отчетности не учитывает коды маршрутных ссылок.  

Компоненты не выбираются до того момента, когда статус запущенного производственного заказа не изменится на *«Завершен»*. Объем списания равен количеству по одной сборке, указанному в производственной спецификации, умноженному на количество родительских товаров, завершенных и помещенных на склад.  

При обратном списании для всего производственного заказа требуется такая же настройка, как и при прямом списании: в каждой карточке товара должен быть указан обратный метод отчетности для всех товаров родительской спецификации, включаемых в отчет. Кроме того, из производственной спецификации необходимо удалить все коды связей маршрутов. 



Например, если для заказа на производство 800 метров требуется 8 кг компонента, то при указании 200 метров на выходе, 2 кг автоматически учитываются как потребление. Вы можете достичь этого, объединив метод обратного списания и коды маршрутных ссылок, чтобы количество, списываемое за одну производственную операцию, было пропорционально фактическому выходу завершенной операции. Для товаров, в которых применяется обратный метод списания, поведением по умолчанию является вычисление потребления компонентов во время изменения состояния запущенного производственного заказа на значение **Завершено**. Если задаются также коды связей маршрута, по окончании каждой операции происходит расчет и учет, и учитывается количество, которое было фактически использовано в операции. Дополнительные сведения см. в разделе [Создание маршрутов](production-how-to-create-routings.md).  

## <a name="to-flush-components-according-to-operation-output" />Списание компонентов в соответствии с производственным выпуском

1.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок введите **Товары**, а затем выберите связанную ссылку.  
2.  Выберите действие **Изменить**.  
3.  На экспресс-вкладке **Пополнение** в поле **Метод списания** выберите **Обратный**.  

    > [!NOTE]  
    >  Выберите **Подбор + обратный**, если компонент используется на данном складе, который настроен расширенного подбора и размещения.  

4.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Маршруты**, а затем выберите связанную ссылку.  
5.  Определите коды связи маршрута для каждой операции, которая потребляет компонент. Дополнительные сведения см. в разделе [Создание маршрутов](production-how-to-create-routings.md).  
    > [!IMPORTANT]  
    > Не используйте одну и ту же ссылку маршрутизации для разных операций в маршрутизации, так как это приведет к регистрации потребления компонента для каждой связанной операции.  
6.  Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Производственная спецификация**, а затем выберите связанную ссылку.  
7.  Определите коды связи маршрута от каждого экземпляра компонента к операции, где он потребляется.

Потребление будет учтено автоматически при регистрации выпуска. Дополнительные сведения см. разделе [Учет в пакетном режиме выпуска продукции и времени работы](production-how-to-post-output-quantity.md)

## <a name="flushing-methods" />Методы списания

В следующей таблице описаны доступные варианты методов списания, которые можно указать на карточках **Товар** и **Единица хранения**.

|Параметр|Описание|
|------------|-------------|  
|Ручной|Требуется вручную ввести потребление в журнал потребления и выполнить его учет.|
|Прямой|Автоматически учитывает учитывает потребление в соответствии со строками компонента производственного заказа. <br><br>По умолчанию учет потребления компонентов имеют место при изменении статуса производственного заказа на **Выпущено**. Если вы используете поле **Код связи маршрута** в строках компонентов производственного заказа, учет производится для каждой операции на момент запуска операции. Дополнительные сведения см. в разделе [Практическое руководство. Создание связей маршрута](production-how-to-create-routings.md#to-create-routing-links). <br><br> **Примечание**<br>При прямом списании учет конкретной операции, который получен с кодами связи маршрута, основан на ожидаемом количестве, которое указано в строке компонента. Дополнительные сведения о списании конкретной операции на основе фактического выхода см. в описании **Обратного** списания в этом разделе.<br><br>Если в настройках склада или ресурсов, которые расходует данный компонент, указана структура ячейки по умолчанию, товар расходуется из **Кода ячейки комплектования**. Дополнительные сведения см. в разделе [Практическое руководство. Настройка базовых складов с помощью зон операций](warehouse-how-to-set-up-basic-warehouses-with-operations-areas.md). <br><br> **Важно!** <br>Прямое списание происходит также после выбора **Обновить** в запущенном производственном заказе, созданном с нуля. В этих непосредственно созданных осуществленных производственных заказах нельзя изменить информацию о ячейках, поскольку строки компонента производственного заказа генерируются при обновлении заказа, и при этом списываются компоненты. Следовательно, если требуется изменить данные ячейки в строках компонентов производственного заказа до выполнения прямого списания, этот заказ должен быть создан со статусом *Запланирован* или *Утвержден*.|
|Обратный|Автоматически вычисляет и учитывает потребление в соответствии со строками компонента производственного заказа.<br><br> По умолчанию расчет и учет потребления компонентов имеют место при изменении статуса выпущенного производственного заказа на **Завершено**. Если вы используете поле **Код связи маршрута** в строках компонентов производственного заказа, вычисление и учет производятся, когда каждая операция завершена.<br><br> **Примечание** <br>Обратное списание и коды маршрутных ссылок могут объединяться таким образом, чтобы количество, списываемое за одну операцию, было пропорционально фактическому выходу данной операции. Дополнительные сведения см. в разделе [Практическое руководство. Списание компонентов в соответствии с производственным выпуском](#to-flush-components-according-to-operation-output).<br><br> Если в настройках склада или машинного центра, которые расходует данный компонент, указана структура ячейки по умолчанию, товар расходуется из **Кода ячейки комплектования**.|
|Подбор + прямой|То же, что и метод прямого списания, за исключением того, что он работает только для местоположений, использующих либо расширенную конфигурацию склада, либо базовую конфигурацию склада с обязательными ячейками.<br><br> Потребление вычисляется и учитывается из ячейки, которая определена в поле **Код ячейки, направл. в пр-во** на складе или машинном центре после того, как компонент подобран со склада.<br><br> **Примечание** <br>Если компонент настроен с помощью метода списания "Подбор + прямой", он не может иметь код связи маршрута с операцией, которая настроена с помощью метода прямого списания. Затем компонент будет автоматически списан при начале производственной операции, что не позволяет запросить подбор.|
|Подбор + обратный|То же, что и метод обратного списания, за исключением того, что он работает только для местоположений, использующих либо расширенную конфигурацию склада, либо базовую конфигурацию склада с обязательными ячейками.<br><br> Потребление вычисляется и учитывается из ячейки, которая определена в поле **Код ячейки, направл. в пр-во** на складе или машинном центре после того, как компонент подобран со склада.|
