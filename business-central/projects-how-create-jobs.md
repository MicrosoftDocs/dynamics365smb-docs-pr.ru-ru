# <a name="create-projects"></a>Создание проектов

Начиная новый проект, вы должны создать карточку проекта с интегрированными задачами проекта и строками планирования проекта, разделенными на два уровня.  

Первый уровень состоит из задач проекта. Для каждого проекта должна быть создана хотя бы одна задача проекта, поскольку весь учет ссылается на задачу проекта. Наличие хотя бы одной задачи проекта в проекте позволяет создавать строки планирования проекта и учитывать потребление по проекту.

Второй уровень состоит из строк планирования проекта, в которых подробно указываются используемые ресурсы, товары и различные расходы из главной книги.

Структура уровней позволяет поделить проект на более мелкие задачи и, таким образом, получить более конкретные сведения для бюджетирования, предложений и регистрации. Кроме того, она позволяет анализировать ход выполнения проекта. Например, вы можете отследить, соблюдаете ли вы назначенные этапы или вы добились ли вы ожидаемых результатов бюджета.

> [!TIP]
> Выберите действие **Создать проект** в ролевом центре **Менеджер проекта**, чтобы запустить мастер настройки, который поможет вам создать проект с интегрированными заданиями и строками планирования. Следующая процедура описывает выполнение этих шагов вручную. <!-- For an example of how to create a project manually, go to [Video: How to create a project in Dynamics 365 Business Central](https://www.youtube.com/watch?v=VqaPWr7BWmw).-->

## <a name="to-create-a-project-card"></a>Выставление счетов одному или нескольким клиентам за задачи проекта

Иногда сторона, получающая услугу, отличается от стороны, оплачивающей счет. Кроме того, иногда вам может потребоваться выставить счет нескольким клиентам за выполнение задач в проекте. На странице **Карточка проекта** используйте поле **Метод выставления счетов по задачам**, чтобы указать, выставляете ли вы счет одному клиенту или нескольким клиентам.

Если клиент, получающий услугу, также будет оплачивать счет, в полях **Плательщик** и **Получатель** выберите **По умолчанию (клиент)** и **По умолчанию (адрес покупателя)**.

Если вы выставляете счета нескольким клиентам, вы можете указать клиента, который получит услугу, и клиента, которому будет выставлен счет за каждую задачу в проекте. Можно также предоставить следующую информацию:

* Где будет выполняться работа, выбрав из списка адресов отгрузки для клиента.
* Добавьте информацию о внешних ссылках, чтобы упростить общение по проекту.
* Перезапишите стандартные финансовые условия проекта.

## <a name="to-create-tasks-for-a-project"></a>Выставление счета за несколько задач проекта одному клиенту

Вы можете упростить процесс выставления счетов, отправив клиенту один счет за несколько проектов. Вы можете за один раз добавить строки планирования проекта из нескольких проектов в счет продажи. Этот процесс аналогичен созданию счета продажи из строки планирования проекта и вводу значения в поле **Приложить к счету продажи номер**.

Вот обзор процесса.

1. Создайте новый счет продажи и заполните поле **№ клиента покупателя** . При необходимости также заполните **№ клиента плательщика** и **Код валюты**.
2. На экспресс-вкладке **Строки** выберите действие **Получить строки планирования проекта**. На странице **Получить строки планирования проекта** отображаются подлежащие оплате строки планирования проекта из проектов для клиента-покупателя, клиента-плательщика и валюты выставления счетов, где количество для выставления счета больше нуля. 
3. Выберите строки, которые вы хотите добавить в счет, а затем выберите **ОК**.

Повторите эти шаги, если хотите добавить еще один набор строк планирования проекта. Вы также можете удалить счет или его строки и начать все сначала.

> [!NOTE]
> Существует несколько ограничений:
>
> * Действие **Получить строки планирования проекта** недоступно в заказах на продажу или предложениях по продаже.
> * Невозможно фильтровать данные по полям **Код получателя** или **№ контакта** .

## <a name="invoice-one-or-more-customers-for-project-tasks"></a>Создание карточки проекта

Вы создаете карточку проекта, а затем создаете для нее строки задач проекта и строки планирования проекта.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, затем выберите соответствующую ссылку.  
2. Выберите действие **Создать**, затем введите в поля требуемые данные. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]
3. Чтобы создать проект на основе информации из другого проекта, выберите действие **Копировать проект**, заполните поля необходимым образом, а затем нажмите кнопку **ОК**.

> [!NOTE]  
> Если в проекте используется табели учета рабочего времени, то необходимо также указывать ответственное лицо. Это лицо может утверждать табели учета рабочего времени, затрачиваемого на выполнение сотрудниками задач, связанных с проектом. Дополнительные сведения см. в разделе [Настройка табелей учета рабочего времени](projects-how-setup-time-sheets.md).

При желании можно отметить действия с проектом как заблокированные с помощью поля **Заблокировано**. В следующей таблице приведено описание влияния параметров этого поля.

|Параметр  |Описание  |
|---------|---------|
|Пробел |Все действия разрешены.|
|Учет    |Можно работать со строками планирования, однако учет из проекта заблокирован. Этот вариант означает, что учитывать ни потребление, ни продажи в рамках проекта нельзя.|
|Все  |Все действия заблокированы.|

## <a name="specify-a-default-location-for-project-items"></a>Создание задач для проекта

Основной задачей при создании проекта является определение задач, входящих в состав проекта. Задачи определяются путем создания одной строки на задачу на экспресс-вкладке **Задачи** на странице **Карточка проекта**. Каждый проект должен содержать хотя бы одно задание.

1. Выберите значок ![Лампочка, которая открывает функцию «Что вы хотите сделать»](media/ui-search/search_small.png "Что вы хотите сделать"), введите **Проекты**, затем выберите соответствующую ссылку.
2. Откройте карточку проекта для соответствующего проекта.
3. На экспресс-вкладке **Задачи** заполните требуемые поля в новой строке.
4. Для формирования отступов задач и создания иерархии выберите действие **Задачи**, затем выберите действие **Сдвинуть задачи проекта**.
5. Повторите шаги 3 и 4 для всех задач, которые необходимы для проекта.
6. Для определения задач проекта на основе информации из других задач проекта выберите действие **Копировать задачи проекта из**, заполните поля необходимым образом, а затем нажмите кнопку **ОК**.

## <a name="to-create-planning-lines-for-a-project"></a>Создание строк планирования для проекта

Вы можете уточнить созданные задачи проекта в строках планирования проекта. В строке планирования может быть отражена любая информация, которую вы хотите отслеживать в проекте. Например, вы можете отслеживать ресурсы, которые требуются для проекта, или товары, которые необходимы. Например, у вас есть задача получить от клиента утверждение проекта. Эта задача связывается со строками планирования для товаров, такими как встреча с клиентом и назначение ресурса.  

Строка планирования проекта может быть одного из следующих типов.  

| Тип | Описание |
| --- | --- |
| **Бюджет** |Позволяет определить предполагаемое потребление и затраты для проекта, обычно в случае проекта с оплатой по факту. Выставлять счета по строкам планирования этого типа нельзя. |
| **К оплате** |Позволяет определить оценочный счет для клиента, обычно в случае проекта с фиксированной ценой. |
| **Бюджет и к оплате** |Позволяет определить бюджетное потребление, равное сумме, на которую выставляется счет. |

> [!NOTE]
> При вводе информации в строки планирования проекта данные о себестоимости подставляются автоматически. Например, себестоимость, цена и скидка для ресурсов и товаров определяются на основе информации, заданной в ресурсе и товаре.

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") введите **Проекты**, затем выберите соответствующую ссылку.
2. Откройте соответствующую карточку проекта.
3. Выберите задачу проекта, у которой поле **Тип задачи проекта** содержит значение **Учет**, затем выберите действие **Строки планирования проекта**.  
4. На странице **Строки планирования проекта** на новой строке заполните требуемые поля.
5. Повторите шаги 3 и 4 для всех строк планирования, которые необходимы для задачи проекта.
