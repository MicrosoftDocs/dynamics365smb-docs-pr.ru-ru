# <a name="transfer-inventory-between-locations"></a>Перемещение запасов между местами хранения

Вы можете перемещать складские товары между складами, создав заказы на перемещение. В качестве альтернативы можно использовать журнал реклассификации товаров.

> [!NOTE]
> Для перемещения товаров, следует настроить склады и маршруты перемещения. Чтобы узнать больше о настройке складов, перейдите в раздел [Настройка складов](inventory-how-setup-locations.md). Вы не можете использовать заказы на перемещение для *пустых* складов.

## <a name="transfer-orders"></a>Заказы на перемещение

Вы можете отправить исходящее перемещение из одного склада и получить входящий перемещение в пункте назначения. Можно:

* Отследить количество в пути
* Определите календари, маршруты и время обработки входящих и исходящих запросов для расчета дат и планирования. Чтобы узнать больше о планировании, перейдите в раздел [О функциональности планирования](production-about-planning-functionality.md).
* Используйте разные функции склада для входящих и исходящих складов.
* С некоторыми ограничениями вы можете использовать заказы на перемещение для прямых перемещений.

## <a name="item-reclassification-journals"></a>Журналы реклассификации товаров

С помощью страницы **Журналы реклассификации товаров** можно:

* Осуществлять прямое перемещение товаров между складами.
* Перемещайте товары между ячейками. Дополнительные сведения о перемещении товаров между ячейками см. в разделе [Незапланированное перемещение товаров в базовых конфигурациях склада](warehouse-how-to-move-items-ad-hoc-in-basic-warehousing.md)
* Измените партию или серийный номер на новую партию или серийный номер. Дополнительные сведения о переклассификации серийных номеров и номеров партий см. в разделе [Переклассификация серийных номеров или номеров партий](inventory-how-work-item-tracking.md#to-reclassify-serial-or-lot-numbers).
* Изменять дату истечения срока годности на новую дату.
* Переклассифицировать товары из пустого склада в фактический склад.
* Создавать складские операции, если вы не управляете складскими действиями.

## <a name="to-transfer-items-with-a-transfer-order"></a>Перемещение товаров с помощью заказа на перемещение

1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Заказы на перемещение**, а затем выберите связанную ссылку.
2. На странице **Заказ на перемещение** заполните требуемые поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]

    > [!NOTE]  
    >   После заполнения полей **Код транзитного склада**, **Код экспедитора** и **Услуги экспедитора** на странице **Спец. транс. маршрута** при настройке маршрута перемещения соответствующие поля в заказе на перемещение заполняются автоматически.

    После заполнения поля **Услуги экспедитора** дата приемки на складе-получателе рассчитывается путем добавления времени услуг экспедитора к дате отгрузки.

3. Существует несколько способов заполнить строки:

    |Параметр  |Описание  |
    |---------|---------|
    |Вручную     | На экспресс-вкладке **Строки** заполните строку для товара или используйте действие **Выбрать товары**, если нужно выбрать несколько товаров.        |
    |Автоматически     | * Выберите действие **Получить содержимое ячейки**, чтобы выбрать существующие товары из конкретной ячейки на складе.<br><br>* Выберите **Получить строки прих. накладной**, чтобы выбрать товары, которые только что прибыли на склад отправки.        |

    Теперь вы можете отгрузить товары.
4. Выберите действие **Учет**, выберите параметр **Отгрузить** и нажмите кнопку **ОК**.

    Товары теперь находятся в транзите между указанными складами в соответствии с маршрутом перемещения.

    Как работник склада на складе отправки продолжите получение товаров. Строки заказа на перемещение — такие же, как при отправке, и редактировать их невозможно.
5. Выберите действие **Учет**, выберите параметр **Получить** и нажмите кнопку **ОК**.

### <a name="post-multiple-transfer-orders-in-a-batch"></a>Учет нескольких заказов на перемещение в пакетном режиме

В следующей процедуре поясняется порядок учета заказов на перемещение в пакетном режиме.

1. 1. Выберите значок ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Заказы на перемещение**, а затем выберите связанную ссылку.  
2. На странице **Заказы на перемещение** выберите заказы, которые нужно принять к учету.
3. В поле **Номер** откройте контекстное меню и выберите **Выбрать больше**.
4. Установите флажок в строках для каждого заказа, который вы хотите принять к учету.
5. Выберите действие **Учет**, а затем выберите действие **Пакетный учет**.
6. На странице **Пакетный учет заказа на перемещение** заполните требуемые поля.

   > [!TIP]
    > Для заказов на перемещение, в которых используется транзитный склад, можно выбрать **Отгрузка** или **Приемка**. Повторите этот шаг, если вам нужно сделать и то, и другое. Для заказов, для которых включен **Прямой учет**, оба этих варианта работают одинаково и полностью принимают заказ к учету.

7. Нажмите **ОК**.
8. Чтобы просмотреть потенциальные проблемы, откройте страницу **Регистр сообщений об ошибках**.

    > [!NOTE]
    > Учет нескольких документов может занять некоторое время и заблокировать других пользователей. Рассмотрите возможность включения фонового учета. Дополнительные сведения см. в разделе [Использование очередей работ для планирования задач](/dynamics365/business-central/admin-job-queues-schedule-tasks).

### <a name="schedule-a-job-queue-entry-to-post-multiple-documents-in-a-batch"></a>Планирование операции очереди заданий для принятия к учету нескольких документов в пакетном режиме

В качестве альтернативы, вы можете использовать очередь заданий, чтобы запланировать принятие к учету на удобное для вашей организации время. Возможно, например, что для вашей организации имеет смысл выполнять некоторые процедуры после того, как введено большинство данных за день.

Ниже приводится процедура настройки отчета **Пакетный учет заказов на перемещение** для автоматического учета заказов на прямое перемещение в 16:00 по будним дням. Это время взято исключительно в качестве примера. Действия будут такими же и для других документов.  

1. Найдите страницу **Операции очереди заданий**, а затем выберите связанную ссылку.  
2. Выберите действие **Создать**.  
3. В поле **Тип запускаемого объекта** выберите **Отчет**.  
4. В поле **Номер запускаемого объекта** выберите **5707, Пакетный учет заказов на перемещение**.
5. Установите флажок **Страница запроса отчета**.
6. На странице запроса **Пакетный учет заказов на перемещение** выберите параметр **Отгрузка**, отфильтруйте по **Прямое перемещение**, а затем выберите **ОК**.

   > [!IMPORTANT]
   > Важно установить фильтры. В противном случае [!INCLUDE [prod_short](includes/prod_short.md)] примет к учету все документы, даже если они не готовы. Рассмотрите возможность установки фильтра по полю **Состояние** для значения **Выпущенные**, и фильтр по полу **Дата учета** для значения **..сегодня**. Дополнительные сведения о фильтрах см. в разделе [Сортировка, поиск и фильтрация](/dynamics365/business-central/ui-enter-criteria-filters).

7. Установите все флажки от **Запускать по понедельникам** до **Запускать по пятницам**.
8. В поле **Время начала** введите **16:00**.
9. Выберите действие **Установить статус "Готово"**.

## <a name="to-transfer-items-with-the-item-reclassification-journal"></a>Перемещение товаров с помощью журнала реклассификации товаров

1. Выберите ![Лампочка, которая открывает функцию Что вы хотите сделать.](media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Журналы реклассификации товаров**, а затем выберите связанную ссылку.
2. На странице **Журнал реклассификации товаров** заполните требуемые поля. [!INCLUDE[tooltip-inline-tip](includes/tooltip-inline-tip_md.md)]
3. В поле **Код склада** введите склад, на котором хранятся товары в данный момент.

    > [!NOTE]  
    > Для перемещения товаров без кода склада оставьте поле **Код склада** пустым.
4. В поле **Новый код склада** введите склад, на который требуется переместить товары.
5. Выберите действие **Учет**.

## <a name="undo-a-transfer-shipment"></a>Отмена расходной накладной на перемещение

Если вы обнаружите ошибку в количестве в учтенном заказе на перемещение, вы можете легко исправить количество, пока отгружаемый товар еще не принят. На странице **Учт. расх. накладная на перемещение** предусмотрено действие **Отменить расх. накладную**, которое создает корректирующие строки следующим образом:

* Значение в поле **Отгруженное кол-во** уменьшается на отмененное количество.
* Значение в поле **Кол-во для отгрузки** увеличивается на отмененное количество.
* Для строк устанавливается флажок **Корректировка**.

Если количество было отгружено в складской отгрузке, то в учтенной складской отгрузке создается корректирующая строка.

Чтобы завершить корректировку, повторно откройте заказ на перемещение, введите правильное количество и примите заказ к учету. Если вы используете складскую отгрузку для отгрузки заказа, создайте и примите к учету новую складскую отгрузку.
