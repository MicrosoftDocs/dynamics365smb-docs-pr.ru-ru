# <a name="resources-for-help-and-support"></a>Ресурсы для справки и поддержки

В [!INCLUDE[prod_long](includes/prod_long.md)] пункт меню справки (вопросительный знак в правом верхнем углу) предоставляет доступ к странице **Справка и поддержка**, которая может помочь найти ответы на вопросы. Можно также увидеть, как можно связаться со службой технической поддержки для вашего [!INCLUDE[prod_short](includes/prod_short.md)].  

На этой странице приведен обзор ресурсов, доступных в различных разделах страницы **Справка и поддержка**.  

> [!TIP]
> [!INCLUDE [ua-get-help](includes/ua-get-help.md)]

## <a name="find-answers"></a>Поиск ответов

Раздел **Найти** предоставляет ссылки на наиболее распространенные места для поиска ответов на вопросы. Ссылки подразделяются на четыре категории и объясняются в следующих подразделах. Некоторые ссылки могут быть настроены партнером [!INCLUDE[prod_short](includes/prod_short.md)], и в результате поведение может быть несколько другим.  

### <a name="product-help"></a>Справка продукта

Раздел **Найти** включает две ссылки для справки по продукту:

- Справка о странице, которая только что просматривалась в [!INCLUDE[prod_short](includes/prod_short.md)]  

  Эта ссылка создается автоматически. Если [!INCLUDE[prod_short](includes/prod_short.md)] не может найти подходящую ссылку, отображается целевая страница на сайте [Docs.microsoft.com](index.md) для версии по умолчанию [!INCLUDE[prod_short](includes/prod_short.md)]. Например, если страница была частью приложения, которое не добавляло сопоставление контекстно-зависимой справки в [!INCLUDE[prod_short](includes/prod_short.md)], то для *Ссылки на обучение* по умолчанию используется целевая страница.  
- Целевая страница справки для [!INCLUDE[prod_short](includes/prod_short.md)]  

  Функциональные возможности в версии по умолчанию [!INCLUDE[prod_short](includes/prod_short.md)] описана на сайте [Docs.microsoft.com](/dynamics365/business-central). Для получения дополнительной информации см. раздел [Использование сайта docs.microsoft.com, чтобы узнать больше](#use-the-docsmicrosoftcom-site-to-learn-more). Вы также можете расширить и настроить [!INCLUDE[prod_short](includes/prod_short.md)] с помощью приложений из AppSource. Все приложения предоставляют ссылки на соответствующую справку продукта, чтобы можно было всегда найти указания для различных рабочих процессов в вашей версии [!INCLUDE[prod_short](includes/prod_short.md)].  

### <a name="community"></a>Сообщество

На [форум Business Central](https://community.dynamics.com/business/f) можно отправить вопрос и получить ответ от других участников сообщества [!INCLUDE[prod_short](includes/prod_short.md)]. Партнеры и сотрудники корпорации Майкрософт принимают участие в разговорах.  

Если у вас еще нет партнера, Сообщество также может помочь решить ваши проблемы, если у вас возникли вопросы. Например, если вы только что подписались на пробную версию, Сообщество может стать отличной отправной точкой.  

### <a name="blog"></a>Блог

Мы рассказываем о новых возможностях в [блоге Dynamics 365](https://cloudblogs.microsoft.com/dynamics365/it/product/business-central/), включая наличие каких-либо особенностей.

Выберите ссылку для просмотра плана последнего выпуска. Можно также получить обзор ожидающихся и недавно выпущенных возможностей в [планах выпуска Dynamics 365](/dynamics365/release-plans/).  

Для более долгосрочного обзора страница [дорожной карты](https://dynamics.microsoft.com/roadmap/business-central/) [!INCLUDE[prod_short](includes/prod_short.md)] показывает общие приоритеты для [!INCLUDE[prod_short](includes/prod_short.md)] в ближайшие годы.  

## <a name="do-more-with-your-trial"></a>Расширить возможности пробной версии

В этом разделе предоставлены ярлыки в Microsoft AppSource, где можно найти решения и сервисы от партнеров [!INCLUDE[prod_short](includes/prod_short.md)]. Используйте эти ссылки для знакомства с полными возможностями [!INCLUDE[prod_short](includes/prod_short.md)] перед принятием решения о приобретении подписки. Также можно найти ссылку для связи с рабочей группой продаж.

> [!NOTE]
> Этот раздел доступен только для пробной версии. После приобретения [!INCLUDE[prod_short](includes/prod_short.md)] этот раздел больше не отображается.

## <a name="give-feedback"></a>Отправка отзыва

На [сайте предложений для Dynamics 365](https://aka.ms/bcideas) вы можете предложить новые функции и возможности. Ваше сообщение направляется непосредственно в инженерный журнал невыполненных работ Business Central для изучения и определения приоритетов.  

Обязательно выполните поиск в списке отправленных предложений. Возможно, что кто-то уже предлагал что-то подобное, и эта запись уже могла получить голоса. Проголосуйте, если предложение уже было отправлено, чтобы повысить его приоритет в журнале невыполненных работ по разработке.  

## <a name="troubleshooting"></a>Устранение неполадок

В разделе **Устранение неполадок** приводятся технические сведения, которые администратор или партнер [!INCLUDE[prod_short](includes/prod_short.md)] может использовать для разблокировки пользователей. Сюда входит информация о самом последнем сообщении об ошибке, текущей версии [!INCLUDE[prod_short](includes/prod_short.md)] и ссылка на страницу **Инспекция страницы**. Дополнительную информацию см. в разделе [Инспекция страниц в Business Central](across-inspect-page.md).  

Как внутренний администратор вы имеете доступ к центру администрирования, где вы можете продолжить устранение неполадок. Вы можете просматривать телеметрию, управлять средами и управлять сеансами пользователя. Дополнительные сведения см. в [Поиск технической информации](/dynamics365/business-central/dev-itpro/administration/manage-technical-support#finding-technical-information) и [Управление средами](/dynamics365/business-central/dev-itpro/administration/tenant-admin-center-environments) в содержимом по администрированию.  

## <a name="report-a-problem"></a>Сообщить о проблеме

Ваш партнер по распространению [!INCLUDE[prod_short](includes/prod_short.md)] будет предоставлять вам техническую поддержку. В этом разделе можно легко связаться с вашим партнером по распространению, если он предоставил свой адрес электронной почты службы поддержки. Если ваш партнер [!INCLUDE[prod_short](includes/prod_short.md)] не может разрешить проблему, он создаст билет поддержки в компании Майкрософт. Для получения дополнительной информации см. [Передача проблем с поддержкой в корпорацию Майкрософт](/dynamics365/business-central/dev-itpro/administration/manage-technical-support#escalating-support-issues-to-microsoft) в материалах для администраторов.  

> [!NOTE]
> Этот раздел доступен только при наличии оплаченной подписки [!INCLUDE[prod_short](includes/prod_short.md)]. Если в подписались на пробную версию и пока еще не приобрели [!INCLUDE[prod_short](includes/prod_short.md)], у вас нет продавца-партнера, и вы не можете связаться с группой поддержки компании Майкрософт. Вы можете найти партнера [!INCLUDE[prod_short](includes/prod_short.md)] на странице [Я ищу поставщика решений](https://go.microsoft.com/fwlink/?linkid=2038145). Для получения дополнительной информации см. раздел [Как найти партнера по перепродаже?](/dynamics365/business-central/across-faq#findpartner).  

### <a name="support-contacts"></a>Контакты службы поддержки

В разделе **Сообщить о проблеме** отображаются адрес электронной почты или гиперссылка, с помощью которых вы можете связаться с человеком или группой, отвечающими за поддержку клиентов. Если вы являетесь партнером по распространению [!INCLUDE[prod_short](includes/prod_short.md)] для арендатора, вы можете указать свои предпочтительные контактные данные для поддержки в центре администрирования. Дополнительные сведения см. в разделе [Настройка технической поддержки](/dynamics365/business-central/dev-itpro/technical-support#configuring-the-support-experience) в содержимом по администрированию.  

Чтобы уведомить своего партнера о проблеме, выберите ссылку, чтобы отправить сообщение электронной почты или открыть сайт поддержки на новой вкладке браузера. После этого вы сможете добавить техническую информацию о своем арендаторе Azure Active Directory из раздела **Устранение неполадок** и любую другую информацию, которую сочтете необходимой.

### <a name="content-on-the-site"></a>Содержимое на сайте

[В этом разделе](index.md) сайта docs.microsoft.com размещается содержимое от Microsoft, которое представляет основу *Узнать больше* в модели помощи пользователю. Частично содержимое *Узнать больше* предназначено для ответа на те вопросы, на которые пользовательский интерфейс не может ответить, например следующие:

- Как эта страница вписывается в более крупный рабочий процесс?  
- Что будет дальше?  
- Какая была бы альтернатива?  

Изнутри продукта вы можете получить доступ к этому содержимому либо через ссылку *Узнать больше* во всплывающих подсказках и обучающих подсказках или с помощью сочетание клавиш *Ctrl + F1*.  

Панель навигации слева, оглавление, группировка статей по областям бизнеса. Например, вы читаете статью [Покупка товаров для продажи](purchasing-how-purchase-products-sale.md), а затем оглавление показывает, что есть также статья под названием [Исправление или отмена неоплаченных счетов покупки](purchasing-how-correct-cancel-unpaid-purchase-invoices.md). Обе статьи находятся в группе под названием *Покупки*, которая объединяет товары, связанные с модулем покупок, в [!INCLUDE [prod_short](includes/prod_short.md)].  

В оглавление входит группа содержимого, описывающая [локальные функции](about-localization.md) для стран, которые поддерживает Microsoft. Если вы не видите свою страну в списке, возможно, это связано с тем, что ваша страна локализована партнером. Дополнительные сведения см. в [Наличие в странах/регионах и поддерживаемые языки](/dynamics365/business-central/dev-itpro/compliance/apptest-countries-and-translations).  

Содержимое локальной функциональности описывает возможности в [!INCLUDE [prod_short](includes/prod_short.md)], что может помочь вам соответствовать требованиям на поддерживаемых рынках, таких как налоговые отчеты и другие отчеты. В некоторых случаях разница между версией по умолчанию и версией для конкретной страны невидима для пользователей. В других случаях пользователи должны настроить отчетность или электронные счета определенным образом. Если вы не можете найти описание определенной кнопки или действия в общем разделе, вы можете найти его в разделе *Локальная функциональность*.  

### <a name="finding-other-content"></a>Поиск другого содержимого

Если вы попали на статью, которая не отвечает на ваш вопрос, вы можете найти другой содержимое двумя способами:

- Использовать поле **Фильтровать по заголовку** в оглавлении в левой части веб-сайта

  Таким образом, вы можете фильтровать слова, используемые в заголовках различных статей. Например, введите *счет*, а затем выберите одну из предложенных ссылок, например [Настройка округления счета](finance-set-up-invoice-rounding.md) или [Создание счетов на предоплату](finance-how-to-create-prepayment-invoices.md).  

  Под каждой ссылкой вы можете увидеть, где находится эта статья в оглавлении. Таким образом, вы можете легко увидеть, находится ли статья, например, в разделе «Покупки» или в разделе «локальная функциональность Австралии».  
- Использовать поле **Поиск** в правом верхнем углу веб-сайта

  Таким образом, вы можете искать любой термин. Затем сайт будет искать весь содержимое для этого термина или выражения. Выполните поиск по *счет* вновь и пролистайте список сотен статей в содержимом [!INCLUDE [prod_short](includes/prod_short.md)], который использует это слово где-то в тексте.  

### <a name="languages"></a>Языки

Содержимое о бизнес-функциональности публикуется на нескольких языках. Когда вы заходите на сайт docs.microsoft.com из продукта, вы видите содержимое на том же языке, который вы указали в [!INCLUDE [prod_short](includes/prod_short.md)], если этот язык не доступен на сайте docs.microsoft.co..  

Вы можете изменить язык, изменив URL-адрес, или использовать переключатель языка в левом нижнем углу веб-сайта.  

Содержимое написано на английском языке и опубликовано по URL-адресу с *en-us* как код языка. Когда новая информация публикуется на английском языке, она отправляется для перевода на поддерживаемые языки. В некоторых случаях вы не сможете читать этот новый содержимое на своем родном языке до четырех недель, но в большинстве случаев это занимает всего одну или две недели. Вы всегда можете просмотреть последнюю информацию, переключив язык на *en-us*.  

Для каждого языка оглавление переводится, но содержание в разделе *Локальная функциональность* на английском языке, за исключением страны, которая соответствует текущему языку. Например, в нидерландской версии сайта docs.microsoft.com (`https://docs.microsoft.com/da-dk/dynamics365/business-central/`), вы можете увидеть записи для австралийской локальной функциональности на нидерландском языке в оглавлении, но статьи на английском языке. Только статьи для нидерландской локальной функциональности переведены на нидерландский язык.  

### <a name="recommended-content"></a>Рекомендуемое содержимое

В марте 2021 года под большинством статей на сайте docs.microsoft.com появился новый раздел. Раздел **Рекомендуемое содержимое** создается автоматически в зависимости от того, как каждая статья используется с остальной частью сайта. Ссылки не статичны и со временем будут меняться.

### <a name="tell-us-what-you-think"></a>Поделитесь вашим мнением

На docs.microsoft.com каждая статья имеет две кнопки в конце статьи. Кнопка *Этот продукт* отправляет вас на сайт предложений, а кнопка *Эта страница* позволяет отправить отзыв о содержимом через GitHub. В обоих случаях вы должны создать учетную запись, если у вас ее еще нет. Для *Отзыв о продукте*, вы должны войти в систему с вашей рабочей или организационной учетной записью электронной почты. Для *доступ к GitHub*, вы можете использовать любой адрес электронной почты при создании учетной записи.  

Мы приветствуем ваши вклады как в виде запросов на получение предложений с предложениями или исправлениями к содержимому, так и в виде проблем GitHub с ошибками или вопросами. Но помните, что отзывы и вклады в *dynamics365smb-docs* о содержимом, а не о продукте.  

> [!IMPORTANT]
> Microsoft принимает запросы на получение только в *dynamics365smb-docs*, а не репозитории для конкретного языка. Если у вас есть отзывы о переводах, вы можете сообщить о проблеме GitHub в соответствующем репозитории.
