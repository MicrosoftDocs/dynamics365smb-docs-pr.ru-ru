# <a name="about-calculating-standard-cost"></a>О расчете стандартной себестоимости

Многие производственные организации выбирают в качестве базы оценки стандартную себестоимость. Это также верно для организаций, которые занимаются легким производством, таким как сборка и комплектация. Система стандартной себестоимости определяет себестоимость единицы товара на основе некоторой обоснованной исторической или ожидаемой себестоимости. Изучает прошлые и оценочные будущие данные себестоимости создает основу для стандартных себестоимостей. Эти себестоимости замораживаются до тех пор, пока не будет принято решение об их изменении. Фактическая стоимость производства продукта может отличаться от оценочной стандартной себестоимости. Для контроля управления фактическая себестоимость сравнивается со стандартной для конкретного товара и разности (*вариации*) определяются и анализируются.  

Стандартные себестоимости могут поддерживаться для товаров, которые пополняются во время покупки, сборки и производства. Для каждого метода пополнения стандартные стоимости могут состоять из следующих элементов.  

|Метод пополнения|Элементы стандартной себестоимости|  
|--------------------------|----------------------------|  
|**Покупка**|Прямая себестоимость материалов и накладные расходы на материалы, при необходимости.|  
|**Сборка**|Прямая себестоимость материала, прямые или фиксированные затраты на труд и накладные расходы.|  
|**Произв. заказ**|Прямая себестоимость материала, затраты на труд, затраты на субподрядчиков и накладные расходы.|  

## <a name="setting-up-standard-costs"></a>Настройка стандартных себестоимостей

Поскольку стандартная себестоимость произведенного или собранного товара может состоять из нескольких элементов себестоимости, включая материалы, производственные мощности (труд), а также прямые и накладные расходы на субподряд, необходимо установить стандартную себестоимость для каждого из этих элементов.  

Задача учета для производственной организации, использующей стандартную себестоимость, состоит из следующих этапов:  

- Оценка стандартной себестоимости для готового товара и ее настройка на карточке товара.  
- Запишите и выделите фактическую себестоимость основных элементов себестоимости и объяснение отклонений.  

Чтобы определить прямую себестоимость готового товара, необходимо суммировать себестоимости всех компонентов. Собранный или произведенный товар может содержать подкомпоненты, которые также состоят из нескольких компонентов.  

Следующие основные элементы себестоимости составляют общую прямую себестоимость завершенного обработанного товара.  

- Себестоимость материалов.  
- Себестоимость произв. мощностей  
- Затраты на субподряд только на произведенные товары.  

### <a name="material-costs"></a>Затраты на материалы

Себестоимость материалов — это затраты, связанные с подкомпонентами и приобретаемым сырьем. Себестоимость единицы материала может состоять из прямых и косвенных элементов себестоимости.  

- Прямая себестоимость материала представляет собой выставленную в счете сумму для приобретенного сырья или себестоимость обработки подкомпонента.  
- Косвенные затраты на материал, или *накладные расходы*, могут, например, представлять затраты на хранение готового товара после его производства.  

Настройка себестоимости материала для приобретенных товаров в отношении прямых и косвенных затрат зависит от выбранного для данного товара метода учета себестоимости. Независимо от метода учета себестоимости информация о себестоимости задается на карточке товара. Дополнительные сведения см. в разделе [Регистрация новых товаров](inventory-how-register-new-items.md).

Себестоимость отходов (только производственных) является еще одним фактором, который следует учитывать при расчете общей себестоимости материала. Когда определенное количество сырья выбраковывается при сборке или производстве изделия, это обычно приводит к росту количества компонентов, требуемых для производства данного изделия. Это увеличивает себестоимость материала компонентов, которые потребляются при производстве родительского товара. Себестоимость отходов материала можно настроить в производственной спецификации или маршруте.  

Себестоимость материала производимого товара можно представить двумя способами, которые соответствуют следующим стандартным типам расчета себестоимости.  

|База для расчета себестоимости|Расчет себестоимости материалов|  
|----------------------------|-------------------------------|  
|Одноуровневый|Произведенный товар равен общей себестоимости всех приобретенных или собранных компонентов в спецификации данного товара.|  
|Свернутый уровень или многоуровневый|Произведенный товар равен сумме себестоимости материалов всех подкомпонентов в спецификации данного товара и себестоимости всех приобретенных товаров в производственной спецификации данного товара.|  

### <a name="capacity-costs"></a>Затраты на производственные мощности

Затраты на производственные мощности — это затраты, связанные с внутренними затратами на труд и оборудование. Эти себестоимости необходимо настроить для каждого ресурса (в управлении сборкой) и рабочего или машинного центра на маршруте (в производстве). Как и с материалами, можно определить как прямые, так и косвенные элементы себестоимости производственной мощности. Например, прямая себестоимость для рабочего центра может быть установленной цеховой ставкой за выполнение определенной функции. К косвенным затратам для рабочего центра можно отнести некоторые общезаводские расходы, например, на освещение, отопление и т.д. Как и с себестоимостью материалов, можно выразить накладные затраты на мощность в виде процента косвенных затрат или фиксированной нормы накладных расходов.  

Настройка затрат на производственные мощности собранных товаров состоит из следующих элементов:  

- Прямая и косвенная себестоимость единицы ресурса.  
- Фиксированный или прямой тип использования ресурса.  

Настройка затрат на производственные мощности произведенных товаров состоит из следующих элементов:  

- Прямая и косвенная себестоимость единицы машинного или рабочего центра.  
- Настройка времени и размера партии.  

Для расчета стандартной себестоимости производственных мощностей следует установить стандартные нормы времени, необходимые для выполнения операций в машинных и рабочих центрах. Общее время для выполнения операции обычно состоит из времени настройки и выполнения, а также времени ожидания и перемещения.  

Нормы времени для каждого из этих типов времени настраиваются для каждого машинного или рабочего центра на отдельных маршрутах.  

> [!NOTE]  
>  Хотя нормы времени выполнения применяются к каждой единице произведенного товара, настройка нормы времени выполнения применяется к каждой партии. Поэтому время настройки маршрута для каждой операции должно быть пропорционально распределено по размеру партии. Размер партии указывается в соответствующем поле на экспресс-вкладке **Пополнение** страницы **Карточка товара**.  

Чтобы указать время настройки на маршруте для планирования, но не включать этот расход в расчет стандартной себестоимости, снимите флажок в поле **Себест. вкл. наладку** на странице **Настройка модуля "Производство"**.  

При одноуровневом расчете это затраты на труд, необходимые для производства готового товара и указываемые в маршруте товара. При многоуровневом расчете это себестоимость производственных мощностей, указанная для каждого отдельно произведенного товара, включенного в спецификацию основного товара.  

### <a name="subcontractor-costs"></a>Затраты на субподряд

Затраты на субподряд — затраты, связанные с услугами, предоставляемыми организации сторонними поставщиками или субподрядчиками. Аналогично затратам на материал и производственные мощности, затраты на субподряд могут содержать как прямые затраты, так и накладные расходы. Прямые затраты на субподряд представляют фактические расходы на единицу представленных услуг. Например, накладные расходы на субподряд могут представлять собой затраты на транспорт и погрузку, начисленные организации в связи с заказом на субподряд.  

Поскольку субподряд представляет внешние (аутсорсинговые) производственные мощности, себестоимость прямых и косвенных услуг субподряда настраивается в карточке рабочего центра, которая представляет операцию субподряда.  

## <a name="updating-standard-costs"></a>Обновление стандартных себестоимостей

Чтобы обновить или рассчитать стандартную себестоимость сборочных элементов, используйте функцию из карточки товара.  

Процесс обновления или расчета стандартной себестоимости обычно состоит из следующих задач:  

1.  Обновление себестоимости на уровне компонента и производственных мощностей. Дополнительные сведения см. в пакетных заданиях **Предложить станд. себестоимость товара** и **Предложить стандартную себестоимость объема**.  
2.  Консолидация и сведение себестоимости компонентов и производственных мощностей для расчета общей себестоимости сборки или производства товаров. Дополнительные сведения см. в статье [Вычисление стандартной себестоимости сборочного элемента](assembly-how-work-assembly-boms.md#to-calculate-the-standard-cost-of-an-assembly-item).  
3.  Внедрение стандартной себестоимости, введенной при запуске предыдущих пакетных заданий. Стандартные себестоимости не вступают в силу до ввода их в действие. Используйте пакетное задание **Выполнить изменения станд. себестоимости**, которое обновляет изменения стандартной стоимости товаров в соответствии с изменениями в таблице стандартных затрат.  
4.  Реализация изменений для обновления поля **Себестоимость единицы** в карточке товара и выполнение переоценки товара на складе. Дополнительные сведения см. в разделе [Переоценка запасов](inventory-how-revalue-inventory.md).

## <a name="use-batch-jobs-to-update-standard-costs"></a>Использование пакетных заданий для обновления стандартных себестоимостей
В следующих разделах описаны пакетные задания, которые можно использовать для обновления стандартных себестоимостей.
### <a name="suggest-item-standard-cost"></a>Предложить станд. себестоимость товара

 Создает предложения по изменению затрат и долей затрат в стандартных себестоимостях по карточкам товаров. По завершении пакетного задания его результат отобразится в окне «Журнал станд. себестоимостей».

> [!NOTE]  
> Это пакетное задание предназначено только для приобретенного товара. Если требуется обновить товар с производственной спецификацией или сборочной спецификацией, необходимо ввести в рабочий журнал все компоненты, а затем запустить пакетное задание "Сворачивание стандартной себестоимости".

Это пакетное задание только создает предложения. Оно не вводит предложенные изменения в действие. Если предложения по изменениям являются приемлемыми и вы хотите ввести их в действие, то есть внести в карточки товаров и вставить в журнал переоценки, выберите "Выполнить изменения станд. себестоимости" в окне "Журнал станд. себестоимостей".
#### <a name="options"></a>Параметры

**Стандартная себестоимость**: введите коэффициент, который вы хотите использовать для обновления стандартной себестоимости. Можно также выбрать метод округления для новой стандартной себестоимости. Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

**Косвенные затраты (%)**: введите коэффициент, который вы хотите использовать для обновления процента косвенных затрат. Можно также выбрать метод округления для новой косвенной затраты (%). Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

**Норма накладных расходов**: введите коэффициент, который вы хотите использовать для обновления накладных расходов. Можно также выбрать метод округления для новой ставки накладных расходов. Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

### <a name="suggest-workmach-ctr-std-cost"></a>Предлож. станд. себест. рабочего/машинного центра

Создает предложения по изменению затрат и долей затрат в стандартных себестоимостях по карточкам рабочих центров, машинных центров и ресурсов. По завершении пакетного задания его результат отобразится в окне **Журнал станд. себестоимостей**.

Это пакетное задание только создает предложения. Оно не вводит предложенные изменения в действие. Если предложения по изменениям являются приемлемыми и вы хотите ввести их в действие, то есть внести в карточки рабочих/машинных центров и ресурсов и вставить в окно "Журнал переоценки", выберите **Выполнить изменения станд. себестоимости** в окне **Журнал станд. себестоимостей**.

Если после выполнения пакетного задания вы хотите просмотреть его влияние на производственные или сборочные участки, запустите пакетное задание **Сворачивание стандартной себестоимости**, чтобы обновить стандартную себестоимость в рабочих центрах, машинных центрах, сборочных ресурсах, производственных спецификациях и сборочных спецификациях.
#### <a name="options-1"></a>Параметры
**Стандартная себестоимость**: введите коэффициент, который вы хотите использовать для обновления стандартной себестоимости. Можно также выбрать **метод округления** для новой стандартной себестоимости. Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

**Косвенные затраты (%)**: введите коэффициент, который вы хотите использовать для обновления процента косвенных затрат. Можно также выбрать метод округления для новой косвенной затраты (%). Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

**Норма накладных расходов**: введите коэффициент, который вы хотите использовать для обновления накладных расходов. Можно также выбрать метод округления для новой ставки накладных расходов. Для процентного увеличения необходимо ввести в поле десятичный знак, например, 1,1.

### <a name="post-inventory-cost-to-gl"></a>Учет себестоимости запасов в ГК

 Записывает изменения количества и стоимости запасов в операциях книги товаров и операциях стоимости при учете складских транзакций, таких как расходные или приходные накладные.

Если в окне **Настройка модуля "Запасы"** не установлен флажок **Автомат. учет себест.**, себестоимости товаров на складе не регистрируются динамически в главной книге и себестоимости продажи товаров не рассчитываются при продаже. Следовательно, необходимо выполнить учет в главной книге вручную путем выполнения пакетного задания **Учет себест. запасов в ГК** для обновления главной книги и потенциальной печати отчета об учтенных стоимостных операциях.

Пакетное задание использует стоимостные операции как основу. Для определения учитываемой стоимости вычисляется разница между полями **Сумма себестоимости (факт.)** и **Стоимость, учтенная в ГК** стоимостных операций. Если в окне **Настройка модуля "Запасы"** установлен флажок **Учет ожидаемой себест. в ГК** то в пакетном задании также учитывается разница между полями **Сумма себестоимости (ожид.)** и **Ожид. стоимость, учтенная в ГК** на промежуточных счетах в главной книге.

> [!NOTE]
> Если флажок **Ожид. стоимость, учтенная в ГК** в окне **Настройка модуля "Запасы"**, то последний раздел отчета будет содержать стоимостные, которые пропускаются, поскольку представляют ожидаемые себестоимости.

> [!NOTE] 
> Если при выполнении пакетного задания происходит ошибка, связанная с измерениями или настройкой измерений, пакетное задание ее обходит и выполняет учет в ГК с помощью измерений стоимостных операций.

Чтобы убедиться в том, что данное пакетное задание выполняется без ошибок, можно запустить отчет **Учет себест. запасов в ГК - тест** для просмотра всех возможных ошибок. Затем можно устранить ошибки и запустить пакетное задание **Учет себест. запасов в ГК,** зная, что оно обработает все записи.
 
> [!IMPORTANT]  
> Перед использованием данного пакетного задания следует выполнить пакетное задание **Коррекция себест. запасов**. Затем, при запуске этого пакетного задания, обновятся значения себестоимости, которые будут учтены в Главной книге.
#### <a name="options-2"></a>Параметры

|Оператор сравнения  |Описанием  |
|--------------|---------|
|**Метод учета**|При выполнении этого пакетного задания можно учесть стоимость запасов в главной книге по группам учета или по учтенным операциям. При учете по операциям дается полное описание влияния складских запасов в ГК, но при этом вы получаете очень много операций ГК. При учете по группам учета пакетным заданием создается операция ГК для каждого сочетания даты и группы учета. Это означает, что операция ГК создается для каждого сочетания даты учета, общей группы учета по деятельности, общей группы учета по продуктам, группы учета по складским запасам и кода местоположения. Кроме того, пакетным заданием будут созданы отдельные операции ГК для затрат с различными знаками.|
|**Номер документа**|В этом поле можно ввести номер документа, если выбран параметр "Учет по учетной группе товаров". Номер документа будет присутствовать в учтенных операциях.|
|**Учет**|Выберите это поле для автоматического учета в главной книге при выполнении пакетного задания. Если нет необходимости учитывать себестоимость товаров в ГК, то в пакетном задании будет распечатан только пробный отчет со значениями, которые можно учитывать в Главной книге; в отчете появится следующее сообщение: **Тестовый отчет (не учтен)**.|

### <a name="roll-up-standard-cost"></a>Сворачивание стандартной себестоимости

Свертывает стандартные себестоимости собранных и произведенных товаров. На эти значения будет влиять изменение стандартной себестоимости компонентов, предлагаемое пакетным заданием **Предложить станд. себестоимость товара**. Кроме того, на них влияет изменение стандартной себестоимости производственных мощностей и ресурсов сборки, предлагаемых пакетным заданием **Предлож. станд. себест. рабочего/машинного центра**.

Если были выполнены оба этих пакетных задания или одно из них, при выполнении свертки все изменения стандартных себестоимостей, сделанные в рабочем журнале, вводятся в соответствующие производственные или сборочные спецификации, и себестоимости применяются на каждом уровне спецификации.

> [!NOTE] 
> Эта функция выполняет свертку стандартной себестоимости только в карточках товаров, но не в карточках единиц хранения.

Это пакетное задание только создает предложения. Оно не вводит предложенные изменения в действие. Если предложения по изменениям являются приемлемыми и вы хотите ввести их в действие, то есть внести в карточки товаров и вставить в окно **Журнал переоценки**, для этого можно воспользоваться пакетным заданием **Выполнение изменения станд. себестоимости**. Данное пакетное задание можно вызвать в окне **Журнал стандартных себестоимостей**.
#### <a name="options-3"></a>Параметры

**Дата расчета**: введите дату, которая используется для той версии спецификации, для которой предполагается выполнить свертку.
 
### <a name="implement-standard-cost-change"></a>Выполнение изменения станд. себестоимости

Обновляет изменения в стандартной себестоимости в таблице **Товар** в соответствии с изменениями в таблице **Журнал станд. себестоимостей**. Предложения по изменению стандартной стоимости могут быть созданы с помощью пакетного задания **Предложить станд. себестоимость товара** и/или **Предлож. станд. себест. рабочего/машинного центра**, и их также можно изменять. Перемещается содержимое всех полей в предложениях по изменению стандартной себестоимости. После ввода в действие предложений по изменениям стандартных себестоимостей их можно увидеть в карточке товара и/или карточках рабочего/машинного центра. Создается также журнал переоценки для обновления стоимости имеющегося запаса.
#### <a name="options-4"></a>Параметры

**Дата учета**: введите дату проведения переоценки.

**№ документа**: введите номер строки журнала переоценки. Если в коде раздела товарного журнала задана серия номеров, номер документа будет следовать за учтенными операциями, полученными при учете журнала переоценки. В противном случае здесь можно ввести номер вручную.

**Шаблон журнала товаров**: введите название шаблона журнала товаров.

**Код раздела журнала товаров**: введите название текущего журнала товаров

Выберите **ОК** для запуска пакетного задания. Если сейчас пакетное задание выполнять не нужно, выберите **Отмена**, чтобы закрыть окно.
