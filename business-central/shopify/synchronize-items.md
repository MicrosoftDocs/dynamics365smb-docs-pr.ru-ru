# <a name="synchronize-items-and-inventory"></a>Синхронизируйте товары и запасы

**Предметы** в [!INCLUDE[prod_short](../includes/prod_short.md)] эквивалентны *товарам* в Shopify, включая физические товары, цифровые загрузки, услуги и подарочные карты, которые вы продаете. Есть две основные причины для синхронизации товаров:

1. В первую очередь управление данными осуществляется в [!INCLUDE[prod_short](../includes/prod_short.md)], вам нужно экспортировать все или некоторые данные в Shopify и сделать их видимыми. Вы можете экспортировать название товара, описание, изображение, цены, наличие, варианты, информацию о поставщике и штрих-код. После импорта товары могут отображаться сразу или сначала можно активировать предварительный просмотр для внесения изменений ответственным лицом.
2. При импорте заказа из Shopify информация о товаре необходима для дальнейшей обработки документа в [!INCLUDE[prod_short](../includes/prod_short.md)].

Эти два сценария всегда включены.

Третий сценарий заключается в управлении данными в Shopify, но пакетом импорте этих товаров в [!INCLUDE[prod_short](../includes/prod_short.md)]. Этот сценарий может быть полезен для событий переноса данных, когда вы хотите связать существующий интернет-магазин с новой средой [!INCLUDE[prod_short](../includes/prod_short.md)].

## <a name="to-define-item-synchronizations"></a>Определение синхронизации товаров

1. Выберите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, вход в магазин **Shopify**. Откройте магазин, для которого вы хотите настроить синхронизацию товаров.
2. В поле **Синхронизировать товар** выберите нужный вариант. <br>В следующей таблице описывается разница между параметрами.

|Параметр|Описанием|
|------|-----------|
|**Пусто**| Продукция импортируется вместе с импортом заказов. Продукция экспортируется в Shopify, когда пользователь выполняет действие **Добавить элемент** со страницы **Shopify Товары**. Этот процесс применяется по умолчанию. |
|**На Shopify**| Выберите этот параметр, если после первоначальной синхронизации, инициированной действием **Добавить элемент** вы планируете обновлять продукцию вручную с помощью действия **Синхронизировать продукт** или через очередь заданий для повторяющихся обновлений. Не забудьте активировать поле **Можно обновить продукцию Shopify**. Если оно не активировано, оно равно варианту **Пусто**. Дополнительные сведения см. в разделе [Экспорт товаров в Shopify](synchronize-items.md#export-items-to-shopify)|
|**С Shopify**| Выберите этот вариант, если вы планируете импортировать продукцию из Shopify оптом; либо вручную с помощью действия **Синхронизировать продукцию** либо через очередь заданий для повторяющихся обновлений. Если никакой вариант не выбран, оно равно варианту **Пусто**. Дополнительные сведения об импорте товаров см. в разделе [Импорт товаров из Shopify](synchronize-items.md#import-items-from-shopify)|

## <a name="import-items-from-shopify"></a>Импорт товаров из Shopify

Если вы импортируете товары из Shopify оптом или вместе с импортом заказов эти позиции сначала добавляются к продукции **Shopify** и варианту таблицы **Shopify**. Следующие настройки позволяют управлять процессом:

|Поле|Описанием|
|------|-----------|
|**Автоматически создавать неизвестные товары**|Когда продукция и варианты Shopify импортируются в [!INCLUDE[prod_short](../includes/prod_short.md)], функция [!INCLUDE[prod_short](../includes/prod_short.md)] всегда пытается сначала найти соответствующую запись в списке товаров. **Сопоставление SKU** влияет на то, как выполняется сопоставление, и создает новый товар и/или вариант товара. Включите этот параметр, если вы хотите создать новый товар или если соответствующей записи не существует. Новый товар будет создан с использованием импортированных данных и **Кода шаблона товара**. Если этот вариант не активирован, вам нужно будет создать товар вручную и использовать действие **Сопоставить продукцию** на странице **Продукция Shopify**.|
|**Код шаблона товара**|Используется вместе с параметром **Автоматически создавать неизвестные товары**. <br> Выберите шаблон, который будет использоваться для автоматически созданных товаров.|
|**Сопоставление SKU**|Выберите, как вы хотите использовать значение **SKU**, импортированное из Shopify во время сопоставления и создания товара или варианта. Для получения дополнительной информации см. раздел [Как SKU и штрихкод, определенные для продукции Shopify, влияют на сопоставление и создание товаров и вариантов](synchronize-items.md#how-skus-and-barcodes-defined-in-shopify-product-affects-mapping-and-creation-of-items-and-variants-in-business-central)|
|**Разделитель полей SKU**|Используется вместе с **Сопоставлением SKU**, установленным для варианта **№ товара + код варианта**.<br> Определите разделитель, который следует использовать для разделения SKU. <br>Например, если в Shopify вы создаете вариант с SKU "1000/001", введите "/" в поле **Разделитель полей SKU**, чтобы получить номенклатурный номер в [!INCLUDE[prod_short](../includes/prod_short.md)] как "1000", а код варианта товара как "001".
|**Префикс варианта**|Используется вместе с **Сопоставлением SKU**, установленным на **Код варианта** или вариантами **№ товара + код варианта** в качестве запасной стратегии, когда SKU, поступающий из Shopify, пустой.<br>Если вы хотите создать вариант товара в [!INCLUDE[prod_short](../includes/prod_short.md)] автоматически, вам нужно будет ввести значение в поле **Код**. По умолчанию используется значение, указанное в поле SKU, импортированное из Shopify. Однако, если поле SKU пусто, будет сгенерирован код, начинающийся с определенного префикса варианта и "001".|
|**Shopify может обновить товар**| Выберите этот вариант, если вы хотите автоматически обновлять товары и/или варианты.|

### <a name="how-skus-and-barcodes-defined-in-shopify-product-affects-mapping-and-creation-of-items-and-variants-in-business-central"></a>Как SKU и штрих-коды, определенные для продукции Shopify влияют на сопоставление и создание товаров и вариантов в Business Central

Когда товары ввозятся из Shopify в **Товары Shopify** и таблицы **Вариантов Shopify**, [!INCLUDE[prod_short](../includes/prod_short.md)] пытается найти существующие записи. В следующей таблице описывается разница между параметрами в поле **Сопоставление SKU**.

|Параметр|Влияние на сопоставление|Влияние на создание|
|------|-----------------|------------------|
|**Пусто**|Поле SKU не используется в процедуре сопоставления товаров.|Не влияет на создание товара. <br>Предотвращает создание вариантов. Эта функция полезна, когда в заказе на продажу используется только основной товар. Вариант по-прежнему можно сопоставить вручную, используя окно **Продукция Shopify**.|
|**Код товара**|Выберите этот вариант, если поле SKU содержит номер товара.|Не влияет на создание товара без вариантов. Для товара с вариантами каждый вариант создается как отдельный товар.<br> Например, если у Shopify есть продукт с двумя вариантами, а их SKU – "1000" и "2000", то в [!INCLUDE[prod_short](../includes/prod_short.md)] система создаст два товара с номерами "1000" и "2000".|
|**Код варианта**|Поле SKU не используется в процедуре сопоставления товаров.|Не влияет на создание товара. При создании варианта товара в качестве кода используется значение поля SKU. Если поле SKU пусто, код генерируется из данных в поле **Префикс варианта**.|
|**№ товара + код варианта**| Выберите этот вариант, если поле SKU содержит номер товара и код варианта товара, разделенный значением, определенным в поле **Разделитель полей SKU**.|При создании товара первая часть значения поля SKU используется как **Номер**. Если поле SKU пусто, то номер товара генерируется с использованием числового ряда, определенного в **Код шаблона элемента** или **Номера товаров** в окне **Настройка запасов**.<br>При создании товара функция варианта использует вторую часть значения поля SKU как **Код**. Если поле SKU пусто, код генерируется из данных в поле **Префикс варианта**.|
|**№ товара поставщика**| Выберите, содержит ли поле SKU номер поставщика товара. В этом случае **Номер поставщика товара** используется не в окне **Карточка товара**, а в окне **Номер поставщика товара** в **Каталоге поставщиков товаров**. Если найденная запись из *Каталога поставщиков товаров* содержит код варианта, этот код варианта используется для сопоставления вариантов Shopify.|Если соответствующий поставщик существует в [!INCLUDE[prod_short](../includes/prod_short.md)], значение SKU будет использоваться как **Номер поставщика товара** в **Карточке предмета** и как **Ссылка на элемент** по типу Продавец. <br>Предотвращает создание вариантов. Это полезно, когда вы хотите использовать основной товар только в заказе на продажу. Вы по-прежнему сможете сопоставить вариант вручную, используя окно **Продукция Shopify**.|
|**Штрихкод**| Выберите этот вариант, если поле SKU содержит штрихкод. Поиск осуществляется среди **Ссылок на товары** по типу Продавец. Если найденная запись из Ссылок на товары содержит код варианта, этот код варианта будет использован для сопоставления вариантов Shopify.|Не влияет на создание товара. <br>Предотвращает создание вариантов. Эта функция полезна, когда в заказе на продажу используется только основной товар. Вариант по-прежнему можно сопоставить вручную, используя окно **Продукция Shopify**.|

В следующей таблице показано влияние поля **Штрихкод**.

|Влияние на сопоставление|Влияние на создание|
|-----------------|------------------|
|Поиск осуществляется среди **Ссылок на товары** по типу штрих-кода для значения, определенного в поле Штрих-код в Shopify. Если найденная запись из Ссылок на товары содержит код варианта, этот код варианта будет использован для сопоставления вариантов Shopify.|Штрих-код сохраняется как **Ссылка на товар** для товара и варианта товара.|

> [!NOTE]  
> Вы можете инициировать сопоставление для выбранного продукта/вариантов или всех импортированных несопоставленных продуктов, выбрав **Попробовать найти сопоставление продуктов** (для избранных) или **Попробовать найти сопоставления** (для всех).

## <a name="export-items-to-shopify"></a>Экспорт товаров в Shopify

Выберите элементы из вашего списка товаров, которые необходимо экспортировать в Shopify. Используйте действие **Добавить товар** на экране **Товары Shopify** для добавления товаров в список продуктов Shopify.

Следующие настройки позволяют управлять процессом экспорта товаров:

|Поле|Описанием|
|------|-----------|
|**Ценовая группа клиентов**|Определите цену товара в Shopify. Берется продажная цена этой ценовой группы клиентов. Если группа не введена, используется цена карточки товара.|
|**Скидочная группа клиентов**|Определите скидку, которая будет использоваться для расчета цены товара в Shopify. Цены со скидкой сохраняются в поле **Цена**, а полная цена хранится в поле **Сравнить по цене**.|
|**Синхронизировать расширенный текст товара**|Выберите, чтобы синхронизировать расширенный текст товара. Когда он будет добавлен в поле *Описание*, он может содержать HTML-код. |
|**Синхронизировать атрибуты товара**|Выберите для синхронизации атрибутов товара. Атрибуты отформатированы в виде таблицы и включены в поле *Описание* на Shopify.|
|**Код языка**|Если выбран этот параметр, переведенные версии используются для заголовка, атрибутов и расширенного текста.|
|**Сопоставление SKU**|Выберите, как вы хотите заполнить поле SKU в Shopify. Среди поддерживаемых вариантов:<br> - **Номер товара** – для использования номера товара как для продуктов, так и для вариантов.<br> - **Номер товара + код варианта** – для создания SKU путем объединения значений двух полей. Для товаров без вариантов используется только номер товара.<br>- **Номер поставщика товара** – используется номер поставщика товара, определенный в *Карточке товара* как для продуктов, так и для вариантов.<br> - **Штрихкод** – используется **Ссылка на товар** по типу штрихкода. Эта опция учитывает варианты. |
|**Разделитель полей SKU**|Определите разделитель для варианта **Номера товара + код варианта**.|
|**С прослеживанием запасов**|Выберите, как система должна заполнять поле **Отслеживание запасов** для продукции, экспортируемой в Shopify. Вы можете обновить информацию о доступности из [!INCLUDE[prod_short](../includes/prod_short.md)] для продукции в Shopify, для которой активировано отслеживание запасов. Дополнительные сведения см. в разделе [Запасы](synchronize-items.md#sync-inventory-to-shopify).|
|**Политика запасов по умолчанию**|Выберите *Отклонить* для предотвращения отрицательных запасов со стороны Shopify. |
|**Может обновлять продукцию Shopify**|Определите, может ли [!INCLUDE[prod_short](../includes/prod_short.md)] создавать исключительно товары, либо может также обновлять их. Выберите этот параметр, если после первоначальной синхронизации, инициированной действием **Добавить товар**, вы планируете обновлять продукцию вручную с помощью действия **Синхронизировать продукт** или через очередь заданий для повторяющихся обновлений. Не забудьте выбрать **В Shopify** в поле **Синхронизация предметов**.|
|**Код шаблона клиента**|Выберите шаблон по умолчанию, который будет использоваться при расчете цены. Дополнительные сведения см. в разделе [Настройка налогов](setup-taxes.md).|

### <a name="tags"></a>Теги

Просмотрите импортированные теги на информационной панели **Теги** на странице **Продукция Shopify**. Для редактирования тегов выберите действие **Теги** на странице **Продукция Shopify**.
Если выбран вариант **В Shopify** в поле **Синхронизировать элемент**, то назначенные теги экспортируются в Shopify при следующей синхронизации.

## <a name="run-item-synchronization"></a>Запуск синхронизации товаров

Полная или частичная синхронизация товаров может выполняться разными способами.

### <a name="initial-sync-of-items-from-business-central-to-shopify"></a>Первоначальная синхронизация товаров из Business Central в Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Продукция Shopify** и выберите связанную ссылку.
2. Выберите действие **Добавить товары**.
3. В поле **Код магазина** укажите код. Если вы откроете окно **Продукция Shopify** из окна **Карта магазина**, то код магазина будет заполнен автоматически.
4. При необходимости определите фильтры для товаров. Например, вы можете фильтровать по номеру товара или коду категории товара.
5. Выберите **ОК**.

В результате товары автоматически создаются в Shopify с ценами, но изображения и уровни запасов не включены. Операция может занять некоторое время, если добавляется большое количество товаров.

### <a name="sync-products-from-shopify-to-business-central"></a>Синхронизировать продукты из Shopify в Business Central

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **магазин Shopify** и выберите связанную ссылку.
2. Выберите магазин, для которого вы хотите синхронизировать товары, чтобы открыть страницу **Карточка магазина Shopify**.
3. Выберите действие **Синхронизировать продукцию**.

В качестве альтернативы вы можете использовать действие **Синхронизировать продукцию** в окне **Продукция Shopify** или найти пакетное задание **Синхронизация продукции**.

Вы можете запланировать задачу для автоматического выполнения. Дополнительные сведения см. в разделе [Планирование повторяющихся задач](background.md#to-schedule-recurring-tasks).

### <a name="ad-hock-updates-of-shopify-products"></a>Ad-Hock обновления продукции Shopify

При обновлении записей в таблице **Продукция Shopify** следующие изменения синхронизируются с Shopify.

Обновить:

* **Статус** – вы можете выбрать между активным, заархивированным или черновиком.
* **Должность директора**
* **Описание директора**

Удаление:

На основании значения в поле **Действия для удаленной продукции** в **Карточке магазина Shopify**, продукция обновляется в Shopify при удалении записи со страницы **Продукция Shopify**.

* **Пусто** – ничего не произойдет. При необходимости пользователю необходимо выполнить требуемое действие со страницы администратора Shopify.
* **Статус черновика** – статус продукции в Shopify установлен на *Черновик*.
* **Статус в архиве** – товар находится в архиве Shopify.

## <a name="sync-item-images"></a>Синхронизировать изображения товара

Синхронизацию изображений можно настроить для синхронизированных товаров. Можно выбрать один из следующих параметров.

* **Пусто** – синхронизация изображений отключена.
* **В Shopify** – фотографии товаров экспортируются в Shopify
* **Из Shopify** – изображения из Shopify импортируются в [!INCLUDE[prod_short](../includes/prod_short.md)]

Синхронизация изображений может быть инициализирована двумя способами.

### <a name="sync-product-images-from-the-shopify-shop-window"></a>Синхронизируйте изображения продукции из окна магазина Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Магазины Shopify** и выберите связанную ссылку.
2. Выберите магазин, для которого вы хотите синхронизировать изображения, чтобы открыть страницу **Карточка магазина Shopify**.
3. Выберите действие **Синхронизировать изображения продукции**.

### <a name="sync-product-images-from-the-shopify-products-window"></a>Синхронизируйте изображения продукции из окна продукции Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Продукция Shopify** и выберите связанную ссылку.
2. Выберите действие **Синхронизировать изображения продукции**.

### <a name="image-synchronization-remarks"></a>Замечания по синхронизации изображений

* При экспорте изображений из [!INCLUDE[prod_short](../includes/prod_short.md)] в Shopify, новые изображения добавляются в Shopify, при этом старые изображения остаются нетронутыми. Если изображение обновляется в [!INCLUDE[prod_short](../includes/prod_short.md)], вам нужно будет удалить старые изображения со страницы администратора Shopify.
* Изображения, которые экспортируются в Shopify и не соответствуют требованиям, установленным Shopify, не будут импортированы. Для получения дополнительной информации см. [Медиатипы продукции на help.shopify.com](https://help.shopify.com/en/manual/products/product-media/product-media-types#images)

## <a name="sync-prices-with-shopify"></a>Синхронизировать цены с Shopify

Цены на синхронизированные товары можно экспортировать способами, описанными ниже.

### <a name="sync-prices-from-the-shopify-products-window"></a>Синхронизируйте цены из окна продукции Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Продукция Shopify** и выберите связанную ссылку.
2. Выберите действие **Синхронизировать цены с Shopify**.

### <a name="price-calculation-remarks"></a>Замечания по расчету цены

* Для расчета цены важно, чтобы было указано значение в поле **Стандартный шаблон клиента**. Дополнительные сведения см. в разделе [Настройка налогов](setup-taxes.md).
* Введите **Код валюты**, если ваш интернет-магазин использует валюту, отличную от LCY. Для указанной валюты должен быть настроен обменный курс. Если ваш интернет-магазин использует ту же валюту, что и [!INCLUDE[prod_short](../includes/prod_short.md)], оставьте поле пустым.
* При определении цены [!INCLUDE[prod_short](../includes/prod_short.md)] использует логику "Самая низкая цена". Это означает, что если цена за единицу, указанная в карточке товара, ниже цены, определенной в ценовой группе, используется цена за единицу из карточки товара.

## <a name="sync-inventory-to-shopify"></a>Синхронизировать запасы с Shopify

Синхронизацию запасов можно настроить для уже синхронизированных товаров. Есть два условия, которые должны быть соблюдены:

1. Отслеживание запасов должно быть включено для продукции в Shopify. Если товары экспортируются в Shopify, попробуйте активировать переключатель **Отслеживание запасов** в **Карточке магазина Shopify**. Дополнительные сведения см. в разделе [Экспорт товаров в Shopify](synchronize-items.md#export-items-to-shopify).
2. Синхронизация запасов должна быть включена для **Места Shopify**.

### <a name="to-enable-inventory-sync"></a>Чтобы включить синхронизацию инвентаря

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **магазин Shopify** и выберите связанную ссылку.
2. Выберите магазин, для которого вы хотите синхронизировать запасы, чтобы открыть страницу **Карточка магазина Shopify**.
3. Выберите действие **Места**, чтобы открыть **Адреса магазинов Shopify**.
4. Выберите действие **Получить места Shopify** для импорта всех местоположений, определенных в Shopify. Вы можете найти их в меню [**Места**](https://www.shopify.com/admin/settings/locations) настроек вашей страницы **администратора Shopify**.
5. В поле **Фильтр мест** добавьте места, если вы хотите включить запасы только из определенных мест. Например, вы можете ввести *ВОСТОК|ЗАПАД*, и тогда запасы только из этих двух мест будут доступны для продажи через интернет-магазин.
6. Деактивируйте переключатель **Отключить**, чтобы включить синхронизацию запасов для выбранных мест Shopify.

Синхронизация запасов может быть запущена двумя способами.

### <a name="sync-inventory-from-the-shopify-shop-window"></a>Синхронизируйте запасы из окна магазина Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Магазины Shopify** и выберите связанную ссылку.
2. Выберите магазин, для которого вы хотите синхронизировать запасы, чтобы открыть страницу **Карточка магазина Shopify**.
3. Выберите действие **Синхронизация запасов**.

### <a name="sync-inventory-from-the-shopify-products-window"></a>Синхронизируйте запасы из окна продукции Shopify

1. Введите в поиске ![Лампочка, которая открывает функцию "Что вы хотите сделать".](../media/ui-search/search_small.png "Что вы хотите сделать") значок, введите **Продукция Shopify** и выберите связанную ссылку.
2. Выберите действие **Синхронизация запасов**.

### <a name="inventory-remarks"></a>Замечания по запасам

* Коннектор вычисляет **Прогнозируемый доступный баланс** и экспортирует его в Shopify.
* Вы можете просмотреть информацию об акциях, полученную от Shopify на странице **Информационная панель запасов Shopify**. На этой информационной панели вы получите обзор на функцию запасов Shopify и последний рассчитанный запас в [!INCLUDE[prod_short](../includes/prod_short.md)]. Там размещена запись по местам.
* Если информация о запасах в Shopify отличается от **Прогнозируемого доступного баланса** в [!INCLUDE[prod_short](../includes/prod_short.md)], тогда запас будет обновлен в Shopify.
